SOFTWARE REQUIREMENTS SPECIFICATION (IEEE 830)
Project: Khiladi - AI-Powered Cricket Bowling Video Analysis App
================================================================================

1. INTRODUCTION
1.1 Purpose
This document details the project vision, software requirements, and UML designs for Khiladi, an Android application that uses AI and ML to analyze cricket bowling videos and provide constructive feedback.

1.2 Scope
The app allows users to upload bowling videos, which are processed by AI to identify technical flaws and generate constructive feedback. It aims to help amateur and semi-professional cricketers improve their skills.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- ML: Machine Learning
- API: Application Programming Interface

2. OVERALL DESCRIPTION
2.1 Product Perspective
Khiladi empowers cricketers to self-assess and improve their bowling techniques by providing immediate, actionable feedback, filling the void left by conventional, lengthy coaching methods.

2.2 Product Functions (Summary)
- User account creation and management.
- Video upload functionality.
- AI-driven video analysis to identify mistakes.
- Personalized feedback generation.
- Progress tracking over time.
- Notifications and reminders.
- Connect with coaches (paid feature).
- Community support.

2.3 User Classes and Characteristics
- Users (Cricket Players): Primary users who will upload videos and receive feedback.
- Admin: Manages the portal, adds/removes users, adds incentives, adds coaches.

2.4 Operating Environment
Android mobile application.

3. FUNCTIONAL REQUIREMENTS
3.1 Create Account
FR1.1: The system shall allow users to create an account by adding necessary details (name, email, phone number).

3.2 Customize Profile
FR2.1: The system shall allow users to customize their profile by adding details (age, experience, bowling style, injury status).

3.3 Input Video
FR3.1: The system shall allow users to input a video of their bowling action for analysis.

3.4 Identify Mistakes
FR4.1: After video upload, the system shall analyze the video and identify mistakes in the bowling technique.

3.5 Generate Feedback
FR5.1: After identifying mistakes, the system shall generate feedback providing solutions and corrections.

3.6 Create Personalized Plans
FR6.1: The system shall allow users to create personalized plans and progress goals based on their ease and availability.

3.7 Track Progress Report
FR7.1: The system shall allow users to track their progress based on past performance.

3.8 Get Notifications
FR8.1: The system shall send users notifications (e.g., deadlines for re-inputting video, friend chats).

3.9 Get Motivational Quotes
FR9.1: The system shall provide users with motivational quotes.

3.10 Connect with Coach
FR10.1: The system shall allow users to connect with specialized coaches (paid feature).

3.11 Provide Incentives
FR11.1: The system shall incentivize users based on their performance and progress.

3.12 Create Events
FR12.1: The system shall allow users to create events with other users (e.g., set a match).

3.13 Connect with other users
FR13.1: The system shall allow users to chat with other users, share feedback, and talk.

3.14 Manage Portal
FR14.1: The system shall allow the admin to manage the portal (add/remove users, add incentives, add coaches).

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance
NFR1: Users should be able to log in within 5 seconds.
NFR2: Uploaded videos should be processed and provide feedback within 1-2 minutes.
NFR3: Notifications about new feedback shall be received within 10 seconds after analysis completion.
NFR4: The application must handle multiple users simultaneously.

4.2 Security
NFR5: Personal data must be encrypted and securely stored during transmission and storage.

4.3 Compatibility
NFR6: The app will be compatible with Android devices running version 6.0 or higher.
