1.3 Modules
Two significant modules have been identified in this project for handling customer service queries
and escalating more complex requests. Each module plays a vital role in efficiently managing user
interactions and ensuring a smooth flow within the banking support system.
1.3.1 Module 1: Customer Query Handling
This module primarily handles routine customer queries through interaction with a virtual assistant
named LUCY. The Assistant communicates with customers via voice to process and respond to
common banking inquiries, such as balance checks, account opening info, and card-related issues.
By automating simple, repetitive tasks, the module reduces the workload on human agents and
ensures prompt responses.
Features
1. Speech Command Capture and Recognition in Urdu
1. The system accurately captures and recognizes speech in Urdu using Natural Language Pro-
cessing (NLP), allowing customers to communicate in their native language. This enhances
the accessibility for Urdu-speaking customers
2. Tools Used: Google Cloud Speech-to-Text API, spaCy (or OpenNLP for natural language un-
derstanding). (tools may vary)
2. Real-time Response for Common Queries
1. LUCY provides immediate responses to frequently asked banking queries, such as balance
inquiries, account opening info etc.
2. Tools Used: Machine learning models integrated with banking APIs to provide real-time re-
sponses.
3. Automated Resolution of Frequently Asked Questions (FAQs)
1. LUCY is designed to automatically identify and resolve the most frequently asked questions
without human intervention, reducing the workload on human agents.

1.4 User Classes and Characteristics
The project identifies several key user classes, each with distinct needs and characteristics:
1. Banking Customers:
The primary users of LUCY are customers, especially those who are Urdu-speaking and prefer
native language interactions. They will use the system for routine banking queries such as
balance checks and transaction histories. The virtual assistant’s ability to provide quick, real-
time responses in Urdu will improve customer satisfaction and make banking services more
accessible to this demographic.
2. Banking Institutions:
The banks implementing LUCY will benefit from reduced operational costs and increased ef-
ficiency in customer service. The scalability of the system allows banks to handle a higher
volume of customer interactions without increasing human resource requirements. This, in
turn, leads to improved service levels and better customer retention.
3. Customer Support Agents:
Human agents will handle complex queries that LUCY cannot resolve. By automating routine
queries, LUCY will reduce the workload on these agents, enabling them to focus on more
critical and complex issues. This will lead to more efficient use of human resources and a
higher standard of service for complex inquiries.
4. Development and Operations Team:
This group includes the developers, system administrators, and IT staff responsible for the
creation, deployment, and maintenance of the LUCY virtual assistant. They will ensure that

2.2 Functional Requirements
2.2.1 Module 1: Customer Query Handling
Voice Command Capture
1. The system must capture and process voice commands in Urdu.
2. The system should utilize a speech recognition API to convert voice input to text.
7
Chapter 2. Project Requirements NUST, Islamabad
Query Recognition
1. The system should recognize predefined banking queries, such as:
label=. Checking account balance.
lbbel=. Retrieving transaction history.
lcbel=. Reporting lost or stolen cards.
ldbel=. Activating a card.
2. The system must also differentiate between simple and complex queries based on keywords.
Real-time Response Generation
1. The system should generate real-time response for recognized queries.
2. These responses should be clear and concise, providing relevant information to the customer.
User Feedback Mechanism
1. The system should allow users to provide feedback on the accuracy of the responses they
receive.
2. User feedback should be logged to improve the system’s performance.
Error Handling
1. The system must handle unrecognized commands gracefully, by prompting users to repeat or
rephrase their queries.
2. The system should log instances of unrecognized queries for future training of the model.
Session Management
1. The system should maintain session context, allowing users to ask follow-up questions with-
out repeating their initial requests.
2. The session should timeout after a predefined period of inactivity.
Solution Impact
• Efficiency: Automating responses to common queries reduces customer wait times, improving
overall customer satisfaction.
• Accessibility: The assistant bridges the language gap for Urdu-speaking customers, providing
them with an interface they are comfortable with for banking operations.
• Scalability: LUCY can becomes an international solution for global banks that face problem
dealing urdu-speaking customers.

2.2.2 Module 2: Escalation to Human Agent
Complex Query Detection
1. The system must identify complex queries that require human intervention based on specific
triggers or patterns in user requests.
2. Queries should be analyzed for keywords and context to determine complexity.
Seamless Handoff Process
The system must provide a seamless transition to a human agent, sharing the user’s context and
previous interactions to avoid repetition.
Integration with Customer Support System
The system must integrate with the bank’s existing customer support infrastructure to efficiently
handle escalated queries. This should include logging details about the interaction for customer
service representatives.
Notification System
The system should notify users when their query is being escalated to a human agent and provide
an estimated wait time for a response.
Agent Assignment
The system should assign escalated queries to available human agents based on their expertise and
availability. Agents should receive notifications about new escalated queries assigned to them.
Feedback Loop
After the escalation, the system should prompt users to rate their experience with the human agent.
Feedback should be logged for continuous improvement of both the virtual assistant and human
agent interactions.
Solution Impact
• Improved Customer Experience: By ensuring that complex queries are handled by human
agents, customers receive the necessary support without feeling frustrated by the limitations
of the virtual assistant.
• Efficiency in Operations: This module allows LUCY to handle a large volume of routine
queries while ensuring that complex issues are escalated appropriately, reducing the overall
workload on human agents.

2.3 Non-Functional Requirements
2.3.1 Reliability
• Uptime Requirement: The system should guarantee at least 99.5% uptime, ensuring that cus-
tomers experience consistent service availability.
• Error Recovery: The system must be equipped with mechanisms to recover quickly from fail-
ures or crashes, minimizing service disruption. Automatic recovery should start within 2 min-
utes.
• Data Integrity: The system must maintain the integrity of data during transactions, ensuring
all transactions are recorded and verified without any data loss or corruption.
2.3.2 Usability
• User Interface: The interface should be simple and intuitive, specifically designed for Urdu-
speaking customers to minimize the learning curve. Users should be able to find information
within three clicks or fewer.
• Accessibility: The system must meet accessibility standards to be usable by individuals with
disabilities, such as visual impairments, and include features like text-to-speech and speech-
to-text functionality.
• User Support: Comprehensive help documentation should be available to guide users through
common tasks and help troubleshoot issues without needing external assistance.
2.3.3 Performance
• Response Time: The system should respond to 95% of user queries within 3 seconds, en-
suring a smooth user experience. This standard should also apply to voice recognition with
maintained accuracy.
• Scalability: The system must be capable of handling up to 10,000 simultaneous users without
any drop in performance, ensuring the infrastructure can scale with growing demand.
• Load Testing: The system must undergo rigorous load testing to maintain performance during
peak traffic times, such as month-end banking activities.
