1.2 Scope
The project encompasses the design and construction of a mobile robot that can detect
and put out fires autonomously in indoor environments, as well as triggering alarms and
reporting its state to a web application. The robot operates on the XY plane and will
be able to safely traverse complex indoor implied spaces while avoiding obstacles and
stairs. The robot will include fire, gas, and smoke detection sensors to detect fire hazards
early on. Once a fire is detected, the robot will be able to safely autonomously navigate
to the fire source using obstacle detection and avoidance algorithms, and then extinguish
the fire using its fire extinguishing arm. In addition to extinguishing fires, the robot will
also generate a map of the room using embedded sensors and cameras, assisting it with
navigation and identifying hazardous items. The system will have real-time notification
alerts that trigger alarms and alerts a web-based application for remote monitoring, alert-
ing, and logging of incidents. It will also be integrated with existing fire alarm systems
to provide a timely response in emergencies. The robot will need to be very vigilant and
continuously monitor the environment for gas and smoke as an extra precaution in deadly
situations. The robot is designed to operate autonomously allowing reduced human in-
tervention and allowing barriers to be removed from dangerous situations. In summary,
the project scope includes fire detection, autonomous navigation, obstacle avoidance, fire
exhausting, and online monitoring alerts for comprehensive and complete fire protection
in sensitive indoor environments.

1.3.1 Module 1
This module includes camera-based room mapping, and obstacle identification. A camera
generates a 2D room map which identifies obstacles for effective path planning. A simple
user-friendly interface which uses the map, obstacles, and optimal paths the robot uses
for navigation.
1. Room Mapping: The camera is mounted on the wall or ceiling, performing a scan
to generate a 2D map of the area. It uses the feed from the camera to recognize
obstacles and their locations thus providing clear vision around the room while
planning paths for the robot to take around obstacles.
2. Path Planning: The system computes the best path for the camera to move for nav-
igating around a room based on the information gathered from detected obstacles.
3. User Interface: A basic GUI showing the map of the room and the obstacles in that
room along with all possible paths the robot may have. (Admin Web Application)
1.3.2 Module 2
This module focuses on advanced fire detection and response mechanisms. Involves a
combination of machine learning models for real-time fire detection and takes the robot
to the fire location for suppression. System comprises a superior navigation system that
avoids obstacles and a user interface for fire logs, manual control, and reporting.
1. Fire Detection: Implement fire detection by using advanced ML models and camera-
based analysis. Accurately increase the precisions of real-time fire detection.
2. Hardware Integration: Develop the fully functional hardware of the FireCar.
3. Better Navigation: Advanced navigation algorithms that will enable the robot to
make adjustments dynamically and travel to the position of the fire quickly by im-
plementing reinforcement learning.
4. User Interaction and Reporting: The user interface needs to be improved with the
input of fire logs, suppression actions, and manual control options.
5. Driven Response: System is guided automatically by the robot to the detected loca-
tion of the fire for corrective action such as triggering of a mechanism which douses
the fire through perhaps the CO2 system.

System Overview
This project focuses on developing an object-oriented automated system that integrates
real-time fire detection, room mapping, and obstacle identification. Utilizing advanced
camera-based technology, the system ensures rapid alerts and response mechanisms for
fire hazards, improving safety in critical environments like data centers and server rooms.
Each feature of the system, such as room mapping and fire suppression, is designed as
an object with its specific properties and methods, facilitating modularity and scalability.
Key functionalities include accurate room layouts, efficient navigation, and automated fire
suppression.
3.1 Architectural Design
The architectural design for the project consists of two main modules:
1. Room Mapping and Obstacle Identification
2. Fire Detection and Response Mechanisms
3.1.1 Modular Structure and Relationships:
1. Camera-Based Room Mapping Module:
• This module scans the environment using camera sensors to create a detailed
map of the room. It also identifies and marks the locations of obstacles within
the room to aid in navigation
• Relationships: The data from this module is sent to the Navigation System to
ensure safe movement throughout the environment, helping to avoid obstacles.

3.3 Data Design
3.3.1 Real-time Data Capture
As previously mentioned, overhead cameras provide live video feeds that are crucial for
creating maps, detecting fires, and spotting obstacles.
3.3.2 Data Structures
Frame Data: Each video frame or processed image can be stored in MongoDB as a docu-
ment, which includes metadata (such as timestamp and image format) along with essential
data (like fire detection status).
Map and Environment Data: The occupancy grid map generated from camera feeds can
also be saved in a JSON-like format in MongoDB, allowing for quick access and up-
dates when the environment changes. For example, each room’s map can be saved as a
document with fields for grid dimensions, obstacle locations, and fire spots.
Fire and Obstacle Data: MongoDB is ideal for storing objects like detected fires and ob-
stacles, enabling you to maintain a collection of fire detection incidents, each represented
by a document. This data can later be queried for reporting or used in real-time decision-
making.
3.3.3 Data Processing
Processed frames can be indexed and stored in MongoDB, while maps and obstacles are
updated in real-time as MongoDB documents.
3.3.4 Data Organization with MongoDB
Collections: MongoDB organizes data into collections, making it easy to separate differ-
ent data types. For instance:
• A frames collection can hold data related to real-time images.
• A maps collection can store occupancy grid maps for each room.
