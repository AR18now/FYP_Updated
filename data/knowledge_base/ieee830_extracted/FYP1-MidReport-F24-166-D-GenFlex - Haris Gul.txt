1.2 Scope
The goal of GenFlex is to give educational institutions a digital platform that completely
automates the examination process. With the use of AI technology, the platform would
enable teachers to easily create and assess exams. The features listed below will be sup-
ported by GenFlex:
1. Making of lecture notes and exam papers digitally.
2. Digital exams that students can take via web interface.
3. Text/Descriptive, multiple-choice, code, and math question’s grading that is auto
mated utilizing sophisticated AI models like LLMs including coding libraries, math
models.
4. Multiple user access: Safe and scalable infrastructure that manages numerous ex-
ams which at once will be available to teachers and students alike.
The project’s main goal is to automate exams for educational institutions so that they can
use a seamless digital platform devoid of the effect of human grading. Teachers may
focus on teaching as GenFlex ensures fair, accurate, and consistent grading in addition to
saving time.

2.4 Non-Functional Requirements
2.4.1 Reliability
1. The GenFlex system shall have a Mean Time Between Failures (MTBF) of at least
168 hours (7 days) of continuous operation during the initial prototype phase.
2. A failure is defined as any unplanned system downtime or inability to access critical
functionalities (exam creation, taking, or result viewing) for more than 10 minutes.
3. In the failure case, the system will automatically save all user progress made in the
last 2 minutes to ensure that current ongoing exams are not disturbed and reduce
any data loss.
4. The system will keep track of the errors and performance problems and then allow
them to be manually reviewed and checked accordingly. Automated detection and
alerts may be added in upcoming versions.
5. If a failure occurs, the system will automatically recover and restore services within
30 minutes. Manual intervention will only be needed when needed urgently.
2.4.2 Usability
1. The GenFlex system shall allow teachers to create a standard exam with at least 20
questions within 60 minutes on their first use.
2. Students shall be able to navigate and start an assigned exam within 5 clicks from
the main dashboard on Student tab/window.
3. The system shall provide basic error messages(error prompt messages) if user put
wrong inputs.
4. The system will let users easily undo their last action in form fields, either by us-
ing familiar keyboard shortcuts (ctrl+z or a convenient "undo" button), wherever
possible.
12
2.4 Non-Functional Requirements
2.4.3 Performance
1. The system shall support up to 100 concurrent users simultaneously during the
initial phase, with a goal of expanding its limit upto 1000 users as the system is
optimized.
2. 80 percent of all database queries, including those for retrieving exam questions
and saving responses, shall complete in less than 1 second.
3. The AI evaluation module shall complete the grading of a standard 50 questions
with mixedup format exam (MCQs and short answers only) within 5 minutes.
4. The system shall render the user interface for exam-taking within 5 seconds of a
page request under normal load conditions.
5. Auto-saving of responses during exams shall occur every 60 seconds, ensuring user
interaction is not interrupted if any kind of failures occur like electricity outage etc.
2.4.4 Security
1. The system shall use TLS 1.2 or might be a higher one to keep all data secure
and safe during transmission, and sensitive data stored in the system shall be pro-
tected by AES-128 encryption at first, with plans to upgrade to AES-256 in future
versions.
2. Teachers and administrators shall use single-factor authentication (like a password)
in initial release, and multi-factor authentication shall be added later in later up-
dates.
3. The system shall be designed to handle common security issues, following the
OW ASP Top 10 instructions,.
4. All user actions shall be held for the audits and the checkups to keep the track of
activities done while interacting with the system throughout the session.
5. The system shall lock accounts of the user if 10 failed login attempts occur within
the 30 minutes time frame, along with the option for the users to recover their
account through some email-based process.
6. The system shall make sure the password policies make strong passwords, with
a requirement of having the minimum of 12 characters, including uppercase Al-
phabets, lowercase Alphabets, numerics, and special characters, with mandatory
changes every 90 days.

3.1 Architectural Design
GenFlex uses a layered architecture with five main layers:
1. Presentation Layer: Web and mobile interfaces
2. Application Layer: Teacher, Student, and Admin modules
3. Service Layer: AI Evaluation
4. Data Access Layer: Database and file system operations
5. Data Layer: MongoDB and file storage
Key modules include:
• Teacher Module: Exam creation and management
• Student Module: Exam taking and result viewing
• AI Evaluation Module: Automated grading using LLMs and specialized libraries
• Exam Management: Handles exam lifecycle
• Security and Authentication: Ensures system-wide data protection
