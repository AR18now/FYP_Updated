## 1. Introduction

1. Introduction
1.2 Scope
This project comprehensively collects, processes, and analyses data in real time in re-
sponse to the needs of the Urdu-speaking farming population within Pakistan. A mobile
application enables workers in the field to record conversations with farmers that can later
be transcribed and analyzed using machine learning models built in Python. It will have
an online database comprising agricultural information, and problems which are visually
represented through our web interface. The other facilities included in this system are an
admin panel for running extensive data queries and filtering for displaying relevant infor-
mation thereby making decisions based on logical reasoning, hence improving farmers’
production ability and making them more productive.
1.3 Modules
Our project, Kisan Rabta, aims to transform agriculture data collection and analysis to use
advanced modern-day technologies by integrating with field personnel and admin man-
agement. There will be two main modules of our project. One is the mobile app, and
the other is the dynamic Web interface. Our mobile has simple functionalities that will
streamline the data collection process to collect the data accurately and in real-time. This
data is then stored in our database for further operations. Here comes our machine learn-
ing models into action. They will use this data to transcribe the extensive recordings into
text, and then by using the agriculture data, we will remove the noise from the data and
keep only agriculture-related things. The goal of each Module of our project is to make
efficiency improvements and simplify the workflow. Following is a detailed description
of our modules with relevant features.
1.3.1 Client Mobile App
Field agents can install an application on their phones that gives them a simple interface
for recording conversations in Urdu. This tool gathers data about agriculture within a
timeline.
1. Secure Login: To make the system more secure, there will be an authentication
mechanism for secure login. To this end, the client cannot sign up directly through
the app. Only the admins can create login credentials for the field personnel, who
can then log in using these credentials.
2. Profile Management: Users can access their profiles’ information and carry out
profile management tasks, such as changing their profile pictures.

For the second module of our project “Kisan Rabta,” we are building a dynamic web

graphs, charts, and maps will be provided. Users can see the agriculture data spe-

3.1 Architectural Design
1. Presentation Layer (User Interface)
The Presentation Layer manages user interactions across both mobile and web plat-
forms. It serves as the interface for agents and administrative users to interact with
the system, facilitating seamless communication and user engagement.
2. Application Layer (Business Logic)
The Application Layer handles the core business logic of the system, including
voice recording, transcription processing, and user management. It is also respon-
sible for invoking machine learning models to transcribe audio inputs and extract
relevant agricultural information.
3. Data Layer
The Data Layer ensures that all collected and processed data is securely stored and
easily accessible for future use, maintaining data integrity and availability across
the system.
4. Machine Learning Layer
The Machine Learning Layer contains models for audio-to-text conversion and
agricultural data extraction. It interacts with the Application Layer to provide
transcription-based analysis and insights that aid in decision-making and informa-
tion processing.

3.3 Data Design
Storage, Processing, and Organization of Major Entities
1. Farmers Collection:
• Storage: Each Farmers information is stored in the farmers document having
attribute Unique identifier, Name etc.
• Processing: When a farmer interacts with the system i.e records audio con-
versation, their information can be easily accessible by the system.
• Organization: All farmer documents are stored in the Farmers collection.
2. Agents Collection:
• Storage: Each agents information is stored in the agents document having
attribute Unique identifier, Name, Role, Contact Information etc.
• Processing: When an agent interacts with the system i.e records audio con-
versation or pushes the recording onto the database, their information can be
easily accessible by the system.
• Organization: All agent documents are stored in the agents collection.
3. Audio Recordings Collection:
• Storage: Each Audio Recordings information is stored in the Audio Record-
ings document having attributes like RecordingID, FilePath, RecordingDate,
Duration, and references to FarmerID and AgentID.
• Processing: When a recording is created, it generates a new document with
the name Audio Recording linking it to the respective farmer and agent. This
allows for efficient retrieval of all related recordings.
• Organization: All audio recording documents are stored in the Audio Record-
ings collection.
4. Transcriptions Collection:
• Storage: Each Transcriptions information is stored in the Transcriptions doc-
ument having attributes like TranscriptionID, Text, Timestamp, and a refer-
ence to RecordingID.
• Processing: Each transcription can be generated from its corresponding au-
dio recording, helping the system maintain a clear link for processing and
retrieval.
• Organization: All transcription documents are stored in the Transcriptions
collection.
5. Agricultural Data Collection:
21
3. System Overview
• Storage: This collection contains documents with DataID, Crop Type, Dis-
ease, Treatment, and a reference to TranscriptionID.
• Processing: Data is extracted from transcriptions to create records of agricul-
tural information, facilitating analysis and reporting.
• Organization: All agricultural data entries are maintained in the Agricultural
Data collection.
6. Reports Collection:
• Storage: Each Reports information is stored in the Reports document having
attributes like ReportID, ReportDate, Summary, and a reference to DataID.
• Processing: Reports can be generated based on the agricultural data, summa-
rizing the findings for farmers or agents.
• Organization: All report documents are stored in the Reports collection.
