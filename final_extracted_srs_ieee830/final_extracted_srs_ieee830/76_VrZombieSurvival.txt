SOFTWARE REQUIREMENTS SPECIFICATION (IEEE 830)
Project: Almost Dead - VR Zombie Survival Game
================================================================================

1. INTRODUCTION
1.1 Purpose
This document outlines the vision and requirements for "Almost Dead," a VR zombie survival game that focuses on strategic decision-making, puzzle-solving, and escape rather than combat.

1.2 Scope
The game is set in a Pakistani university during a zombie virus outbreak. It challenges players to solve puzzles, manage resources, and navigate crises. It includes user authentication, a leaderboard, a 3D environment, non-violent survival mechanics, level progression, AI NPCs, cinematic cutscenes, and a mini-map.

1.3 Definitions, Acronyms, Abbreviations
- VR: Virtual Reality
- NPC: Non-Player Character
- FPS: Frames Per Second
- UI/UX: User Interface/User Experience

2. OVERALL DESCRIPTION
2.1 Product Perspective
"Almost Dead" provides a fresh, intellectually stimulating gameplay experience in a culturally relevant setting, contributing to the growth of the gaming industry in Pakistan.

2.2 Product Functions (Summary)
- Non-violent survival mechanics (escape, stealth, puzzle-solving).
- 3D university environment.
- Level progression from easy to hard.
- AI-driven NPCs.
- Interactive mini-map.
- Cinematic cutscenes.
- User authentication and leaderboard.
- VR integration with hand tracking and haptic feedback.

2.3 User Classes and Characteristics
- Players: End-users seeking an immersive, non-violent zombie survival experience.
- Developers: The internal team responsible for creating and maintaining the game.

2.4 Operating Environment
Oculus Quest and other supported VR platforms.

3. FUNCTIONAL REQUIREMENTS
3.1 User Log Management
FR1.1: The system shall create a back-end database with Firebase to store player logs.
FR1.2: The system shall analyze user logs for future updates and improvements.

3.2 Leaderboard
FR2.1: The system shall display player rankings and scores based on survival time and strategic choices.

3.3 UI/UX
FR3.1: The system shall provide a visually appealing and interactive UI using Figma designs.

3.4 Cinematic
FR4.1: The system shall create cinematic sequences to introduce the game's backstory.
FR4.2: The system shall integrate story-line for emotional engagement.

3.5 Environment
FR5.1: The system shall design game levels based on a university building setting.
FR5.2: The system shall create zombie-themed artifacts within the environment.

3.6 VR Integration
FR6.1: The system shall integrate hand tracking for player interaction.
FR6.2: The system shall provide haptic feedback for enhanced immersion.

3.7 Mini-Map
FR7.1: The system shall implement an interactive map showing the player's current location and nearby landmarks.

3.8 Gameplay Mechanics
FR8.1: The system shall design levels with progression through building floors.
FR8.2: The system shall include survival challenges and item collection.
FR8.3: The system shall increase difficulty with each level.

3.9 AI NPCs
FR9.1: The system shall develop AI-driven non-playable characters.
FR9.2: The system shall provide interaction mechanics to improve gameplay.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance
NFR1: The game shall maintain a minimum frame rate of 60 FPS on supported VR hardware to avoid motion sickness.
NFR2: Loading times between levels shall not exceed 5 seconds.
NFR3: User inputs (hand tracking, controllers) shall be processed with a maximum latency of 100ms.

4.2 Usability
NFR4: All in-game menus must be navigable within two steps from the main menu.

4.3 Security
NFR5: User data, including passwords, must be encrypted.
NFR6: Firebase Authentication with SSL/TLS must be used.

4.4 Reliability
NFR7: The system shall have a 99% uptime.
NFR8: The game shall automatically save progress every 5 minutes.

4.5 Scalability
NFR9: The architecture shall support the addition of new features, levels, or modes.

4.6 Compatibility
NFR10: The game must be compatible with Oculus Quest and other supported VR platforms.
NFR11: The game must support VR controllers with hand tracking and haptic feedback.

4.7 Accessibility
NFR12: The game shall include a user consent policy regarding age and motion sickness.
