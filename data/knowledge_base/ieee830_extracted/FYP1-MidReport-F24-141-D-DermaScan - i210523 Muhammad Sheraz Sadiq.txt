1.3.2 Module 2: Skin Disease Detection
This module focuses on skin disease detection using a CNN model and is trained using
various image datasets. This module includes the integration of the camera to capture user
images, processing them, and predicting outcomes using the CNN model. This module
also includes generation of diagnosis results based on user input.
Following steps will be followed to train the CNN model for skin disease detection.
1. Image capturing using mobile camera.
2. Skin-disease detection using CNN model.
3. Diagnosis results’ report generation.
4. Tracking progress of disease improvement.
5. Doctor recommendations based on disease and region.
* Please note that this module leverages a CNN model, trained on ISIC, HAM10000,
and Kaggle datasets, for disease detection.
1.3.3 Module 3: LLM powered Chatbot
The following module includes the integration of an LLM powered chatbot for improved
disease detection in addition to providing general information and guidance related to skin
diseases relevant to a user.
Following are the features included in this module.
1. Disease detection using symptoms logged in chat bot.
2. Disease-relevant guidance and information generation through Chat bot.
* Please note that this module leverages a LLM model for disease detection and
information generation.
1.3.4 Module 4: Doctor Profile
This module focuses on development of gateway for doctors to access and register on the
app. This module ensures communication between patients and relevant doctors.
1. Doctor profile management
2. Geo-location based recommendations.

Chapter 3
System Overview
DermaScan app is a mobile-based solution designed for the early detection and monitor-
ing of skin diseases efficiently using AI powered image analysis. The app allows patients
to upload images of affected areas of skin, which are then analyzed by the CNN model
(trained over a large dataset). It also include a LLM powered AI Chatbot for the guidence
and symptoms logging and Doctor recommendation system. The overall functionality of
the system is to offer real time skin disease detection and a way to consult relevant doctors
based on the user region.
3.1 Architectural Design
The DermaScan Skin-Disease and Guidance app is divided into a modular structure to
enhance feature integration, maintenance and scalability. The system has 4 main mod-
ules including Patient profile, Skin disease detection, LLM powered chatbot and Doctor
Profile. Each of the 4 module cater to a different section and functionality of the app
and thus promote individuality and independence of app features. However, the modules
collaborate with each other to provide the overall functionality of the app.
3.1.1 Modular Relationship
Below mentioned is a brief description of the relationship and collaboration of each mod-
ule with the rest of the system.
1. Patient Profile This module covers the registration and user profile management
section of the app and collaborates with the rest of the system by feeding user data
including medical records. It provides the user data required for CNN processing in
the second module, provides user information for chatbot functionality in the third
25
3. System Overview
module and provides user location for doctor recommendations functionality in the
fourth module.
2. Skin Disease Detection This module contains the core functionality of the app and
collaborates with the first module by fetching user data in addition to users’ skin
scans for skin-disease detection. It collaborates with the third model by providing
the CNN scan results to the chatbot for further processing. This module fetches
doctor profiles from the fourth module and recommend it to users based on the
"geo-location" obtained from the first module.
3. LLM powered Chatbot This module features the LLM powered chatbot and uses
logged systems from the first module and scan results from second module to im-
prove the efficiency of the skin-disease detection system. This module has no direct
relationship with the doctors’ module.
4. Doctor Profile This module features doctor profile management and related fea-
tures and collaborates only with the user profile module for patient-doctor commu-
nications and recommendations.

3.1 Architectural Design
3.1.2 Architectural Pattern
The "client-server" architecture is used in the development of DermaScan app which is
modularized as follows:
• Client (Mobile App): Includes user profile modules which allows user to access
the app and post skin scans for disease detection.
• Server: The server side is responsible for the functioning of CNN and LLM models
in addition to results and reports generation based on user input.
A Layered (Multi-tiered) Architecture will be adopted.
1. Presentation Layer - User Interface.
2. Application Layer - Handles app logic, chatbot interaction, and model training.
3. Data Layer- Manage patient’s Data, Doctors Information, manage medical records.

3.3 Data Design
DermaScan, being an image-processing application consists of data-intensive features for
the accurate detection and diagnosis of skin-diseases including the data storage of datasets
for training the CNN and LLM models. The information domain of the app contains
multiple entities including patients, doctors and tables for data storage of chabot conver-
sations, maintianing medical records and image storage for skin-disease detection. TO
achieve efficiency in data storage, processing and retrieval, our app uses the Firebase as
its primary database to store all data relevant to the application. A detailed explanation
and a visual representation of DermaScan’s data transformation into data structures is
