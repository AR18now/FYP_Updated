1.2 Scope
The scope of this project is that we are developing a Virtual Reality (VR) application that
changes the way current educational videos are perceived , by creating an immersive and
interactive educational environment.
The main functionalities of our project includes :
Creating an Immersive VR Classroom Environment - It will make users feel like they
are sitting in a real life classroom.
Realistic Teacher Motions - The virtual teacher will make actions just like a real teacher
while teaching.
Realistic Teacher Emotions - The virtual teacher will make emotions just like a real
teacher while teaching.
Realtime Question and Answers - The user will be able to stop the virtual teacher in the
middle of the lecture and ask questions and will receive relevant feedback from the virtual
teacher.
Lifelike content Delivery - The virtual teacher that will be delivering the lecture would
feature a realistic voice to make the lecture feel more natural.
Dynamic slide management - The slides will automatically change according to what
the teacher is currently saying to maintain context.
Note Taking Capability - In order to enhance the feel of a realistic classroom and to
facilitate users we also have a virtual notepad in which users can take notes.
Interactive Quizzes - In order to enhance lecture retention and user engagement , we will
also feature AI generated quizzes during our lectures.
Feedback System - We will take user feedback after each lecture , to keep on improving
our application in the future.
User Authentication - For keeping the data of our users secure.
Multiplayer - Multiple students can attend the same lecture together in a virtual environ-
ment and can interact with each other
Live Teaching Sessions - A real life teacher can come in the place of our virtual teacher
and teach the students in person.
For the current scope of this project we are only covering videos that are related to com-
puter science and specifically those videos that are related to python (Beginner to Interme-
diate level), Furthermore currently we are only catering those videos in which the teacher
is teaching using slides or is doing live coding. In the future we can also cater various
different types of subjects and videos into our application aswell.

1.4 User Classes and Characteristics

Student The students or learners are the primary users of this application. They can attend
lectures in a virtual classroom environment whilst interacting with many things

3.1 Architectural Design
The Architecture that our project LEARNSCAPE mainly follows is a client-server model.
In which the unity client will request services from our central server. The user will
interact with the interface and the different visuals through the Scene Editor Module and
the Interaction Module which are present in the unity client.
The server will include multiple key components :
• Python Backend for processing
• LLM model for AI driven tasks
• PhotonPun for multiplayer functionality
The server side will also communicate with Firebase , which will act as the main database
for our application, and also relies on Firebase authentication for secure user access.
The Front-end environment will basically be handles by the unity client which includes
the Scene Editor and the Interaction modules. Meanwhile, all of the major processing
including authentication , AI driven features , multiplayer will take place on the server
side.
