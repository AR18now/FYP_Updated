1.2 Scope
The purpose of our project is to develop a software tool that will simplify and speed up
the process of both data extraction and analysis from financial reports in PDF format.
It will automatically retrieve data from both regular and scanned PDFs, reducing manual
work and saving time. The tool will be user-friendly, making it easy for financial analysts,
investors, and others to access the data they need. FinXtract will include features like sen-
timent analysis to assess company reports, helping users understand business performance
and trends. It will also have an intelligent chatbot for real-time help with queries and data
retrieval, ensuring accuracy and security. The system will be able to securely store all
extracted data and ensure it is standardized and cleaned for accurate analysis. The project
will focus on accuracy and consistency to ensure reliable results. Overall, FinXtract is
designed to make financial data extraction and analysis easier, more accurate, and secure.

System Overview
FinXtract is a software tool that automates extracting and analyzing financial data from
PDF reports, saving time and improving accuracy. It helps analysts, investors, auditors,
and executives to quickly access clean and standardized financial information.
FinXtract does the job of extracting and standardizing financial data from PDFs, thus en-
suring that all of its users have access to clean and ready-to-use information. It uses OCR
for extracting text from scanned documents, NLP for performing Sentiment Analysis,
and complex algorithms for standardizing different terms like "sales", and "revenue" used
by the companies. FinXtract also includes a smart chatbot that can be of great use for
providing real-time assistance to users, answering their queries, and fetching important
information from reports.
Companies, analysts, and auditors often spend a significant amount of their time going
through numerous financial documents manually to retrieve information, which can lead
to errors. The fact that companies have used different terms over time only further com-
plicates data comparison. The use of scanned documents also doesn’t simplify the process
of information retrieval.
FinXtract overcomes all these problems by automating data extraction, ensuring consis-
tency and accuracy of the financial data. Its easy-to-use interface allows professionals to
focus more on analysis and decision-making rather than worrying about the accuracy of
the extracted data.
It is a cloud application for pulling data from PDFs. Its key modules are:
• PDF Processing: Extracts text from regular and scanned PDFs using OCR.
• Data Standardization: Maintains consistency in financial terms.
• Sentiment Analysis: Analyzes the performance of the companies over time.
• Chatbot: Provides real-time answers to user queries.
• User Interface : A web-based interface for easy access to the reported and stan-
dardized data and a chatbot for better user experience.

3.1 Architectural Design
FinXtract’s system architecture is a modular architecture that helps to make the system
very flexible, scalable and easy to maintain. Most of the modules have some sort of
functionality or design associated with specific tasks that help to reach the main goal
of extraction and analysis of financial data. All these modules are automatically linked
together to cover the entire workflow from scraping financial PDFs, to standardizing data
and showing them in a user-friendly interface.
3.1.1 High-Level Design Overview
FinXtract uses a Layered Architecture model, which divides the system into separate,
independent layers for better organization:
1. Presentation Layer (Front-end) : It consists of a User Interface (UI) that shows
the data scraped through PDFs in the form of reported and standardized data and a
chatbot. This user-friendly interface allows the users to use the system conveniently
according to their needs.
2. Business Logic Layer:
• This layer consists of primary modules that manage data extraction, standard-
ization, and analysis.
• PDF Processing Module: Converts raw data from PDF reports into structured
formats.
• Data Standardization Module : Ensures that the data of multiple companies
is consistent and ready for analysis.
• Chatbot Module : Responds to user questions and provides real-time assis-
tance using natural language processing.
3. Data Layer : This layer securely stores all the processed and standardized data,
making it easily accessible for analysis and chatbot responses.

3.2.5 ERD
Figure 3.10: ERD
The ERD Diagram of FinXtract includes entities such as a User who will be interacting
with the website after entering his/her credentials. There are various companies who have
financial PDF documents from whom the financial data is extracted. The User views
the extracted data (reported or standardized) according to their needs as well as interacts
with the chatbot. The chatbot processes financial data when generating an answer and
provides sentiment analysis which is a weak entity in our architecture since sentiment
analysis cannot exist without a chatbot. The sentiment analysis is performed on financial
data which is also a weak relationship due to the dependency of sentiment analysis on the
chatbot.
The UserID is primary for each user since each user is unique. Same is the case regarding
CompanyID for a company and DocumentID for a PDF document. A single company
releases many pdf documents (1-Many). Financial data is extracted from many Pdfs (1-
Many) and the User can view many types of extracted financial data (1-Many). At a
given point of time, when generating an answer, a chatbot only processes one piece of
financial data (1-1). The User can ask multiple queries from a chatbot (1-Many) which
provides many sentiment analysis (1-Many). The sentiment analysis is a weak entity
due to its dependency on the chatbot and therefore the relationship between the chatbot
and sentiment analysis is an identifying relationship. Sentiment Analysis is performed
on a single piece of extracted financial data at a time (1-1) which is also an identifying
relationship.

3.3 Data Design
3.3 Data Design
The information domain of FinXtract is centered on financial data extracted from PDF
reports and its transformation into data structures involves the following steps:
Data Storage:
MongoDB is used to store the financial data extracted from PDF reports in JSON format
including the standardized and textual data.
FAISS vector database is used to store embeddings of data extracted from PDFs. These
embeddings are representations of the extracted data in vector form used for similarity
queries while interacting with the chatbot.
Backend Processing:
Node.js handles all the core backend processes which involve extracting data from PDFs,
managing the API for user interactions and communicating with the MongoDB and the
FAISS vector database for data retrieval.
Node.js uses the Ollama server to handle interaction with the chatbot, performing simi-
larity searches across stored embeddings to retrieve the most relevant financial data for
efficient query retrieval and utilizing natural language processing techniques such as sen-
timent analysis.
