Project: Scholarly - AI-Driven Smart Learning App

1. INTRODUCTION
1.1 Problem Statement
Research shows average human attention span is 15-20 minutes while watching video, but decreases to only 5-8 minutes when reading textual content. Every 3 out of 4 students experience difficulty retaining focus while studying and need more concise, engaging learning materials.

1.2 Scope
Scholarly allows users to upload books and automatically generate personalized learning roadmaps with short video reels for each topic. Features include book upload, roadmap generation, textual summaries, short video reels (reels), progress tracking, and personalized dashboard.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- MTBF: Mean Time Between Failures
- PDF: Portable Document Format

2. OVERALL DESCRIPTION
2.1 Product Perspective
Scholarly revolutionizes how students comprehend and interact with learning content by breaking down complex topics into smaller, manageable sections with visual representations.

2.2 Product Functions (Summary)
- User authentication (register, login, profile management)
- Book upload (PDF format)
- Automatic roadmap generation
- Topic identification from book content
- Textual summary generation
- Short video reel generation
- User feedback system
- Progress tracking

2.3 User Classes and Characteristics
- Students: Primary users who upload learning materials, generate study roadmaps, engage with reels and summaries

2.4 Operating Environment
Mobile application (Three-tier architecture: Presentation, Application Logic, Data), Firebase for storage.

3. FUNCTIONAL REQUIREMENTS
3.1 User Management
FR1.1: Students shall create new accounts by providing email, username, password, personal details.
FR1.2: The system shall allow students to login and authenticate as users.
FR1.3: The system shall offer interface for users to view profile.
FR1.4: Users shall update personal details (username, date of birth, password).
FR1.5: The system shall provide user dashboard.

3.2 Roadmap Generation
FR2.1: The system shall allow users to upload a book in PDF format.
FR2.2: Users shall see loading screen while waiting for book upload completion.
FR2.3: The system shall analyze uploaded content and clean it.
FR2.4: The system shall identify main topics in book for roadmap generation.
FR2.5: The system shall visualize the roadmap.
FR2.6: Users shall view roadmap generated from book.
FR2.7: Users shall select a topic from roadmap.
FR2.8: Users shall view progress of generated roadmap.

3.3 Reels Generation
FR3.1: The system shall summarize text in topic selected by user.
FR3.2: The system shall present option to select textual summary or reels.
FR3.3: Users shall select either textual summary or reels.
FR3.4: The system shall generate short video reels from selected topic.
FR3.5: Users shall see generated reels based on selected topic.
FR3.6: Users shall provide feedback on viewed reels.
FR3.7: The system shall store feedback.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: MTBF ≥ 500 hours during normal usage.
NFR2: Subsystem failure shall not cause entire system failure; fallback mechanisms to notify users.
NFR3: System shall log necessary errors for debugging.

4.2 Usability
NFR4: First-time users shall easily navigate key features with minimal assistance.
NFR5: Book upload ≤ 2 interactions (selecting file, confirming upload).
NFR6: Progress bars for all lengthy processes (uploading, analyzing).
NFR7: Interactive roadmap allowing navigation through topics and subtopics.
NFR8: Error messages with clear description for user recovery.

4.3 Performance
NFR9: Book upload ≤ 30 seconds for size ≤ 50MB.
NFR10: Personalized roadmap generation ≤ 15 seconds after successful upload.
NFR11: User dashboard load ≤ 3 seconds after login.
NFR12: Topics and subtopics display ≤ 3 seconds after clicking roadmap.
NFR13: System shall handle ≥ 100 concurrent users.
NFR14: Average response time for standard operations ≤ 3 seconds.

4.4 Security
NFR15: Secure authorized login for all registered users.
NFR16: Data integrity preventing unauthorized access.
NFR17: Minimum two levels of authentication for accessing sensitive user data.
