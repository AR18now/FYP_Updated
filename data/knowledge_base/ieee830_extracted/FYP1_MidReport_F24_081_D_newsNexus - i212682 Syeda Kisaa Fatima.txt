Chapter 3
System Overview
"newsNexus" is a smart news aggregation system that’s designed to give users a highly person-
alized and engaging experience when it comes to staying informed. Think of it as your personal
news assistant—it brings together articles, real-time video insights, and predictive forecasts, all
customized to your preferences.
3.0.1 What It Does
The main idea behind "newsNexus" is to helpyou cut through the noise. There’s a lot of news
out there, and it can be overwhelming to find what’s relevant to you. "newsNexus" simplifies
this by continuously collecting articles from a variety of reputable sources, processing them to
understand the meaning, and then letting you find exactly what you’re interested in through an
easy-to-use interface.
You can search using text or even voice commands, and the system will present results based
on what it knows you care about. Whether you’re interested in traditional articles, want to see
what people are saying on YouTube, or are curious about upcoming trends and predictions,
"newsNexus" has you covered.
3.0.2 Why It Matters
We live in an age of constant information bombardment—often too much, and not always
trustworthy. With "newsNexus", you’re in control. You decide the sources you trust and the
topics you care about, and the system tailors what you see accordingly. It’s designed to make
sure you’re getting the most accurate and up-to-date information in the way that works best for
you.
The system is also smart enough to bring in different forms of media, including articles and
videos, so you can get a well-rounded perspective on current events. Plus, with AI-driven
22
predictive insights, you can get a glimpse into potential future trends for topics that interest
you.
3.0.3 How It’s Built
Below are the steps to easily understand how it will be built:
1. Collecting Data: First, the system goes out and scrapes news from different sites(samaa,
ary, geo, 24news, dunya news). This happens regularly to make sure everything is fresh.
Once the raw data is collected, preprocessing is done to filter out irrelevant information.
The preprocessing includes cleaning the text, extracting useful information, and structur-
ing the data to make it uniform. For further processing only cleaned data is used.
• Processing Data: Once the data is collected, it’s turned into something the system can
understand better—vector embeddings. These are smart representations of the content,
making it easier to match what’s being asked to what’s available. The data is stored in
two places—a Vector Database for efficient content retrieval and a SQL Database for the
structured details.
• Processing Query: When you interact with newsNexus, this is where the magic happens.
The system figures out what type of query you’re making—basic text searches, questions
involving YouTube videos, or future predictions. Depending on what you’re asking,
it either looks up content in the vector database, processes video clips, or uses AI to
generate predictions.
• Front-End: The user interface is designed to be clean and intuitive. It lets you log in,
set your preferences, search for content, and save your interactions for later. Everything
comes together here to give you the news experience you want.
• Cloud Integration : Behind the scenes, everything will be hosted in the cloud, which
helps the system to handle large amounts of data and serve a lot of users at once. It keeps
the system scalable and reliable.
• Generation Response: After the system processes your query, this layer is responsi-
ble for putting together all the results—from articles, videos, and AI insights—into one
comprehensive response, which then gets displayed to you.

3.0.4 Background Inspiration
newsNexus is being developed to solve a real problem. People are overwhelmed by the sheer
volume of news and information out there, and it’s getting harder to separate valuable, trusted
content from everything else. The goal is to build a tool that feels like it’s working for you,

We also wanted to make it future-oriented. By including AI-based predictions, we help users

Overall, newsNexus is about giving people a personalized, in-depth news experience that’s easy

3.1 Architectural Design
The architecture is designed to be modular and efficient, ensuring that user queries are handled
swiftly and the right information is served back to the user in a meaningful way. Each layer
plays a distinct role, and together they form a robust pipeline from data collection to content
delivery.
1. Front-End Layer The front-end layer serves as the interface between the user and the
system, providing a user-friendly and intuitive experience. It enable users to interact with the
system by submitting queries, viewing results, and accessing their preferences. The focus here
is on creating a seamless experience for users to quickly and easily find the news content or
other information they need. This layer is responsible for gathering input from the user (like
text or audio queries) and displaying organized results from various sources.
2. Query Processing Layer This is the heart of the system where the magic happens. Once
a query is submitted by a user, this layer is responsible for classifying the type of query. The
query can be categorized as a Basic Query, a YouTube Query, or a Future Prediction Query.
• Basic Query: For text-based searches or direct information retrieval.
• YouTube Query: If the user wants to find specific content from YouTube, this module
processes videos to extract relevant segments.
• Future Prediction Query: When a user asks about a trend or forecast, this module uses
an AI/ML Model to generate predictive insights.

Depending on the type of query, this layer interacts with various modules like the AI model or

them to the front-end for display.
3. Data Processing Layer The data processing layer ensures that all the data is in the right
form for efficient retrieval and analysis. It has two major components:
• Vector Database: Stores vector embeddings that are derived from text data. This helps
in quickly matching and retrieving relevant content based on similarity.
• SQL Database: Contains structured data like the cleaned, preprocessed text, saved re-
sponses and metadata.
This layer is responsible for generating these vector embeddings, which transform text into
a format suitable for quick retrieval and comparison. Once the data is processed, it is stored
in a format that allows fast access and effective searching, making the system efficient and
responsive.
4. Data Collection Layer The data collection layer is where raw data comes into the system.
It handles the process of web scraping, where news and information from various sources are
collected periodically. After the data is scraped, it undergoes preprocessing to clean, standard-
ize, and extract relevant information before it is handed over to the data processing layer. This
ensures that only the most accurate and useful data moves forward in the pipeline. The entire
process is automated to keep the system updated with fresh content regularly.
5. Cloud Integration Layer This layer supports the entire architecture by utilizing cloud
services that gives scalability, ensuring that as more data is collected or more users access the
system, the system can handle the increased load smoothly.
6. Response Generation Layer This layer acts as a bridge between the query processing and
front-end layers. After a query is classified, processed, and relevant content is retrieved, this
layer aggregates all the pieces of information (e.g., text-based articles, videos, predictions) into
a coherent response. It ensures that the final output is organized, easy to understand, and ready
for presentation on the front-end for the user.
3.1.1 High-Level Overview of Architectural Design
The newsNexus system efficiently fetches, processes, and displays news content in a user-
friendly way. Here’s how all modules work together to achieve this:
1-Data Collection to Storage:
The process starts with the Data Collection Layer, where the Web Scraping Module gathers
news from various sources. The data is then preprocessed, cleaned, structured, and transformed
into vector embeddings for efficient retrieval. The processed data is stored in:

• Vector Database: For content retrieval based on semantic meaning.
• SQL Database: For structured data and metadata.
2-User Interaction and Query Handling:
Users interact with the system via the Front-End Layer to search, update preferences, or change
sources. The Query Processing Layer classifies the user’s query as:
• Basic Query: Direct content retrieval.
• YouTube Query:Searches for and processes relevant video clips.
• Future Prediction Query: Uses AI/ML to predict outcomes based on data.
3-Fetching and Generating Responses:
The Query Processing Layer interacts with the Data Processing Layer to fetch articles, process
YouTube videos, or generate predictions. The Response Generation Layer then aggregates all
the results into a comprehensive response.
4-Personalization and Cloud Integration:
User preferences are considered throughout the query processing for personalized content. The
architecture is hosted in the Cloud Integration Layer, ensuring scalability, storage, and seamless
communication.
5-Displaying Results to User:
The Response Generation Layer sends the aggregated content back to the Front-End Layer for
display. This enables users to interact with content types and receive highly relevant, personal-
ized results in real time.
