SOFTWARE REQUIREMENTS SPECIFICATION (IEEE 830)
Project: Mumtahin - AI-Powered Automated Assignment Demo Evaluation System
================================================================================

1. INTRODUCTION
1.1 Purpose
This document outlines the project description, scope, and requirements for Mumtahin, a tool to revolutionize the current system of assignment demos by integrating automation and AI to reduce bias and manual labor.

1.2 Scope
The system automates the entire demo evaluation process, including checking submissions, marking assignments, asking AI-generated questions, taking quizzes, evaluating demos, and generating reports.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- NLP: Natural Language Processing
- API: Application Programming Interface

2. OVERALL DESCRIPTION
2.1 Product Perspective
Mumtahin aims to alleviate the burden on students and evaluators by automating time slot allocation, verification, evaluation, and reporting, ensuring a fair and efficient process.

2.2 Product Functions (Summary)
- Facial recognition for identity verification.
- Plagiarism detection for assignments.
- Slot management.
- Submission checking.
- AI-generated questions and answer verification.
- Facial expression evaluation.
- Report generation.

2.3 User Classes and Characteristics
- Students: End-users who give demos.
- Teacher Assistants: Evaluators who assess demos.
- Faculty: Oversees the process.

2.4 Operating Environment
Web application.

3. FUNCTIONAL REQUIREMENTS
3.1 Video Demos
FR1.1: The student will join the meeting at the allocated time and complete the quiz.

3.2 Slot Booking
FR2.1: The student selects an available time slot for the demo and confirms the booking.

3.3 Download Submission
FR3.1: The student can download their submitted assignment for review or personal records.

3.4 Facial Recognition
FR4.1: The system performs facial recognition to verify the identity of the student during the assignment demo.

3.5 Generate Report
FR5.1: The system generates a report summarizing the student's assignment demo performance.

3.6 Plagiarism Check
FR6.1: The system performs a plagiarism check on the submitted assignment by comparing its content against a database of existing documents and online sources.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance
NFR1: The user will be able to login within 5 seconds after providing the right credential.
NFR2: The user should be able to enter the meeting within 10 seconds of the meeting.

4.2 Usability
NFR3: The system should be easy to use, as its interface should be intuitive.

4.3 Reliability
NFR4: The system should regularly backup the data and allow quick recovery in case of loss.

4.4 Security
NFR5: The personal information of students will be secure in the database.
