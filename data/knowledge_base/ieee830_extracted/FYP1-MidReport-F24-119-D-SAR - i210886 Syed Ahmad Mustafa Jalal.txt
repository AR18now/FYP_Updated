1. Introduction
1.2 Scope
The proposed project aims at developing a cross-platform social networking platform.
Our project’s scope includes the development of both a desktop application as well as a
mobile application to cater to cross-platform interactions. Our team will start working on
the design of our app’s user interface and gathering datasets on which the recommender
model will be trained. In subsequent iterations the team will focus on developing the user
profile module first followed by other modules. It is to be noted that the desktop app will
consist of the user profile module only. The player profile module will consist of user
profiles where user can list the games they play and our app will be responsible for track-
ing the screen time of each game listed on the user’s profile. It would also enable user
to share gameplay and content. This module will be followed by developing the recom-
mender model in addition to implementing the user feed and connection system module
simultaneously. The recommender model will be developed from scratch using gaming
datasets and would be trained to function based on shared gaming niche and interests in
addition to incorporating Geo-location of users. The connection system is a network-
ing module that would empower the app’s users to connect with each other based on
shared interests. Lastly the modules related to esports organizational management would
be developed and the recommender model designed earlier would be incorporated in the
features where necessary. This module will provide features including tournament/event
organization and listing, team and staff recruitment and sponsored posts. A detailed ex-
planation of each module is listed below.

1.3 Modules
1.3.1 Module 1: User Profile (Desktop App)
The desktop application will consist of the user profile module only. This module mainly
focuses on screen-time tracking for each game the user plays and creating various stats
on the basis of that data in addition to simple profile management features. It will also
enable user to post gameplay and achievements on their profile. The main features of this
module include:
1. Personal profile management
2. Playtime analytics dashboard including total playtime across all games, total play-
time per game, average playtime, peak playtime, favorite game, favorite genre
3. Post gameplay and achievements
4. Achievement/ranking system based on total playtime across all games played.
1.3.2 Module 2: User Profile (Mobile App)
This module of the mobile application is similar to that of the desktop application and
provides the same features as mentioned below.
1. Personal profile management
2. Playtime analytics dashboard including total playtime across all games, total play-
time per game, average playtime, peak playtime, favorite game, favorite genre
3. Post gameplay and achievements
4. Achievement/ranking system based on total playtime across all games played.
1.3.3 Module 3: Networking System (Mobile App)
This module focuses on the development of features related to user engagement and in-
teraction with peers across the app and leverages a recommender system.
1. Interest-based user feed
2. Synergy (Connection system)
3. Threads (Chats)
* Please note that feature 1 and 2 will leverage a recommender system that will
cater to user’s gaming interests and Geo-location

2.2 Functional Requirements
Below listed are the functional requirements of each module of our app.
2.2.1 Module 1: User Profile (Desktop App)
Following are the requirements for module 1:
1. The system shall allow users to register on the app, create and edit their profiles.
2. The system shall allow users to upload content including text, photos and videos on
their profile.
3. The system shall allow users to list games on their dashboards.
4. The system shall fetch screen time and generate analytics for the games played by
a user.
5. The system shall record and maintain user’s game-time analytics.
6. The system shall track and display total hours spent across all games.
7. The system shall rank users based on their total playtime across all games.
2.2.2 Module 2: User Profile (Mobile App)
Following are the requirements for module 2:
1. The system shall allow users to register on the app, create and edit their profiles.
2. The system shall allow users to upload content including text, photos and videos on
their profile.
3. The system shall allow users to list games on their dashboards.
4. The system shall fetch screen time and generate analytics for the games played by

System Overview
ArenaX is a cross-platform networking app that provides a socio-professional environ-
ment for the gaming industry. It comprises of a personalized user feed system, interest
based connections, threads, game-time tracking and analytics and tools for esports man-
agement. The product aims to converge the gaming community on a single platform by
providing all necessary features and functionalities that attract gamers across genres.
3.1 Architectural Design
The ArenaX project is modularized based on distinct features to create a multi-layered
architecture to ensure maintainability and scalability. The system is divided into five pri-
mary modules, each comprising of various features as sub-modules. These sub-modules
are interconnected within their respective modules, creating a cohesive system. Each main
feature within a module functions as an independent entity, ensuring the three-layered ar-
chitecture of ArenaX, which consists of primary modules, sub-modules, and individual
features within each sub-module. Below is an overview of the relationships between the
primary modules of the system.

related to all the other modules that leverage the recommender system in their func-
