Project: SmartPark - IoT Based Smart Parking Management System
================================================================================

1. INTRODUCTION
1.1 Purpose
This document provides an overview of the IoT Based Smart Parking Management System, which aims to enhance user experience by providing a seamless car parking solution using IoT technology.

1.2 Scope
The project involves installing IoT sensors to monitor parking spaces and developing a mobile application where users can view available slots, make reservations, and handle payments. It includes database integration with Firebase and a pilot deployment.

1.3 Definitions, Acronyms, Abbreviations
- IoT: Internet of Things
- MTBF: Mean Time Between Failures
- OBD: On-Board Diagnostics (mentioned in other reports)

2. OVERALL DESCRIPTION
2.1 Product Perspective
The system maximizes the use of existing parking spaces by introducing a mobile app that monitors availability, allows reservations, and processes payments, eliminating the hassle of traditional methods.

2.2 Product Functions (Summary)
- Sensor data handling (reading, transmission).
- Real-time slot monitoring via mobile app.
- Slot reservation and billing.
- User and admin dashboards.

2.3 User Classes and Characteristics
- Car Owners: Primary users who check availability, make reservations, and pay.
- Parking Lot Administrators: Manage parking spaces, monitor occupancy, process payments.
- Payment Handlers: Banks/payment gateways that facilitate digital transactions.
- Government Authorities: Oversee implementation and ensure compliance.

2.4 Operating Environment
Mobile application with cloud-based backend (Firebase).

3. FUNCTIONAL REQUIREMENTS
3.1 Sensor Data Handling
FR1.1: The system shall read data from ultrasonic sensors to determine if a parking slot is occupied.
FR1.2: The system shall transmit sensor data to Firebase for storage and real-time updates.

3.2 User Interface and Real-time Monitoring
FR2.1: The mobile app shall display real-time availability of parking slots based on sensor data.
FR2.2: The mobile app shall allow users to reserve a parking spot.
FR2.3: The system shall notify users when a reserved parking spot is about to expire.

3.3 Reservation and Billing
FR3.1: The system shall allow users to reserve parking spots in advance.
FR3.2: The system shall provide seamless payment options for parking sessions.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: The system shall achieve 99.5% uptime.
NFR2: The system will have backup sensor systems in case of failure.

4.2 Usability
NFR3: The app must be intuitive, requiring no more than 5 minutes for a new user to understand.
NFR4: A user shall reserve a parking spot in 3 clicks or less.
NFR5: The system will generate clear error messages and suggest alternatives.

4.3 Performance
NFR6: The system shall update parking slot availability within 5 seconds of a car entering/leaving.
NFR7: The mobile app shall load parking availability within 3 seconds.
NFR8: The database shall handle up to 1,000 parking spots in the initial phase.

4.4 Security
NFR9: The system shall enforce 2-factor authentication for users.
NFR10: Access to the database shall be restricted to authorized personnel only, with logs maintained.
