1. Introduction
1.2 Scope
The 3D AutoMod project concerns the creation of an entire web application involving a
3D modeling toolkit and a marketplace of automotive enhancements. It comes with an
interactive 3D viewer that will allow users to upload images of their vehicles and get 3D
models in return. Users shall be able to make changes of these models in real time –
changing rims, mirrors, paint jobs or any other aspect. There will be a bidding system
where mechanics will be able to bid on modification jobs depending on what the user
want. Explaining the features of the application, there will be an option of live chat to en-
hance the communication between users and mechanics. A reliable payment gateway will
be incorporated for purposes of accomplishing the transactions. Modification history and
preferences for the user account will be kept in order to improve the quality for the end
user. It will also involve notification on new offers and mechanics responses that should
be made to the users of the application. The primary aim of the first few sprints is to de-
liver a basic, but reliable web application, though the groundwork will be set to make app
development possible. It also examines the user management, 3D modeling and visual-
ization, mechanic bidding, communication and transaction management functions which
will form the basis for creating an integrated solution for the automotive modification
market.

3.1 Architectural Design
3.1.1 Overview
The structure of the Car Customization Platform is developed using the architectural pat-
tern called Model-View-Controller (MVC). MVC is chosen because it provides clean
separation of concerns, improves modularity and promotes easy maintainability. This ap-
proach makes it possible to grow, test, and deploy each component of the system without
the other components affecting the result.
3.1.2 Rationale for Choosing MVC
• Separation of Concerns: MVC distinctly separates data management (Model),
user interface (View), and control logic (Controller), enabling organized and man-
ageable code.
• Modularity: Each component can be developed and maintained independently,
promoting reusability and ease of updates.
• Scalability: The architecture supports scaling individual components based on de-
mand without affecting the entire system.
• Enhanced Collaboration: Facilitates parallel development, allowing frontend and
backend teams to work simultaneously without conflicts.

profile updates.

• ExpertController: Manages interactions with car experts, including bid submis-
