2.1.1 Performance Comparison of Binary Machine Learning Clas-
sifiers in Identifying Code Comment Types: An Exploratory
Study [55]
2.1.1.1 Summary of the Research Item
In this exploratory study, Indika et al. [ 55] examine the performance of various binary
machine learning classifiers in identifying different types of code comments across Java,
Python, and Pharo. Utilizing a dataset of 6,738 code comments, they developed 19 binary
classifiers, each focused on specific comment categories. The study involved prepro-
cessing the data through text normalization and oversampling to optimize classification
results. Classifiers such as Decision Trees, Logistic Regression, Random Forest, and Sup-
port Vector Classifier were tested. The results indicate that the Linear SVC achieved the
highest average F1 score of 0.5474, although the performance of the classifiers varied
depending on the language and comment type.
2.1.1.2 Critical Analysis of the Research Item
Strengths of the Research Item:
• The study evaluates a broad range of machine learning classifiers, providing a com-
prehensive comparison of their performance.
• Utilization of the TF-IDF approach for converting text data into numerical features
enhances the model’s ability to process and classify comments effectively.
• The Linear SVC classifier achieved the highest average F1 score, indicating its
strong performance in the context of code comment classification.
Weaknesses of the Research Item:
• Attention based deep learning models could have improved upon the performance
of the tool leading to better precision and recall.
• Alternative preprocessing and oversampling techniques, features and hyperparam-
eter tuning could lead to further improvement in the scores achieved.
• While the Linear SVC performed well overall, no single classifier excelled across
all comment types.

Proposed Approach
In this approach, we enhance comment classification using transformer models and opti-
mized preprocessing to improve performance across various programming languages.

3. Proposed Approach
3.1 Dataset Acquisition
We will acquire the dataset from theNLBSE Tool Competition Website, which provides
a large corpus of code comments across multiple programming languages such as Java,
Python, and Pharo. This dataset will be used for training, validation, and testing of our
models for comment classification tasks.
3.2 Preprocessing
The preprocessing steps involve cleaning the raw dataset to ensure consistency and better
model performance. Key steps include:
• Removing Null Spaces : Identifying and removing records with missing data to
maintain data integrity.
• Removing Special Characters: Filtering out unnecessary symbols and characters
that do not contribute to semantic understanding.
• Converting to Lowercase: Standardizing the text by converting all characters to
lowercase, making the data uniform.
• Data Splitting: Splitting the cleaned dataset into training and testing sets, ensur-
ing a representative sample is used for both.
3.3 Feature Extraction
After the prepossessing is done, we will extract features by converting code comments
into vectorized representations. We will be leveraging techniques such as TFIDF for this
task.
3.4 Oversampling for Low-Resource Languages
To address the imbalance in datasets, especially for low-resource languages like Pharo,
we will apply the EDISON oversampling technique. This approach will help balance the
dataset, ensuring that underrepresented classes receive sufficient attention during training,
thus improving any selected model’s ability to generalize across multiple languages.
24
3.5 Model Selection and Evaluation
3.5 Model Selection and Evaluation
We will experiment with multiple transformer-based models such as:
• CodeBERTa: Refined pre-training focused on code-comment interactions.
• GraphCodeBERT: Incorporates code structure through graph representations.
• PLBART: Excels at code generation and summarization tasks.
Each model will be evaluated based on its performance on the validation dataset.
3.6 Training the Selected Model
Once the best model has been selected and features have been extracted, we will pro-
ceed with the model training. The training process will focus on optimizing the model’s
performance by tuning key parameters such as batch size,learning rate and number of
epochs.
To prevent overfitting, we will monitor performance using early stopping techniques, en-
suring the model generalizes well across unseen data. Hyperparameter optimization
will be applied to fine-tune the model for better accuracy across multiple programming
languages. Additionally, multi-level layer aggregation will be employed to improve the
model’s ability to capture complex patterns in the comment text.
The training will utilize the augmented dataset, ensuring that both high-resource and low-
resource language comments are well-represented.
3.7 Domain Post-Training
We will conduct domain post-training for the selected model, refining it to specialize
in code comment classification rather than general code and natural language data. This
step will help the model better understand domain-specific vocabulary, especially in the
context of programming languages and comments, further improving its performance on
classification tasks.
3.8 Evaluation Metrics
The following metrics will be used to evaluate model performance:

• F1-score: The harmonic mean of precision and recall, particularly important for
