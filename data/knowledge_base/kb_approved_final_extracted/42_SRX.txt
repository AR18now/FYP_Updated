1. INTRODUCTION
1.1 Problem Statement
Unexpected transportation dangers from landslides, floods, and earthquakes commonly affect travelers. Traditional guidance systems cannot promptly notify users of these hazards, leading to dangerous circumstances, traffic jams, and accidents. Typical GPS systems do not account for unexpected changes in road safety.

1.2 Scope
SRX is a Real-Time Road Safety Alert System using NLP to filter social media posts and news for relevant dangers, updating safety indexes of different routes in real-time. Features include route safety index calculation, real-time alerts, map-based interface, offline capability, and system administration for manual map management.

1.3 Definitions, Acronyms, Abbreviations
- NLP: Natural Language Processing
- GPS: Global Positioning System
- API: Application Programming Interface

2. OVERALL DESCRIPTION
2.1 Product Perspective
SRX improves traveler safety by providing early alerts about potential traffic risks from various disasters, integrating user-generated data from news channels and social media sites.

2.2 Product Functions (Summary)
- Safe route planning based on real-time social media data
- Real-time social media monitoring (Twitter, Facebook, Instagram) for potential hazards
- NLP processing to filter and extract hazard-related information
- Hazard severity classification
- Map updating and display with safety index
- User notifications for route updates and hazard alerts
- System administration for manual route management
- Offline capability with cached information

2.3 User Classes and Characteristics
- Travelers (Regular Users): Use GPS-based navigation, fetch warnings on dangers
- Commuters: Daily travel, avoid hazards or traffic problems
- Road Safety Authorities: Monitor hazard frequency, send assistance, inform public
- System Administrators: Ensure backend working, manage data flow, oversee system uptime
- Data Analysts: Collect insights on hazard impact on travel behavior

2.4 Operating Environment
Mobile and web applications, Firebase NoSQL database, real-time synchronization.

3. FUNCTIONAL REQUIREMENTS
3.1 Safe Route Planning
FR1.1: The system shall allow user to enter starting location and destination.
FR1.2: The system shall show safest route based on received and processed data.
FR1.3: The system shall dynamically update map if road hazard occurs along route.
FR1.4: The system shall display estimated time to reach destination.

3.2 Real-time Social Media Monitoring for Potential Hazards
FR2.1: The system shall continuously fetch tweets, Facebook, Instagram posts in real-time based on keywords.
FR2.2: The system shall process and filter incoming data to extract relevant information.
FR2.3: The system shall identify severity of detected hazard.
FR2.4: The system shall identify location where potential road hazard occurred.
FR2.5: The system shall classify hazard severity and update map.

3.3 Map Updating and Display
FR3.1: The system shall update routes if new hazards are dynamically detected.
FR3.2: The system shall remove or not display hazardous routes as top route.

3.4 User Notifications
FR4.1: The system shall send notifications to user when route is updated.
FR4.2: The system shall send alerts when hazards are detected along planned route.

3.5 System Administration
FR5.1: The system shall allow administrator to manually remove paths that are potentially dangerous.
FR5.2: The system shall allow administrator to set severity levels and update map.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: Highly reliable ensuring safest route always shown.

4.2 Usability
NFR2: User-friendly, interactive, visually appealing, easy-to-navigate map.

4.3 Performance
NFR3: Smooth functionality during hazardous activity or high data traffic.

4.4 Security
NFR4: Maintain integrity; no unauthorized access to routes or data.

4.5 Availability
NFR5: Available at all times to detect hazardous activity and update routes accordingly.
