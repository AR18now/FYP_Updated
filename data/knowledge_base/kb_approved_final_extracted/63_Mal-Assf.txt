1. INTRODUCTION
1.1 Problem Statement
Malware complexity is growing; classic static analysis approaches often fail to address new threats due to obfuscation, polymorphism, and packing techniques. Current dynamic analysis tools have scalability and accuracy issues in understanding API call sequences critical for studying malware behavior.

1.2 Scope
Mal-Assf is a desktop application for analyzing and detecting malicious binary executables using dynamic analysis, API sequence analysis, and semantic fusion. Features include dataset collection (MalwareBazaar, VirusTotal), sandbox execution, API sequence extraction, API2Vec for vectorization, BiLSTM for sequence analysis, attention mechanism, MLP classifier, and comprehensive report generation.

1.3 Definitions, Acronyms, Abbreviations
- API: Application Programming Interface
- BiLSTM: Bidirectional Long Short-Term Memory
- MLP: Multi-Layer Perceptron
- GNN: Graph Neural Network

2. OVERALL DESCRIPTION
2.1 Product Perspective
Mal-Assf provides a multi-tier malware analysis and detection framework using dynamic analysis, API sequence analysis, and semantic fusion to determine malicious actions.

2.2 Product Functions (Summary)
- Data collection from MalwareBazaar, VirusTotal, Cuckoo Sandbox
- Data cleaning, tagging, pre-processing
- Sandbox execution in isolated environment
- API sequence extraction preserving timing context
- API2Vec for vectorizing API calls
- BiLSTM for sequence pattern analysis
- Attention mechanism for focusing on relevant sequence parts
- MLP classifier for malware/benign classification
- Comprehensive report generation with behavior details

2.3 User Classes and Characteristics
- Security Analysts: Monitor and analyze malware detection results
- Users: Upload files, receive analysis reports
- System Administrators: Manage deployment, sandbox environment, classifiers
- Developers: Improve malware detection system components

2.4 Operating Environment
Desktop application, sandbox environment, machine learning models.

3. FUNCTIONAL REQUIREMENTS
3.1 Module 1 - Data Collection and Sandbox
FR1.1: The system shall collect datasets containing malware and benign samples from sources like MalwareBazaar.
FR1.2: The system shall perform data cleaning, tagging, and preliminary pre-processing of collected samples.
FR1.3: The system shall execute binary files in an isolated sandbox environment to track system behavior.

3.2 Module 2 - API Sequence Extraction, Semantic Analysis, and Classification
FR2.1: The system shall capture and record API call sequences produced by binary file execution.
FR2.2: The system shall preserve timing context of API calls for accurate sequence analysis.
FR2.3: The system shall employ API2Vec to convert API calls into vector forms for semantic evaluation.
FR2.4: The system shall use BiLSTM to identify patterns in API sequences and attention model to highlight relevant portions.
FR2.5: The system shall classify API sequences as malicious or benign using MLP classifier.
