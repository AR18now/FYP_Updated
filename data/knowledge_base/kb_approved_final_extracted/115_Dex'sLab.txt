Project: Dexter’s Laboratory: A VR Laboratory Simulator

1. INTRODUCTION
1.1 Purpose
This proposal describes the goals, characteristics, and driving force behind Dexter's Laboratory (Dex's Lab), a 3D virtual chemistry lab for O-level and 9-10 pupils, designed to improve learning experiences by providing a safe, immersive VR environment for experimentation.

1.2 Scope
The simulator is designed for chemistry education in O Levels and Pakistan Matric standards (classes 9-11). It covers experiments like cation/anion tests, acid-base titrations, pH testing, and exothermic/endothermic reactions. It focuses on providing virtual hands-on experience with accurate 3D models and realistic simulations, developed exclusively for the Meta Quest platform.

1.3 Definitions, Acronyms, Abbreviations
- VR: Virtual Reality
- 3D: Three-Dimensional
- SDK: Software Development Kit

2. OVERALL DESCRIPTION
2.1 Product Perspective
Dex's Lab bridges the gap between theoretical learning and practical application by offering a realistic virtual lab that replicates actual laboratory experiences in a secure setting, overcoming limitations of physical labs.

2.2 Product Functions (Summary)
- Asset Creation: 3D environment, equipment, textures, player movement, virtual hands.
- Mechanic Design: Chemical reaction simulation, equipment handling, liquid pouring, experiment management.
- Level Design: Implementation of curriculum-specific experiments.
- Application Design: Safety protocols, UI for instructions, tutorial system.
- Testing: Validate mechanics, performance, player movement.
- Deployment and Optimization: Performance optimization, graphics enhancement, Meta Quest integration.

2.3 User Classes and Characteristics
- Students: Primary users who engage in experiments and learning activities. They require guidance, navigation, and self-directed learning opportunities.
- (Other stakeholders like project managers, testers, documentation writers are also intended readers).

3. FUNCTIONAL REQUIREMENTS
3.1 Asset Creation
FR1: The system shall use interactive 3D modeling for a fully immersive environment.
FR2: The system shall fully support 3D equipment creation with precise textures and color schemes.
FR3: The system shall provide smooth player movement for easy navigation in the VR environment.

3.2 Mechanic Design
FR1: The system shall use a real-time engine to simulate chemical reactions (e.g., mixing acids and bases).
FR2: The system shall provide laboratory apparatus handling (burettes, pipettes, thermometers).
FR3: The system shall support pouring of liquid and measure mechanics, guaranteeing precise volume calculations.
FR4: The system shall manage sample tutorials and guidance for chemistry experiments.

3.3 Level Design
FR1: The system shall cover educational experiments on acids, bases, and salts for 9-10 and O-levels students.
FR2: The system shall teach students about acidity and alkalinity by enabling pH testing.
FR3: The system shall teach students about conducting cation and anion tests for substance analysis.
FR4: The system shall demonstrate heat exchange experiments by simulating exothermic and endothermic reactions.
FR5: The system shall conduct experiments on the physical states of matter (solid, liquid, gas).
FR6: The system shall perform experiments on chemical reactivity based on the set curriculum.
FR7: The system shall walk through basic organic chemistry concepts.

3.4 Application Design
FR1: The system must offer a virtual safety protocol (virtual safety equipment, hazard warnings).
FR2: The system shall have a user-friendly UI displaying real-time feedback and instructions.
FR3: The system shall walk users through tutorials and guidance for interacting with the virtual environment.
FR4: The main menu interface shall allow users to choose and launch various experiments.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: The application shall store the user's progress and save the main phases of an experiment.
NFR2: The system should replicate chemical reactions as close to the real world as possible (max 2% difference).
NFR3: The system should handle a minimum of 500 users without delay or inaccuracy.

4.2 Usability
NFR4: Users should be able to begin a new experiment from the home screen with no more than five clicks.
NFR5: New users shall be able to complete the basic tutorial and a small experiment within the first fifteen minutes.
NFR6: The application shall allow users to switch between controller and hands using the settings icon.

4.3 Performance
NFR7: The system should respond to user interaction with the environment within 50 milliseconds.
NFR8: Experiments conducted by the user shall be calculated and displayed in real-time without noticeable delay.

4.4 Security
NFR9: The application shall have admin control to fully manage experiments (adding, deleting, modifying).
NFR10: The system shall not save any other personal information apart from basic requirements and progress tracking.

4.5 Compatibility
NFR11: The system must be fully compatible with Meta Quest 2 and utilize VR-based functionality across multiple platforms.

4.6 Scalability
NFR12: The application must be capable of storing and managing data for at least 500 individual users.
NFR13: The system must be able to accommodate future expansions without restructuring.
