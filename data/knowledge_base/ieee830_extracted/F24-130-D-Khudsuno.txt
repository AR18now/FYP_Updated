# F24-130-D-Khudsuno

## 1. Introduction

1. Introduction 
The Khudsuno mobile application ,through avatars and real time translation aims to remove 
communication barriers for the deaf and dumb members of the society and integrate AI 
models for sign language translation(PSL only).We are conducting a test plan below to 
outline all the techniques that will be followed and the approaches taken to test all 
functionalities as well and to outline the core responsibilities and levels the app maintains 
and can be expected to fulfill 
 
2. Objectives 
 Ensure accuracy in translation 
 Validate avatar animations and their functionality 
 online offline (apps behavior testing) 
 usability, availability and security of the system. 
 Overall app performance 
 smooth integration verification (with external APIs and databases) 
3. Scope 
 Functional Testing: Gesture-to-text/speech translation, speech-to-sign translation, avatar 
accuracy. 
 Non-functional Testing: Performance (speed, scalability), usability, and compatibility 
(across devices and platforms). 
 Security Testing: User authentication, data privacy, secure communication between the 
app and cloud services. 
 Integration Testing: Test the integration between the frontend and backend also with 
external APIs. 
 Regression Testing: Ensure new changes do not affect existing functionality of Khudsuno 
mobile app. 
4. Testing Levels 
1. Unit Testing: 
o Objective: Test individual components, such as the gesture recognition model, 
speech recognition model, avatar system, etc. 
2 
o Tools: Unit testing frameworks (e.g., JUnit, pytest for Python components). 
o Responsibility: Development team. 
2. Integration Testing: 
o Objective: Verify the interaction between integrated components, like speech 
recognition and gesture-to-text conversion, backend APIs, and avatar animations. 
o Tools: Postman for API testing, Selenium for UI testing. 
o Responsibility: QA team. 
3. System Testing: 
o Objective: End-to-end testing of the entire system which ensures that the app is 
functioning as it was planned. 
o Techniques: Functional testing, usability testing, performance testing etc.. 
o Responsibility: QA team. 
 
4. User Acceptance Testing (UAT): 
o Objective: Ensure the app meets user expectations and requirements. This testing 
is done by the target user group (deaf/mute users and hearing users). 
o Techniques: Real-world testing with end-users. 
o Responsibility: User group, overseen by the product owner. 
5. Testing Techniques 
1. Black Box Testing: 
o Used for testing the gesture-to-text and speech-to-sign translation features 
without focusing on internal code logic. 
o Focused on input-output validation. 
2. White Box Testing: 
o Used for unit tests, particularly for testing the algorithms in the AI models (e.g., 
gesture recognition, speech processing). 
3. Regression Testing: 
o Regular testing after any updates to ensure that new features don't break existing 
functionality. 
o Automated testing scripts will be set up for continuous integration. 
4. Exploratory Testing: 
o Focused on testing the app's usability and performance in real-world scenarios. 
Performed by testers without predefined scripts to discover unexpected bugs. 
5. Load and Stress Testing: 
o Ensure the system can handle high volumes of requests (e.g., many users 
inputting gestures or speech simultaneously). 
o Tools: Apache JMeter for simulating load and stress conditions. 
6. Security Testing: 
3 
o Test the user authentication mechanism (login, registration), data storage 
(encryption of sensitive data), and communication security (e.g., SSL/TLS 
encryption). 
6. Test Environment 
 Devices: Android and iOS smartphones and tablets. 
 Simulators: Android Studio and Xcode simulators for mobile testing. 
 Testing Tools: Postman, Selenium, JMeter, pytest. 
 Cloud Integration: Ensure the app interacts correctly with cloud-based models and 
databases.

Software Development Plan for Khudsuno 
1. Project Overview 
The Khudsuno mobile app facilitates communication between deaf/mute and hearing 
individuals through real-time translation of sign language gestures and speech using AI 
models and avatars. The app will support multiple modes (online and offline), ensuring 
communication is possible even without internet access. 
4 
Primary Objectives 
 Develop a seamless PSL translation app with gesture-to-text and text-to-speech capabilities. 
 Integrate avatars for PSL translation, ensuring accurate representation of sign language 
gestures. 
 Ensure smooth functionality across different platforms (Android, ios). 
 
 Secondary Objectives 
 Compatibility across Devices: Making sure the app runs smoothly on various mobile platforms 
and different devices. 
 User Interface Improvements: Enhancing the app’s UI/UX based on feedback from beta testing. 
 Scalability: Preparing the app for potential future expansion to support other sign languages 
beyond PSL. 
 
Key Functionalities 
 PSL to text/audio conversion. 
 Text/audio to PSL conversion using avatars. 
 Offline mode for translation without internet. 
 Secure user authentication and data privacy. 
2. Development Methodology 
 Agile: We will follow this methodology to ensure flexibility, iterative progress and 
continuous feedback. Each sprint will focus on delivering functional increments of the app 
 Scrum: Daily Scrum meetings will help keep the team aligned and on track also. 
3. Development Phases 
1. Requirement Gathering & Analysis (2 weeks) 
o Identify user needs through stakeholder meetings and research. 
o Define clear requirements for the core functionalities (sign-to-speech, speech-to-
sign, avatar creation, etc.). 
o Tools: Jira, Confluence for documentation. 
2. Design Phase (3 weeks) 
o UI/UX Design: Create wireframes and mockups of the application interface using 
Figma. 
o System Architecture: Design the apps architecture e.g., cloud services integration, 
local processing, database structures 
o Avatar Design: Use Blender and Unity for avatar modeling and animation. 
5 
o Review designs with stakeholders. 
3. Development Phase (8 weeks) 
o Frontend Development (React Native): 
 Develop the app’s UI based on the approved designs. 
 Implement user interaction elements like gesture input and speech input. 
o Backend Development (Flask, Python, SQL): 
 Set up the server to handle requests for translation (gesture to speech 
and vice versa). 
 Implement APIs for real-time communication with external services (e.g., 
Microsoft Azure for cloud computing, Google APIs for speech). 
o Avatar Integration: 
 Animate avatars using Unity and integrate with the system to display sign 
language gestures. 
o AI Model Integration: 
 Train and integrate the gesture recognition models (CNN models) and 
speech recognition models. 
 Tools: TensorFlow, PyTorch for model development. 
4. Testing & Debugging Phase (4 weeks) 
o Perform unit, integration, and system testing (as detailed in the test plan). 
o Bug fixing and refinement based on feedback from testers and stakeholders. 
5. Deployment & User Training (2 weeks) 
o Deploy the app on Google Play Store and Apple App Store. 
o Prepare a user guide to help users understand how to use the app effectively. 
o Tools: Google Firebase for app distribution and analytics. 
6. Post-Deployment Maintenance & Updates (Ongoing) 
o Release patches and updates based on user feedback and any new requirements. 
o Maintain cloud-based services and models, ensuring smooth functioning of online 
services.
