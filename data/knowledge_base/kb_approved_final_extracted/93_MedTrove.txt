SOFTWARE REQUIREMENTS SPECIFICATION (IEEE 830)
Project: MedTrove - Affordable Medication Finder and Healthcare App
================================================================================

1. INTRODUCTION
1.1 Purpose
This document provides the requirements for MedTrove, a cross-platform application that helps users find alternative medications with similar ingredients at lower costs, along with other features like drug interaction checking and chatbot assistance.

1.2 Scope
The app allows users to search for medication, view detailed information and cheaper alternatives, compare substitutes, check drug interactions, manage profiles, use a chatbot (Medibot), make payments, donate to help others, and locate nearby pharmacies.

1.3 Definitions, Acronyms, Abbreviations
- DRAP: Drug Regulatory Authority of Pakistan
- BMI: Body Mass Index
- UI: User Interface

2. OVERALL DESCRIPTION
2.1 Product Perspective
MedTrove addresses the rising cost of medication in Pakistan by bridging the gap between patients and affordable healthcare, offering innovative solutions to find cheaper substitutes.

2.2 Product Functions (Summary)
- Substitute Drug Identifier.
- Smart Search and Recommendation.
- Drug Interaction Checker.
- Medication Reminder.
- Substitute Comparison.
- Profile Management.
- Medi-bot.
- Payment.
- Donation.
- Pharmacy Locator.

2.3 User Classes and Characteristics
- User: Primary users who seek alternatives for prescribed medicines.

2.4 Operating Environment
Cross-platform (iOS/Android) mobile application.

3. FUNCTIONAL REQUIREMENTS
3.1 Substitute Drug Identifier
FR1.1: The system shall allow users to input the name of the prescribed medication.
FR1.2: The system shall access a drug database.
FR1.3: The system shall identify alternative medicines with similar ingredients.
FR1.4: The system shall compare alternative drugs based on price.
FR1.5: The system shall display a list of alternative drugs ranked by relevance.
FR1.6: The system shall display detailed information on each alternative drug.
FR1.7: The system shall give the option to buy the alternative drug.
FR1.8: The system shall display a disclaimer advising consultation with a healthcare professional.

3.2 Search and Recommendation
FR2.1: Users shall be able to search for medicines by entering its name.
FR2.2: The system shall provide auto-complete suggestions.
FR2.3: The system shall allow filtering of search results based on price.
FR2.4: The system shall give the option to buy the medicine.
FR2.5: The system shall provide recommendations to users based on previous searches.

3.3 Drug Interaction Checker
FR3.1: The system shall allow users to input the name of two medications.
FR3.2: The system shall access a drug interaction database.
FR3.3: The system shall classify interactions based on severity.
FR3.4: The system shall display whether the medications can be taken together.

3.4 Medication Reminder
FR4.1: The user shall be able to set a medication reminder.
FR4.2: The system shall allow users to input medication details.
FR4.3: The system shall provide timely reminders and notifications.
FR4.4: The system shall maintain a log of medication intake.

3.5 Substitute Comparison
FR5.1: The system shall allow users to input the name of the prescribed medication to compare substitutes.
FR5.2: The system shall compare alternatives based on price and potential side effects.
FR5.3: The system shall highlight key differences.

3.6 Profile Management
FR6.1: The system shall allow new users to register by providing personal details.
FR6.2: The user shall be able to login using their email and password.
FR6.3: The system shall allow users to update personal information.

3.7 Medi-bot
FR7.1: The system shall provide a user-friendly interface for initiating conversations with a chatbot.
FR7.2: The user shall be able to enter medical questions.
FR7.3: The system shall provide preliminary medical advice and information about medication.
FR7.4: The system shall guide users on when to seek professional help.

3.8 Payment
FR8.1: The system shall allow users to select a payment method (credit/debit cards, JazzCash).
FR8.2: The system shall provide an option to save payment information.
FR8.3: The system shall display an order summary before payment.
FR8.4: The system shall send a confirmation notification upon successful payment.
FR8.5: The system shall maintain a record of all user transactions.

3.9 Donation
FR9.1: The system shall allow users to make donations through various payment methods.
FR9.2: The system shall maintain a log for all donations.
FR9.3: The system shall provide users with a history of their donations.
FR9.4: The system shall allow allocation of donated funds to discount orders for underprivileged users.
FR9.5: The user shall fill a form to prove eligibility for discounts.

3.10 Pharmacy Locator
FR10.1: The system shall allow users to search for nearby pharmacies based on their current location.
FR10.2: The system shall display detailed information about nearby pharmacies.
FR10.3: The system shall provide directions to searched pharmacies.
FR10.4: The system shall allow users to save favorite pharmacies.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance
NFR1: The application shall load within 4 seconds.
NFR2: Medicine results shall be displayed within 5 seconds.

4.2 Usability
NFR3: The user interface shall be easy to navigate.
NFR4: There shall be at least 2 helping screens between each frame.

4.3 Reliability
NFR5: The system shall have an uptime of 95%.
NFR6: Backups and recovery mechanisms shall be implemented to prevent data loss.
