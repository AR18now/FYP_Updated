1.4 User Classes and Characteristics
User class Description
Interior Designers The interior designers are the people who will renovate the houses of people
based on the designs ArchiVisionAI will generate. They will also use this
system to help the customers understand better about the end results of reni-
vation process. This can also help interior designers to pick a good piece of
furniture for users.
Land Owners Land and owners are people who own a piece of land and want to see how
their house will look based on the floorplan or generate a floor plan and
visualize it.
House Owners House owners are people who have their own houses and wish to renovate
their rooms or entire house , they can use ArchiVisionAI to create the re-
modelled version of their houses.
Furniture Businesses These businesses sell furnitures to people, they can enhance their customers’
decision making by using AR to see how a particular furniture i,e, sofas or
chairs will look inside customers’ rooms.

2.4 Non-Functional Requirements
2.4.1 Reliability
The application should have a mean time between failures (MTBF) of at least 500 hours.
Failure is an event where users cannot use main functionalities like 3D modeling, AR
visualization, or interior design options. The system will be able to recover from the
failure in 15 minutes.
2.4.2 Usability
The application has easy to understand interface which allows users to effortlessly gener-
ate floor plans, view furniture in AR, and receive design suggestions with minimal learn-
ing time. Users would be able to learn how to use the application within 15 minutes. voice
based prompt feature will make usability more easier even for non technical users. A user
error rate of less than 10 percent is expected. Example:
• USE-1: The user can change the interior design styling for the same model multiple
times.
• USE-2: The user can preview the furniture item using AR within 1 minute after
selecting the item.
• USE-3: The user can take a picture of their room . see how an item would look in
their room.
• USE-4: The user can see the different interior design styles.
• USE-5: The user can use voice commands to request different design ideas to re-
model a specific room (e.g., "Show me modern living room").
2.4.3 Performance
The application should have a maximum response time of 1 minute to get AR visual-
ization and 10 seconds for user interactions like generating a floor plan. For larger and
complex tasks like creating a 3D model or running interior design suggestions, the task
should not take longer than 15 seconds. Example:
• PER-1: 85 percent of user interactions should meet the 10 second response time.

3.3 Data Design
The information domain of our system is transformed into structured data that is suitable
for the implementation of different functionalities including architectural, interior design,
augmented reality (AR) and so on.The data is stored and processed in a such a way that
these features are implemented efficiently.
3.3.1 Data Entities and Structures
The major data entities as follows:
1. User Data:
• Storage:To be stored in a database with tables for users and authentication creden-
tials.
• Data Fields: UserID, Username, Email, Password and Project files.
• Processing: User data is retrieved for authentication in login/sign-up part. Project
files are stored for reuse and modifications.
32
3.3 Data Design
2. Floor Plans Data:
• Storage: Floor plans are stored as files (2D and 3D models).
• Data Fields: FloorPlanID, FilePath.
• Processing: The user inputs the prompts regarding the generation of floorplans.
This is saved for future use such as conversion to 3D model, designing interior of
3D model and so on.
3. Interior Design Data:
• Storage: Interior designs generated by AI are stored as files (2D and 3D),
• Data Fields: DesignID, FilePath.
• Processing: The users inputs the prompts regarding the styling of 2D or 3D spaces
and can store it for future reference.
4. AR Data:
• Storage: Real-time data captured by AR devices is stored temporarily in memory
or on-device storage.
• Data Fields: ObjectID, ARPlacementData.
• Processing: Real-time visualization allows users to overlay virtual furniture on
their real-world space. This information is deleted when the session ends.
3.3.2 Databases or Data Storage
The key databases and data storage items used are:
1. User Authentication Database: Stores all user credentials such as username ,pass-
word and project files.
2. Database for interior, architecture design and AR: Holds the 2D and 3D floor
plans generated by users along with interior design files.AR information will be
temporarily stored. Mostly cloud storage will be used.
