1. Introduction
1.2 Scope
The scope of the V ocalCraftproject includes:
• V oice Command Functionality: This comprises a voice recognition model which
is to transcribe the user’s voice instructions in controlling the aspects of the pam-
phlet design such as layout, images, and text content.
• Image Generation using Diffusion Model: It integrates the Diffusion Model,
which automates the generation of visual and textual content to make pamphlets
attractive-looking with correctly and professionally styled text according to user
prompts.
• Customizable Templates: To this end, the application will grant users the ability
to edit and finalize pamphlets that have been generated according to their brand
identity and marketing requirements.
• Print-Ready Output: With the application, the user can create pamphlets ready
for printing. These could be exported into numerous formats such that sharing is
easy, even in SVG format.
• Cloud-Based Storage: This lets the user save their projects online. Cloud solutions
enable them to access and manipulate their projects from anywhere; thus, much
convenience is achieved with such an option.
• Mobile App Accessibility: It will make the system accessible through the use of
mobile applications. The system will therefore have a wider usability scale for
different users.
• User-Friendly Interface: Creating a user-first interface that will ease the process
of creating pamphlets, making it friendly for users with minimum design experience
to produce quality promotional materials.

• Inference-time inpainting using a binary mask to place text and objects precisely.

3.1 Architectural Design
The architectural design followed by V ocalCraft will be API-Centric Microservices with
Event-Driven Workflow. This architecture offers flexibility, maintainability, and scalabil-
ity. Most of the core functionalities of the app are abstracted within the external APIs
and the app provides a procedural event-based workflow to the user. Event-based en-
sures smooth asynchronous communication between services, allowing the user real-time
interaction with the app. This architecture is also secure due to the API gateways, and
token-based authentication that protects the sensitive operations of the app and user’s
data.

Glossary
ASR (Automatic Speech Recognition): A technology that converts spoken language into
text using algorithms and machine learning models. In this report, it refers to the
use of the Whisper model to transcribe voice input.
Diffusion Model: A generative model that generates images by iterative denoising a ran-
dom noise image based on a given input prompt. Used for creating images in V o-
calCraft’s poster generation process.
NER (Named Entity Recognition): A process in natural language processing (NLP) that
identifies and classifies key information (entities) such as names, dates, and loca-
tions from text data. In this project, it helps extract relevant details from user input
to generate the pamphlet content.
Prompt Engineering: The process of designing input prompts for AI systems to guide
their output. In this project, the Prompt Engineering will done on the textual prompt
for the diffusion models to create a pamphlet.
Inpainting: A technique used to fill in missing or masked parts of an image. In the
V ocalCraft project, inpainting is used to add placeholders for text and object areas
within the generated image.
STT (Speech-to-Text): A technology that converts spoken language into written text.
Whisper, an STT model, will be used in this project to transcribe the user’s voice
input into text.
