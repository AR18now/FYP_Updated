1. INTRODUCTION
1.1 Purpose
This project proposal describes the intended objectives, features, and technical architecture of EduVantage, a web-based platform to guide graduates in selecting appropriate universities for higher studies using AI to automate and customize university search.

1.2 Scope
The system focuses on top 150 universities in US, UK, and EU, providing personalized recommendations based on user profiles (academic achievements, exam results, career goals). Features include scholarship recommendations, real-time monitoring of university websites for updates, university comparisons (programs, fees, rankings), and interactive preference query system.

1.3 Definitions, Acronyms, Abbreviations
- NLP: Natural Language Processing
- NER: Named Entity Recognition
- LLM: Large Language Model
- RAG: Retrieval-Augmented Generation

2. OVERALL DESCRIPTION
2.1 Product Perspective
EduVantage automates the university search process using AI (NLP, NER, LLM) to provide personalized suggestions and explanations for why specific universities are suitable fits.

2.2 Product Functions (Summary)
- User registration and authentication
- Profile creation and management
- CV upload with auto-population
- Academic data scraping from university websites
- Smart data identification using NER
- NLP-based query analysis
- LLM precision tuning for fit score calculation
- Priority URL retrieval engine
- Application deadline tracking
- Database management (MongoDB)

2.3 User Classes and Characteristics
- Graduate Students: Seek admission to master's programs, use for tailored suggestions
- Academic Advisors: Guide students through application journey
- University Admissions Officers: Ensure accurate program representation
- Parents/Guardians: Support role, concerned with financial factors

2.4 Operating Environment
Web application (MERN stack), cloud deployment.

3. FUNCTIONAL REQUIREMENTS
3.1 UI/UX Development
FR1.1: Develop responsive user interface using React.js for web and mobile platforms.
FR1.2: Ensure smooth transitions between forms and pages with uniform consistency.
FR1.3: Implement ARIA-compliant features for accessibility.

3.2 User Information Portal
FR2.1: Permit users to establish profiles via CV upload (PDF) with automatic field extraction.
FR2.2: Permit users to manually provide extra information.
FR2.3: Provide profile update and deletion options.

3.3 Academic Data Scraper
FR3.1: Gather academic information by scraping 150 specific university domains.
FR3.2: Effectively manage pagination for multi-page data retrieval.
FR3.3: Schedule data recollection (weekly/monthly) for updates.

3.4 Smart Data Identifier (NER)
FR4.1: Use pre-trained NER model to extract key information (program details, costs, deadlines).
FR4.2: Validate extracted data for accuracy and format appropriately.
FR4.3: Ensure extracted structured data is accessible for further processing.

3.5 NLP-based Query Analysis
FR5.1: Analyze user queries, extract relevant keywords, integrate with profile data.
FR5.2: Order search results based on relevance to user criteria.
FR5.3: Offer options for quick results or detailed analysis.

3.6 LLM Precision Tuning
FR6.1: Collect and maintain data to fine-tune LLM for accurate matching.
FR6.2: Develop fit score calculation normalizing and weighing user data and requirements.
FR6.3: Dynamically generate and highlight university recommendations with real-time updates.

3.7 Priority URL Retrieval Engine
FR7.1: Respond to deep analysis requests by refining queries and conducting targeted searches.
FR7.2: Implement TF-IDF based system to rank URLs by relevance.
FR7.3: Use web crawling tools to efficiently locate and retrieve data from identified URLs.

3.8 Application Deadline Tracker
FR8.1: Periodically monitor university websites for application deadline updates.
FR8.2: Send automated notifications to users about impending deadlines.

3.9 Database Management
FR9.1: Design and implement flexible NoSQL schema for diverse data types.
FR9.2: Provide efficient data ingestion, retrieval, and management for CRUD operations.
