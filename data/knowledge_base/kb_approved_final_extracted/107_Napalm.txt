SOFTWARE REQUIREMENTS SPECIFICATION (IEEE 830)
Project: Napalm - The Web Creation Kit (No-Code Website Builder)
================================================================================

1. INTRODUCTION
1.1 Purpose
This document describes the requirements for Napalm, a web development tool that facilitates drag-and-drop website building, AI-powered content generation, and real-time collaboration.

1.2 Scope
Napalm provides a no-code platform with exportable, production-ready code and customizable AI integrations. It targets freelancers, startups, and small enterprises who need flexible and open-source solutions.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- UI: User Interface
- API: Application Programming Interface
- MVC: Model-View-Controller

2. OVERALL DESCRIPTION
2.1 Product Perspective
Napalm overcomes the limitations of proprietary platforms like Webflow and Wix by providing an open-source alternative that offers complete control over the website creation process and exportable code.

2.2 Product Functions (Summary)
- UI Design, Editor, and Templating Module (drag-and-drop).
- AI Content Generation Module (text, images).
- User Authentication, Management, and Database Module.
- WebSocket Module (real-time collaboration).
- Third-Party Deployment Module (Vercel, Netlify).
- Testing and Optimization Module.

2.3 User Classes and Characteristics
- User: Regular users who create and manage websites.
- Admin: Manages platform settings, user access, and templates.
- Project: Represents an individual project.
- Template: Pre-designed website structures.

2.4 Operating Environment
Web application.

3. FUNCTIONAL REQUIREMENTS
3.1 UI Design, Editor, and Templating Module
FR1.1: The system shall provide a user-friendly drag-and-drop interface for website creation.
FR1.2: Users shall be able to choose from a set of pre-designed templates.
FR1.3: Users shall be able to edit and customize templates.
FR1.4: The system shall store project files in a format that can be exported as production-ready code.

3.2 AI Content Generation Module
FR2.1: The system shall integrate open-source AI models to generate content (text, images, etc.) based on user inputs.
FR2.2: Users shall have the option to customize AI-generated content.
FR2.3: The system shall allow users to regenerate content or provide additional inputs to refine it.

3.3 User Authentication, Management, and Database Module
FR3.1: The system shall provide secure login and user authentication.
FR3.2: Users shall have the ability to manage their profiles.
FR3.3: The system shall store user and project data in a MongoDB database via Prisma.

3.4 WebSocket Module
FR4.1: The system shall allow multiple users to work on the same project in real-time, with synchronized updates.
FR4.2: Users shall be able to see changes made by others in real-time without refreshing.

3.5 Third-Party Deployment Module
FR5.1: The system shall allow users to deploy their website projects to Vercel and Netlify.
FR5.2: Users shall be able to configure deployment settings within the platform.

3.6 Testing and Optimization Module
FR6.1: The system shall run automated tests to ensure all functionalities work as expected.
FR6.2: The system shall provide performance optimizations to reduce page load times.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: The system shall have an uptime of at least 99.9%.

4.2 Usability
NFR2: Users shall be able to create a new project within 5 minutes.
NFR3: The system shall allow users to customize a template with no more than 10 clicks.

4.3 Performance
NFR4: 95 percent of webpages generated by the platform shall load within 3 seconds on a standard 20Mbps connection.
NFR5: The system shall support up to 5 users working simultaneously on the same project.

4.4 Security
NFR6: Users' passwords shall be stored securely using hashing algorithms like bcrypt.
NFR7: The system shall have protection against common web attacks (SQL injection, XSS).
