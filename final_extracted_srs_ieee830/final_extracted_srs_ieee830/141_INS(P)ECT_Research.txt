Project: INS(P)ECT - Optimized Flea Beetle Counting using Edge Devices
================================================================================

1. INTRODUCTION
1.1 Purpose
This document outlines the requirements for INS(P)ECT, a research project focused on developing and optimizing a deep learning AI model for detecting flea beetles on sticky traps, optimized for deployment on edge devices such as mobile phones.

1.2 Scope
The project aims to create an accurate, energy-efficient, and readily accessible pest detection solution for farmers. It involves developing an AI model, optimizing it for mobile devices (reducing inference time and energy consumption), and integrating it into a mobile application. The app features image capture, automatic insect counting, offline functionality, results summary, data visualization, and cloud backup.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- FB: Flea Beetle
- SIT: Smart Insect Trap
- YOLO: You Only Look Once
- SVM: Support Vector Machine

2. OVERALL DESCRIPTION
2.1 Product Perspective
INS(P)ECT addresses the gap in accessible, automated pest detection by moving from expensive, specialized hardware (like the SIT with NVIDIA Jetson Nano) to a solution that works on commonly available mobile phones, enabling small-scale farmers to benefit from AI-driven pest management.

2.2 Product Functions (Summary)
- Image Capture and Upload: Capture images of sticky traps.
- Automatic Insect Counting: AI model processes images to detect and count flea beetles.
- Edge-AI: Model inference runs on the mobile device itself.
- Easy-to-use User Interface: Simple and intuitive UI.
- Offline Functionality: Detection and counting possible without internet.
- Results Summary and History: Compile results and keep history.
- Data Visualization: Visualize data for better understanding.
- Cloud Backup: Backup data to the cloud for storage.

- (Primary user is the farmer, though not explicitly detailed in a table, the context indicates the farmer as the end-user who captures images and receives results.)

3. FUNCTIONAL REQUIREMENTS
3.1 Functional Requirements (from SRS section)
FR1: The application will allow users to capture images of sticky traps which can be analyzed for the detection of flea beetles.
FR2: The application will have an AI model process these images to accurately identify and count the flea beetles captured on the sticky trap.
FR3: The application will have a user-friendly and intuitive user interface for ease of navigation and use.
FR4: The application will generate reports for the analysis of the population of flea beetles present in the field and provide information about the trend of population increase or decrease.
FR5: The application will allow these reports to be stored on the cloud where they can be easily accessed and stored.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Non-Functional Requirements (from SRS section)
NFR1: The application will inference the model to provide the results in a minimal time span.
NFR2: The application will be optimized for low battery consumption to prolong battery life.
NFR3: The application will have a user-friendly and intuitive user interface for ease of navigation and use.
NFR4: The application will be compatible with a range of mobile devices to ensure wide availability.
NFR5: The application will handle data securely, only allowing access to authorized users.
