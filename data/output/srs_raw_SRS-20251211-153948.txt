Software Requirements Specification (SRS)

1. IntroductionPurpose:To define the functional and non-functional requirements for the e-commerce system that will manage issue logging, assignment, and tracking for HVAC, electrical, and plumbing issues.

Scope:This document covers the requirements for a web-based application that allows staff to log issues, managers to assign tasks, and technicians to update their progress. The system also includes reporting and mobile access functionalities.

Definitions/Acronyms:- HVAC: Heating, Ventilation, and Air Conditioning
- SLA: Service Level Agreement
- UI: User Interface

References:- IEEE 830-1998 Standard for Software Requirements Specifications
- Project Charter

Overview:The e-commerce system will enable efficient management of maintenance requests by providing a structured process for logging, assigning, updating, and reporting on issues related to HVAC, electrical, and plumbing systems.

---

2. Overall DescriptionProduct Perspective:The e-commerce system will be a web-based application accessible via both desktop and mobile devices. It will support multiple users including staff, managers, and technicians.

Product Functions:- Staff can log issues with photos and specify priority.
- Managers can assign tickets to technicians based on skills and availability.
- Technicians can update ticket status, record parts used, and log time spent.
- The system will automatically escalate overdue tickets.
- Weekly SLA reports will be generated.
- A simplified mobile view will be available for technicians.
- Tickets will capture location and contact details for the requesting employee.

User Characteristics:- Staff: Individuals responsible for identifying and logging maintenance issues.
- Managers: Individuals responsible for assigning tasks to technicians.
- Technicians: Individuals responsible for resolving maintenance issues.

Constraints:- The system must be compatible with major web browsers and mobile devices.
- Data security and privacy must be maintained.
- The system must have a response time of less than 5 seconds for critical operations.

Assumptions/Dependencies:- The system will use a relational database for storing issue logs and other data.
- Internet connectivity is required for all users.

---

3. Specific RequirementsExternal Interface Requirements:- User Interface (UI):- Staff: Web interface for logging issues, viewing assigned tickets, and updating ticket status.
  - Managers: Web interface for assigning tickets, viewing SLA reports, and managing technician assignments.
  - Technicians: Simplified mobile interface for viewing assigned tickets, updating status, recording parts used, and logging time spent.

- Hardware:- Desktops and laptops running modern operating systems.
  - Mobile devices (smartphones and tablets) supporting web browsers.

- Software:- Web browser compatibility: Chrome, Firefox, Safari, Edge.
  - Mobile app: Android and iOS.

- Communication:- Secure HTTP (HTTPS) for data transmission.
  - Email notifications for overdue tickets and SLA report generation.

Functional Requirements:- FR-1 (High Priority):Staff can log issues with photos and specify priority.
  - Input: Issue description, photos, priority level.
  - Processing: Store issue in the database.
  - Output: Confirmation message to staff.

- FR-2 (Medium Priority):Managers can assign tickets to technicians based on skills and availability.
  - Input: Technician ID, skill set, availability.
  - Processing: Match tickets to suitable technicians.
  - Output: Assignment confirmation to manager and technician.

- FR-3 (High Priority):Technicians can update ticket status, record parts used, and log time spent.
  - Input: Status update, parts used, time spent.
  - Processing: Update ticket in the database.
  - Output: Confirmation message to technician.

- FR-4 (Medium Priority):The system will automatically escalate overdue tickets.
  - Input: Ticket age.
  - Processing: Check if ticket is overdue.
  - Output: Escalation notification to manager.

- FR-5 (Low Priority):Weekly SLA reports will be generated.
  - Input: Date range.
  - Processing: Generate report summarizing ticket resolution times.
  - Output: Report sent via email.

- FR-6 (Low Priority):A simplified mobile view is needed for technicians.
  - Input: Mobile device.
  - Processing: Provide a streamlined UI for viewing and updating tickets.
  - Output: Mobile-friendly interface.

- FR-7 (Low Priority):Tickets will capture location and contact details for the requesting employee.
  - Input: Location and contact details.
  - Processing: Store information in the database.
  - Output: Confirmation message to staff.

Non-functional Requirements:- Performance:Response time for critical operations (e.g., logging issues, assigning tickets) should not exceed 5 seconds.
- Security:Data must be encrypted during transmission and stored securely.
- Reliability:The system must be available 99.9% of the time.
- Usability:The user interfaces must be intuitive and easy to navigate.
- Maintainability:The codebase should be modular and well-documented for easy updates and maintenance.
- Portability:The system must be deployable on various platforms and devices.

---

4. ConclusionIn conclusion, the e-commerce system will provide a comprehensive solution for managing maintenance issues related to HVAC, electrical, and plumbing systems. The system will ensure efficient logging, assignment, and tracking of issues while maintaining high standards of security, performance, and usability. Next steps include detailed design, development, and testing phases. Recommendations include conducting user acceptance testing and continuous improvement based on feedback.
