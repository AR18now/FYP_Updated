1.3 Modules
The ASLi project consists of several key modules, each designed to fulfil specific func-
tionalities of the application. The modules are organized according to their roles within
the system, including the frond-end (Client Mobile App) and the Backend Service.
1.3.1 Client Mobile App
In this module, we will focus on the user-facing aspect of the ASLi application, providing
an interface for deaf individuals to interact with the app and access its features.
1. ASL Gesture Recognition: This module captures ASL signs using the mobile’s
camera, and converts ASL gestures into written English in real-time.
2. Text-to-ASL Translation: Here we will translate written English responses from the
LLM into ASL and display translated ASL using a digital avatar.
3. User Interface: It is essential that ASLi provides a user-friendly interface for easy
navigation and interaction.
4. Real-Time Performance: We need to ensure smooth, real-time translation and com-
munication and optimize performance for various mobile devices.
1.3.2 Backend Service
Here we are handling the core processing and functionality of the ASLi app, including
data management and communication with external systems.
1. ASL-to-English Translation Engine: We will utilize advanced ML models to accu-
rately translate ASL gestures into written English, continuously improving transla-
tion accuracy.
2. Information Retrieval System: Now, we will query the internet for relevant infor-
mation based on user input and refine responses from the large language model
(LLM).

Chapter 3
System Overview
Our project is an interactive, mobile-based application aimed at providing a seamless
communication bridge for the deaf and hard-of-hearing community through American
Sign Language (ASL) gesture recognition and translation.
The app captures ASL gestures through the mobile’s camera, translates them into English
text, processes the text to retrieve answers, and converts the retrieved text back into ASL.
This back-and-forth communication is facilitated by a 3D avatar that performs the ASL
gestures in real-time, offering users a visual and immersive experience.
The context for ASLi’s development is to offer deaf users the ability to perform tasks like
querying the internet, searching for information, or expressing complex ideas using their
natural language, ASL.
The project was conceptualized and developed as a final-year university project by us,
Qasim and Musfirah. We identified the challenges faced by the deaf community. Our
personal experiences, particularly with Qasim’s nephew Ali Raza who has a hearing im-
pairment, inspired the creation of ASLi.
The aim is to build an inclusive platform that breaks down the communication barriers
and enables deaf individuals to access the same wealth of information and services that
hearing individuals take for granted.

3.4 Data Design
The data design for ASLi focuses on how the system’s information domain is represented
and transformed into data structures.
In this section, we will outline the major system entities, how they are stored, processed,
and managed to ensure smooth operation and secure handling of user information.
3.4.1 Data Structures and Organization
The ASLi system deals with various data elements, including user credentials, ASL ges-
ture data, LLM queries and responses, and user preferences.
The core data is organized and stored in a database designed to ensure data integrity,
security, and accessibility.
3.4.1.1 Major Data Entities
The following are the key data entities within the ASLi system:
• User Information: This includes user profiles, login credentials, and personal set-
tings.
• ASL Gesture Data: Data that represents ASL gestures recorded from the user.
• LLM Queries and Responses: Text data sent to the LLM for generating responses
and the respective results returned.
• 3D A vatar Gesture Mapping: Data that maps ASL gestures to the 3D avatar for
real-time rendering.
All data entities are structured in such a way that they can be efficiently queried and
processed to provide real-time translation and gesture rendering.
3.4.1.2 User Login Credentials
To ensure the security of user login credentials, ASLi employs the following strategies:
• Password Encryption: User passwords are hashed using a strong one-way hashing
algorithm such as bcrypt. This ensures that even if the database is compromised,
the actual passwords cannot be retrieved.

3.4.3 Data Flow and Processing
The system processes ASL gesture data by first capturing it through the mobile interface
and sending it to the backend for translation into written English.
This data is temporarily stored and then forwarded to the LLM for response generation.
Once the LLM response is received, it is processed, converted into ASL gestures, and
displayed using the 3D avatar.
ASLi’s data design is structured to ensure that user information is stored securely, gesture
data is processed efficiently, and the overall system is both scalable and reliable as the
application grows.
