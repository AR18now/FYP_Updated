1. INTRODUCTION
1.1 Purpose
This document outlines the requirements for NeuroGen, a research project focused on using advanced inpainting techniques (Denoising Diffusion Probabilistic Models, GANs, VAEs) to generate healthy 3D brain tissue in regions affected by glioma tumors, thereby improving the reliability of MRI analysis for clinical tasks.

1.2 Scope
The research involves training models on the BraTS dataset (T1 MRI modality) to synthesize healthy tissue in tumor-affected areas. The goal is to enhance data integrity for downstream tasks like brain tissue segmentation, anatomical parcellation, and tumour growth modelling. The project explores various models (DDPMs, GANs, VAEs, LDMs) and evaluation metrics (SSIM, PSNR, MSE).

1.3 Definitions, Acronyms, Abbreviations
- MRI: Magnetic Resonance Imaging
- DDPM: Denoising Diffusion Probabilistic Model
- GAN: Generative Adversarial Network
- VAE: Variational Auto-encoder
- LDM: Latent Diffusion Model
- SSIM: Structural Similarity Index Measure
- PSNR: Peak Signal-to-Noise Ratio

2. OVERALL DESCRIPTION
2.1 Product Perspective
NeuroGen addresses the challenge of analyzing brain MRIs with gliomas, where standard algorithms (trained on healthy brains) underperform. By inpainting tumor regions with realistic healthy tissue, it enhances the performance of clinical analysis tools.

2.2 Product Functions (Summary)
- Data Preprocessing: Rescaling to consistent voxel size (1x1x1 mm³), normalization (Z-score, percentile), quality control.
- Model Training: Train and compare inpainting models (DDPMs, GANs, VAEs, LDMs) using T1 MRI scans from BraTS 2024.
- Inpainting: Generate healthy brain tissue within the masked tumor-affected region.
- Evaluation: Assess model performance using SSIM, PSNR, MSE.

2.3 User Classes and Characteristics
- (The primary "users" are researchers and clinicians who would use the inpainted images for better diagnosis and analysis. The document focuses on the research methodology.)

3. FUNCTIONAL REQUIREMENTS
3.1 Verifying the BraTS Challenge Dataset
FR1: Download the BraTS dataset and confirm the presence of ground truth segmentation masks for tumor regions.
FR2: Perform visual inspection on a sample MRI volume to check alignment and identify corrupted files.
FR3: Calculate statistical measures (mean, variance, intensity histograms) to find anomalies.

3.2 Workflow for Rescaling, Normalization, and Ensuring Consistent Voxel Size
FR1: Rescale all MRI volumes to a consistent voxel size of 1x1x1 mm³ using 3D interpolation.
FR2: Apply Z-score normalization (subtract mean, divide by standard deviation) for each modality to standardize intensity ranges.
FR3: Verify that all MRI volumes have consistent voxel sizes by checking metadata; resample mismatched images.
FR4: Perform quality control checks (visual, statistical) to ensure preprocessing did not distort data.

3.3 Model Training (implied)
FR1: The system shall train inpainting models (e.g., DDPMs) to generate healthy tissue in tumor-affected regions.
FR2: The system shall use evaluation metrics (SSIM, PSNR, MSE) to assess the quality of inpainted images.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance
NFR1: Models should be computationally efficient enough to handle 3D volumetric data, potentially using techniques like pseudo-3D or memory-efficient architectures.
NFR2: Inpainting should produce high-quality reconstructions with high SSIM and PSNR scores.

4.2 Scalability
NFR3: The methodology should be scalable to handle large datasets like BraTS.
