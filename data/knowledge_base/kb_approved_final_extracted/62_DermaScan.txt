1. INTRODUCTION
1.1 Problem Statement
Lack of dermatologists and healthcare services, particularly in remote areas, results in delayed treatment of skin diseases. Visiting doctors is costly and time-consuming. Mobile app provides AI-driven system for skin disease detection and doctor recommendations.

1.2 Scope
DermaScan is a mobile application enabling early detection of skin diseases (Psoriasis, Acne, Eczema, Vitiligo, Skin Cancer) using CNN models, LLM-powered medical chatbot for symptom logging and guidance, and doctor recommendations based on disease and geo-location.

1.3 Definitions, Acronyms, Abbreviations
- CNN: Convolutional Neural Network
- LLM: Large Language Model
- ISIC: International Skin Imaging Collaboration

2. OVERALL DESCRIPTION
2.1 Product Perspective
DermaScan helps users manage skin healthcare and reduces burden on healthcare systems through early detection, progress tracking, and dermatologist recommendations.

2.2 Product Functions (Summary)
- Patient profile management
- Medical records management
- Disease symptom logging
- Skin disease detection using CNN model
- Diagnosis results report generation
- Progress tracking of disease improvement
- Doctor recommendations based on disease and region
- LLM-powered chatbot for symptom analysis and guidance

2.3 User Classes and Characteristics
- Patients: Diagnose skin conditions, get information from home
- Dermatologists (Healthcare Providers): Offer services, connect with patients
- Researchers (Medical/AI/ML): Analyze algorithms, evaluate AI effectiveness

2.4 Operating Environment
Mobile application (Android/iOS), cloud-based CNN and LLM models.

3. FUNCTIONAL REQUIREMENTS
3.1 Module 1 - Patient Profile
FR1.1: The system shall allow users to login/register on the app.
FR1.2: The system shall allow users to view and update user profiles.
FR1.3: The system shall allow users to maintain medical records and disease history.
FR1.4: The system shall allow users to log symptoms for a particular disease.
FR1.5: The system shall allow users to write and submit feedback reports related to app functionality and doctors.

3.2 Module 2 - Skin Disease Detection
FR2.1: The system shall allow users to access phone's camera to capture images for disease detection.
FR2.2: The system shall enable users to obtain analysis results generated by CNN model.
FR2.3: The system shall allow users to save or download obtained reports.
FR2.4: The system shall provide users with recommended doctors based on disease and geo-location.

3.3 Module 3 - LLM Powered Chatbot
FR3.1: The system shall allow users to interact with chatbot using text prompts.
FR3.2: The system shall ensure improved disease detection by catering to user-prompted symptoms in addition to disease report.
FR3.3: The system shall provide users with accurate information and feedback for entered user prompts.

3.4 Module 4 - Doctor Profile
FR4.1: The system shall provide separate gateway for healthcare professionals.
FR4.2: The system shall allow healthcare professionals to view and update profiles.
FR4.3: The system shall enable and ensure connection and communication between patients and doctors.
FR4.4: The system shall cater to doctor's geo-location when recommending to patients.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance
NFR1: Skin disease detection results within 5 seconds after image processing.
NFR2: Chatbot response within 5 seconds.
NFR3: CNN model processes at least 100 image requests per minute.

4.2 Security
NFR4: All user data stored in Firebase with Firebase security.

4.3 Availability
NFR5: System uptime ≥ 99.9%.

4.4 Usability
NFR6: User-friendly interface with smooth navigation.

4.5 Reliability
NFR7: CNN model accuracy ≥ 91%.
NFR8: LLM-powered chatbot accuracy up to 95%.
