1. INTRODUCTION
1.1 Purpose
This document outlines the requirements for NewsNet, a system that uses GraphRAG (Retrieval-Augmented Generation) and causal knowledge graphs to analyze news, detect causal relationships between events, and provide users with a deeper understanding of news stories.

1.2 Scope
The system scrapes unstructured news text, extracts named entities and events (NEER), identifies causal relationships, constructs a causal knowledge graph, and uses GraphRAG for enhanced information retrieval. Features include category-based filtering, timeline generation, personalized recommendations, trend analysis, anomaly detection, and a chatbot interface.

1.3 Definitions, Acronyms, Abbreviations
- RAG: Retrieval-Augmented Generation
- GraphRAG: Graph-based Retrieval-Augmented Generation
- LLM: Large Language Model
- NEER: Named Entity and Event Recognition
- KG: Knowledge Graph

2. OVERALL DESCRIPTION
2.1 Product Perspective
NewsNet addresses the difficulty of understanding cause-effect relationships in news events by building a causal knowledge graph and using it to enhance LLM retrieval, providing users with a coherent narrative and deeper insights.

2.2 Product Functions (Summary)
- Data Collection and NEER: Web scraping, data cleaning, Named Entity and Event Recognition.
- Causal Knowledge Graph: Identify causal relationships, construct and update KG.
- GraphRAG: Fine-tune LLM, use causal KG to enhance retrieval.
- GraphRAG vs Baseline RAG: Implement baseline RAG, compare performance.
- Chatbot Creation: Chatbot interface, personalized KG visualization, timeline layout, article references, trend analysis.
- Quality Assurance: Performance metrics, unit testing, system testing.
- User Experience Optimization: Category-based search, user-tailored recommendations.

2.3 User Classes and Characteristics
- News Viewer: Regular consumer wanting current updates on topics of interest.
- History Learner: Studies past events and backgrounds, uses timelines and resources.
- Researcher: Academia/industry needing trustworthy, detailed information for work/publications.
- Media Enthusiast: Loves media trends, production, communication analysis.

3. FUNCTIONAL REQUIREMENTS
3.1 Data Collection and Named Entity and Event Recognition (NEER)
FR1: The system shall collect data from news websites through web scraping tools.
FR2: The system shall pre-process and clean the data.
FR3: The system shall implement Named Entity and Event Recognition model on the news data.

3.2 Causal Knowledge Graph
FR1: The system shall identify causal relationships among entities in the data.
FR2: The system shall construct a causal knowledge graph (KG) using the causal relationships.
FR3: The system shall update and add new relationships to the knowledge graph as more data is available.

3.3 GraphRAG
FR1: The system shall use the data from the news to fine-tune a Large Language Model (LLM).
FR2: The system shall use the causal knowledge graph to improve and enhance information retrieval using Graph Retrieval-Augmented Generation (GraphRAG).

3.4 GraphRAG vs Baseline RAG
FR1: The system shall implement a baseline Retrieval-Augmented Generation (RAG).
FR2: The system shall utilize the baseline RAG to perform a performance comparison between RAG and GraphRAG.
FR3: The system shall test the system performance of GraphRAG using predefined metrics.

3.5 Chatbot Creation
FR1: The system shall create a chatbot interface that provides options to users to select news.
FR2: The system shall create and provide a personalized knowledge graph to the user according to their query.
FR3: The system shall display the news results according to the relative timeline.
FR4: The system shall provide references for the articles in which the news was provided.
FR5: The system shall provide a trend analysis based on the news.

3.6 Quality Assurance
FR1: The system shall be monitored and tested according to performance metrics.
FR2: The system shall be tested to verify the functionality of each module.
FR3: The system shall have system-wide testing to ensure all functionality and their integration as well as performance.

3.7 User Experience Optimization
FR1: The system shall provide options for searching using different categories.
FR2: The system shall recommend news to users according to their search history.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: The system will provide a detailed log of errors in case of system failure.
NFR2: The system shall provide an error window to the user that will provide the nature of the failure.
NFR3: The system shall be designed by a strategy that will minimize the percentage of error or system failure to less than 5%.

4.2 Usability
NFR4: The system user interface (UI) will be highly intuitive.
NFR5: The system will allow the user to search up relevant news with a single interaction.
NFR6: The system shall provide a simple guide for users to be able to utilize the application with ease.

4.3 Performance
NFR7: The system will load different modules in the application within 3 seconds.
NFR8: The system shall utilize fast retrieval and high-speed databases to ensure high performance.
NFR9: The system will utilize top-of-the-line models and libraries to ensure quick processing and minimum delays.

4.4 Security
NFR10: The system will be secure against breach in user data in storage or transit.
NFR11: The system will respond appropriately to requests that surpass a normal human standard.
