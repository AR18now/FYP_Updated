Use Case Name: Doctor Search and Appointment Booking- Primary Actor: User Stakeholders and Interests: User: wants the system to complete 'Doctor Search and Appointment Booking-' correctly and quickly. Preconditions: Patient searches for doctors by specialty. - Postconditions: Display list of available doctors and allow patient to select a doctor and book an appointment. - Main Success Scenario: 1. Actor initiates 'System retrieves available doctors and their schedules. -'. 2. System validates request details. 3. System executes the requested operation. 4. System returns confirmation to the actor. Extensions: If validation fails, the system displays a clear error and asks for correction. Special Requirements: Response must meet defined performance and security constraints. Frequency of Occurrence: Multiple times per day during normal system usage. Assumptions: Users have valid access rights and network connectivity is available.
