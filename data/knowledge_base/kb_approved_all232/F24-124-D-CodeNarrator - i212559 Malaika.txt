3.1 Architectural Design
The architecture of CodeNarrator follows a layered architecture pattern, which is well-
suited for its modular and scalable nature. This approach provides a clear separation of
concerns, with each layer responsible for distinct tasks. The system is made up of three
layers:
1. UI Layer: User interactions are handled by the UI layer. Through the CodeNar-
rator Web UI, users can submit requests to generate README files, update code
comments, or update documentation. Serving as the user interface, this layer makes
it easier for users to communicate with the application layer.
2. Application Layer: Requests and business logic are processed by this layer. The
CodeBERT (NLP) model is used by the Request Handler module to analyze
code, handle user commands, and carry out dynamic changes to the README
files. Based on commits and changes made to the code in the repository, the Dy-
namic Update Handler makes sure that the comments and documentation are up
to date.
3. Data Layer: This layer controls communication with outside services and storage
of documentation. It synchronizes the generated README files and code com-
ments with the CodeNarrator Database and interfaces with the GitHub API to fetch
and push updates to repositories.
This layered approach provides clear responsibility for each component:

• The UI Layer handles all user interactions,
• The Application Layer processes the logic and commands, and
• The Data Layer stores and manages the data.

3.2 Design Methodology
Code Narrator’s layered architecture aligns perfectly with the Object-Oriented (OO)
methodology. Each layer—UI, Application, and Data—follows the OO principles of
modularity and encapsulation, making components like the Request Handler, Code-
BERT NLP, and Dynamic Update Handler independent and manageable. The clear
component interaction ensures seamless communication between modules, while the
system’s scalability allows new features to be added without disrupting existing func-
tionality. This combination of OO and layered design provides flexibility, maintainability,
and supports real-time updates efficiently.
