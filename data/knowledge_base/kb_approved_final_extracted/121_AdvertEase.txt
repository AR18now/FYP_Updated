SOFTWARE REQUIREMENTS SPECIFICATION (IEEE 830)
Project: AdvertEase - AI-Generated Poster and Video Advertisements
================================================================================

1. INTRODUCTION
1.1 Purpose
This document outlines the requirements for AdvertEase, a web application that automates the creation of poster and video advertisements using AI (LLM, image/video generation) to help businesses, especially small and middle-tier furniture businesses, market products easily and cost-effectively.

1.2 Scope
The system generates ads from a product URL. Features include web scraping, LLM script generation, poster image generation (with prompt-based editing), video ad generation with a human avatar (male/female) and enhanced emotional voiceover, and cloud deployment (AWS).

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- LLM: Large Language Model
- URL: Uniform Resource Locator
- AWS: Amazon Web Services

2. OVERALL DESCRIPTION
2.1 Product Perspective
AdvertEase simplifies and accelerates the ad creation process, allowing businesses to input a product link and receive customized, compelling ads without needing specialized design or marketing teams.

2.2 Product Functions (Summary)
- AI-generated poster ads from product URL (with web scraping, text preprocessing, LLM script generation, image generation).
- AI-generated video ads (LLM script, video generation via inpainting/control techniques, avatar generation, templates).
- Website development (MERN stack).
- Cloud deployment (AWS).

2.3 User Classes and Characteristics
- End User: Individuals/companies creating ads. They need an easy, fast process.
- Admin: Oversees platform operation, user management, system performance.
- Marketing Experts: Improve scripts, avatars, and templates.
- Developers: Maintain backend, deploy, integrate AI modules.

3. FUNCTIONAL REQUIREMENTS
3.1 Authenticate User
FR1.1: The system shall provide a login page.
FR1.2: The system shall verify user credentials using a database.
FR1.3: The system shall grant access to the dashboard after a successful login.

3.2 Input Product URL
FR2.1: Users shall be able to enter a legitimate product URL.
FR2.2: The system shall verify the format of the entered URL.

3.3 Scrape Product Data
FR3.1: The system shall extract product data (photos, descriptions, features) from the given URL.
FR3.2: The system shall handle data scraping errors and alert the user.

3.4 Generate Poster Ads
FR4.1: The system shall use scraped product data to produce aesthetically appealing poster adverts.
FR4.2: Users shall be able to alter the poster advertisement's design aspects.
FR4.3: The user shall be given a preview of the generated poster advertisement.

3.5 Generate Advertisement Script
FR5.1: The system shall use product data to create an advertisement script.
FR5.2: Users shall be able to modify the generated script.

3.6 Edit Advertisement
FR6.1: The system shall allow users to edit the generated script and poster ad.
FR6.2: The system shall save user edits.

3.7 Select Avatar
FR7.1: The system shall let users choose a video ad avatar.
FR7.2: The system shall make a library of selectable avatars available.

3.8 Modify Script
FR8.1: After generation, users shall be able to edit the advertisement script.
FR8.2: The system shall validate the changes to ensure coherence.

3.9 Synthesize Speech from Script
FR9.1: The system shall employ text-to-speech technology to translate the commercial script into audio.

3.10 Enhance Emotional Tone in Speech
FR10.1: In accordance with the script's nature, the system shall make the synthesised speech sound more emotionally charged.

3.11 Generate Video Ads
FR11.1: The system shall combine the script, synthesised speech, and poster advertisement to produce a video advertisement.
FR11.2: The user shall be given a preview of the generated video.

3.12 Download Ads
FR12.1: Users shall be able to download produced video advertisements and posters in various formats.

3.13 Share Ads
FR13.1: Users shall have the ability to share created adverts on social media sites via system options.

3.14 Manage Users
FR14.1: The admin shall be able to view a list of registered users.
FR14.2: The admin shall be able to add, modify, and remove user accounts.
FR14.3: The system shall notify users when the administrator modifies their accounts.

3.15 Generate Reports
FR15.1: The system shall allow the administrator to provide reports on user behaviour and ad creation.
FR15.2: The system shall require customisation choices for report criteria (date range, user activity).
FR15.3: The admin shall be able to retrieve created reports in various forms (PDF, CSV).

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: The system must maintain an average MTBF of at least 1000 hours.
NFR2: Should a failure occur, the system will record the error and notify the administrator in five minutes.
NFR3: The system must have automated recovery methods to function again within two minutes.
NFR4: To guarantee data integrity, the system must regularly backup user data and configurations (once every 24 hours).

4.2 Usability
NFR5: The system must have an interface that lets users create advertising from the dashboard in only three clicks or less.
NFR6: The system will include tooltips and contextual help for all significant functions.
NFR7: With only one click, users will be able to access previously created advertisements from their user dashboard.
NFR8: The system must include accessibility features (keyboard navigation, screen reader compatibility).

4.3 Performance
NFR9: The system will create a poster advertisement in no more than five seconds after the user enters the product URL and data.
NFR10: The system should be able to parse and scrape data from a product URL in 10 seconds.
NFR11: The system must guarantee that a video ad is created in a maximum of 5 minutes and poster ads in 10 seconds.
NFR12: The system must support a minimum of 100 concurrent users without compromising performance.

4.4 Security
NFR13: The system must encrypt all sensitive user data (login passwords, personal information) both in transit and at rest.
NFR14: Role-based access control (RBAC) will be implemented to limit access to various capabilities.
NFR15: The system must conduct routine security audits to find and fix vulnerabilities.
NFR16: The system must offer defence against SQL injection, XSS, and CSRF.
