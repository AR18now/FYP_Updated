1. INTRODUCTION
1.1 Purpose
This report outlines the problem statement, scope, module divisions, and system design for TalentVerify, an AI interviewing web application with a focus on integrity and evaluating soft skills.

1.2 Scope
TalentVerify provides an interface for employers to set up interviews and for candidates to participate. It evaluates soft skills in real-time, verifies integrity, and flags suspicious activities, acting as an automated screening layer.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- HR: Human Resources
- MTBF: Mean Time Between Failures

2. OVERALL DESCRIPTION
2.1 Product Perspective
TalentVerify automates the initial hiring process to filter candidates, saving companies time and cost. It uses deep learning models to accurately identify audio and video emotions for soft skill evaluation.

2.2 Product Functions (Summary)
- Web interface for employers and interviewees.
- Audio and video emotion detection.
- Cheating detection (multiple entity detection, deepfake detection).
- Question generation (technical and behavioral) based on resume and job description.
- Answer evaluation.
- Avatar generation for a human-like interface.
- Dashboard for employers to review candidate performance.

2.3 User Classes and Characteristics
- Employer: Sets up interviews, provides shortlisting criteria.
- Interviewee: Participates in the scheduled interview.
- Admin: Overviews the system, checks logs, monitors shortlisting.

2.4 Operating Environment
Web application.

3. FUNCTIONAL REQUIREMENTS
3.1 Web Application
FR1.1: The interviewee shall be able to view all interviews created by the employer.
FR1.2: The employer shall be able to set up an interview by setting filtering criteria.
FR1.3: The system shall authenticate users and provide role-based interfaces.
FR1.4: The interviewee shall be able to filter interviews by name and organization.

3.2 Emotion Detection Pipeline
FR2.1: The system shall be able to identify emotions from video key frames.
FR2.2: The system shall be able to identify emotions from audio samples.

3.3 Cheating Detection
FR3.1: The system shall detect multiple facial entities during the interview process.
FR3.2: The system shall detect deepfake faces from a given video key frame.

3.4 Question Generation
FR4.1: The interviewee shall receive personalized behavioral questions based on their resume.
FR4.2: The employer should be able to generate technical questions based on the job description.

3.5 Answer Evaluation
FR5.1: The system shall verify facts in a response to a technical question.
FR5.2: The system shall score answers to behavioral questions.

3.6 Avatar Generation
FR6.1: The interviewee shall be able to view an avatar during the interview.
FR6.2: The interviewee shall be able to listen to the avatar during the interview.

3.7 Dashboard
FR7.1: The employer shall be able to view candidates' performance after the interview.
FR7.2: The employer shall be able to view shortlisted candidates after the interview.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: The application shall have an MTBF of at least 100 hours.
NFR2: In the event of failure, the system shall recover within 3 minutes (interview restarts).
NFR3: System failures shall trigger an automatic error log and revert to the checkpoint before the interview.

4.2 Usability
NFR4: First-time users shall achieve a 90% first-time completion rate.
NFR5: The employer's dashboard shall be understandable within 2 minutes.
NFR6: The system shall display clear error messages and suggestions, reducing input errors by 80%.

4.3 Performance
NFR7: 95% of interview sessions shall have less than a 2-second delay for real-time emotion detection.
NFR8: The resume analysis function shall generate questions within 10 seconds of submission.

4.4 Security
NFR9: The application shall have role-based access control.
NFR10: Sensitive information such as passwords shall be encrypted.
