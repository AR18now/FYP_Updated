1.3 Research Problem Statement
Current mobile games deliver preset commentaries that are available in limited commen-
tators’ voices. These commentaries are not only repetitive but they are also unable to cap-
ture the complete context of the game. We will provide a user-friendly web interface that
will handle commentary generation that will be non-repetitive and contextually aware.
This will increase audience engagement who will benefit from informed commentary.
1.4 Software Requirements Specification
1.4.1 Functional Requirements
• User Authentication: The software should support secure login and account man-
agement, allowing users to access personalized settings and stored data.
• Video Upload and Processing: Users must be able to upload cricket match clips,
which will then be trimmed, analyzed, and processed for event detection, score-
board extraction, and commentary generation.
• Preset Selection: The system should allow users to choose predefined commentary
styles and configurations to suit different commentary tones.
• Voice Selection: Users should be able to select from a variety of voices for the
generated commentary, with options for accents, genders, and tones.
• Real-Time Processing: The software should provide commentary generation in
real-time or near real-time, based on the uploaded video clips.
1.4.2 Non-Functional Requirements
These define the operational qualities and constraints of the software, ensuring perfor-
mance, usability, and scalability:
• Performance: The system should be able to handle multiple video uploads simul-
taneously and process them efficiently without significant delays.
• Usability: The interface must be intuitive, enabling users to easily navigate the
software and make selections for uploads, presets, and voices.
• Scalability: The system must be capable of scaling to accommodate an increasing
number of users and video uploads as demand grows.

2.1.1 MatchTime: Towards Automatic Soccer Game Commentary
Generation
In recent years, significant work has been conducted to improve automatic soccer com-
mentary generation, with datasets like SoccerNet-Caption serving as the foundation for
many models. This paper proposed MatchTime, a curated dataset that corrects these tem-
poral inconsistencies. MatchTime is built using a two-stage multi-modal temporal align-
ment pipeline that filters and adjusts the timestamp discrepancies in existing datasets. By
aligning the visual content with the commentary text more precisely, MatchTime enables
the training of models that generate more accurate and engaging soccer game commen-
tary.
2.1.1.1 Summary of the research item
This paper focuses on the challenge of video-text misalignment in soccer commentary
datasets, specifically SoccerNet-Caption, which collects commentary from over 400 matches.
The study highlights a significant issue where commentary timing can lag behind game

2.1.2 CricShotClassify: An Approach to Classifying Batting Shots
from Cricket Videos Using a Convolutional Neural Network
and Gated Recurrent Unit
This paper focuses on the challenging task of classifying cricket batting shots using deep
learning techniques. Traditional methods of manual feature extraction from video frames
are time-consuming and prone to errors due to the high similarity between different cricket
shots. This paper introduces a hybrid deep learning model combining a Convolutional
Neural Network (CNN) and a Gated Recurrent Unit (GRU) to classify 10 distinct cricket
batting shots, from cover drives to pull shots, using the CricShot10 dataset. By lever-
aging automatic feature extraction through CNN and addressing temporal dependencies
via GRU, the model achieves a high classification accuracy. The study further investigates
various transfer learning models, with the VGG16–GRU architecture demonstrating supe-
rior performance, providing a significant contribution to the field of cricket video analysis.
2.1.2.1 Summary of the research item
This paper introduces a novel method for classifying 10 different cricket batting shots
using a hybrid deep-learning model. A new dataset, CricShot10, is created, containing
videos with varying lengths and lighting conditions. The authors develop a CNN–GRU
architecture and explore transfer learning models like VGG16, InceptionV3, Xception,
and DenseNet169. By freezing layers in these models, the VGG16–GRU combination
achieves the highest shot classification accuracy of 93. The study demonstrates how ef-

2.1.4 Optimized deep learning-based cricket activity
Activity recognition in sports plays a crucial role in analyzing the performance of individ-
ual players and teams across different athletic events. Each sport has distinct dynamics,
from simple activities like walking and sitting to more complex actions such as overhead
smashes in tennis. This paper introduces a new cricket activity dataset and a deep learn-
ing model that effectively handles the complex task of activity recognition in cricket. By
combining CNNs and LSTMs, the proposed model captures both spatial and temporal fea-
tures, outperforming existing methods on both the new cricket dataset and four standard
benchmarks.
2.1.4.1 Summary of the research item
The proposed model consists of multiple stages, beginning with keyframe selection and
resizing to a standard size of 100x100 pixels for efficient training. The CNN layers are
used to extract spatial features from the resized keyframes. These spatial features are
then passed through LSTM cells to capture temporal dependencies between frames. The
CNN component comprises multiple convolutional layers, each with a set of filters that
detect patterns in the input keyframes. The output of these layers is a set of high-level
feature maps that capture important details relevant to activity recognition. The LSTM
component then analyzes these feature maps to identify the sequence of actions occurring
in the video.
2.1.4.2 Critical analysis of the research item
The authors have conducted extensive experiments on the newly created cricket dataset
as well as four benchmark datasets: UCF101, HMDB51, YouTube Action, and Kinet-
ics. The experiments were carried out using Keras with TensorFlow-GPU on a system
equipped with an NVIDIA GeForce GTX 3060 GPU. The results indicate that their model
significantly outperforms other RNN-based approaches in terms of accuracy, recall, and
F1 score.
2.1.4.3 Relationship to the proposed research work
This paper also introduces a medium-scale dataset that includes five cricket activities that
will be used in our project. The authors have experimented with various vision encoders
with multiple RNN-based decoders to produce an efficient model. We are planning on
using similar methods on our dataset.

2.1.5 Towards Automated Video Game Commentary Generation us-
ing Generative AI
Video gaming is growing as an industry and it provides unique challenges which could
be solved using AI. One of the popular use cases of video games has been in the field
of Reinforcement Learning where the effectiveness of Agents is determined by how well
they perform on the chosen game. With the advent of Generative AI, the authors of
this paper have tried to incorporate deep learning models and APIs to generate dynamic
commentary of League of the Legends video game.
2.1.5.1 Summary of the research item
The authors have used pre-trained CNN models to extract frame-level features of size
1028 which is then passed to multi-layer perceptron for classifying the video clip into one
of the 20 classes. Then a simple matching dictionary is created that assigns a text to each
class which along with a pre-defined prompt is sent to the LLM, to generate commentary.
The generated is sent to a FakeYou API for generating the speech. The whole process is
very effective in generating commentary, as everytime a unique commentary is generated
for the same action. The authors have also incorporated a priority queue mechanism to
determine which events should be used to generate commentary. This ensures that there
is commentary generated every 15-20 seconds.
2.1.5.2 Critical analysis of the research item
Although the authors have utilized state-of-the-art models for classifying events, generat-
ing text, and converting the text into speech, they fail in capturing the complete scene of an
event as combat games provide very challenging scenes to be understood by the machine.
Additionally, they are working on processing one frame every second which provides
very little information as the temporal information has not been utilized. Thirdly, the gen-
erated commentary is very generic and no mechanisms have been adopted to tackle the
problem of hallucination. Therefore, despite providing useful techniques this paper has
some weaknesses that need to be catered to when adopted in our project.

2.1.6 Automatic Cricket Highlight generation using Event-Driven and
Excitement-Based features
Generating sports highlights is a labor-intensive task that traditionally requires a certain
level of expertise. With a special focus on cricket, a sport known for its complex rules and
extended playing time, the authors propose an automated framework for generating sports
highlights. Cricket matches include numerous key moments such as boundaries, wickets,
and milestones, and the proposed model incorporates both event-driven and excitement-
driven elements to capture important moments from the match. Key features like replays,
audio intensity, player celebrations, and match scenarios are used to detect and extract
crucial highlights.
2.1.6.1 Summary of the research item
The presented model operates in a structured manner, aiming to break down a cricket
match’s footage into essential segments. This is accomplished by dividing the match into
overs, further breaking them down into individual video shots. Each of these shots repre-
sents an important aspect of the match such as player actions, boundary hits, and wickets.
The process of highlight generation follows three stages: first, the model identifies various
video shots within the innings (Stage 1); second, it extracts relevant cues like player cele-
brations, scoreboard updates, and audio cues (Stage 2); and finally, it produces highlights
based on either event-driven or excitement-driven criteria (Stage 3).
2.1.6.2 Critical analysis of the research item
This model addresses a significant challenge in sports broadcasting: automating the pro-
cess of highlight generation, particularly for a complex sport like cricket. The approach
of combining both event-driven (boundaries, wickets, etc.) and excitement-driven cues
(replays, audio intensity, player celebrations) is particularly commendable, as it ensures
the highlights not only capture key moments but also the overall excitement and atmo-
sphere of the match. By using such multimodal cues, the model aligns well with human
viewing preferences, focusing on both the technical events and the emotional highs of the
game.

Proposed Approach
To generate dynamic cricket commentary we have to implement various Computer Vision,
Natural Language Processing, and Speech Generation models. The proposed method
involves extracting scoreboard information through OCR, player detection, shot classi-
fication, commentary generation through a Large Language Model, and text-to-speech
conversion for audio generation.
3.1 Pre-Processing
The pre-processing stage focuses on preparing raw video footage for subsequent analysis.
In this step, cricket clips are trimmed to isolate specific events, such as when a ball is
bowled and the batsman plays a shot. By trimming clips, we ensure that only relevant
portions of the video are passed through the processing pipeline, reducing computational
overhead and improving the accuracy of downstream models.
3.1.1 Clip Trimming
During the test phase, the first step is to trim the incoming video clips based on cricket-
specific events. We utilize heuristics and predefined rules to identify key moments in the
game, such as the start of a bowler’s run-up, the point when the ball is delivered, and
when the batsman makes contact with the ball. These segments form individual "events,"
and the video is trimmed accordingly to include only the footage around these moments.

3.1.2 Bounding Box Detection
To accurately detect the scoreboard in cricket videos, we manually annotated data to mark
where scorecards typically appear. Using this annotated data, we trained a model to detect
a bounding box around the scorecard in each frame. This helps localize key information
like scores and overs.
By identifying the scorecard’s position, we ensure that it can be reliably extracted for
further processing, such as Optical Character Recognition (OCR). This step strengthens
the model’s robustness across different camera angles and lighting conditions, allowing
the system to consistently capture scoreboard information for commentary generation.
3.2 Event Detection
The current context of the game, which we will call an event, can be extracted through the
scoreboard. The scoreboard provides various information such as runs, wickets, current
run rate, required run rate, etc. This information could be extracted using OCR models.

Conclusions and Future Work
In conclusion, the engagement and excitement of any sport are highly dependent on how
good the commentary is. The automatic generation of cricket commentary for mobile
games will help in achieving this goal. We plan to provide a pipeline to generate auto-
