1. Introduction
1.2 Scope
This project aims to develop an advanced virtual glasses try-on system that uses 3D face
and glasses models to let users try on and customize glasses. The Project is divided
into various phases, starting from face 3D model generation which focuses on detecting
and analyzing facial features to create the 3D mesh of the face. Next, in the Glasses
3D Model Generation module images and videos of glasses are used to produce a 3D
mesh of glasses. This involves image segmentation to isolate the glasses, then using
photogrammetry to generate a mesh from multiple viewpoints. After that the face and
glasses models are merged to produce the mesh of face wearing glasses. Additionally
it provides users the ability to personalize the appearance of their glasses by adjusting
color and size. To share this capability to the retailers, it will include an API to link
it with retailers system. To demo the results, there will be a demo website which will
have the API integrated. The website includes a content management system, a 3D frame
previewer and the virtual glasses try on feature.
1.3 Modules
1.3.1 Face 3D Model Generation
This module allows us to extract features from the face to generate a 3D mesh.
1. Face Detection: Scan the video to locate and identify the face by utilizing different
facial features such as eyes, nose, mouth, and jawline.
2. Feature Extraction: Identify and categorize facial features using landmark detec-
tion, and analyze texture and shape details.
3. Face Mesh Generation: Create a 3D model based on the extracted features using
predefined models to fit the detected features into a 3D shape.
4. Face size estimation: Estimate the size of the users face using depth estimation via
diameter of the iris.
1.3.2 Glasses 3D Model Generation
This module involves generating a 3D model of the glasses using images and videos.
1. Image Segmentation: Preprocess images to separate the glasses from the back-
ground, ensuring that only the glasses are visible.

2. Glasses Mesh Generation: Use images from different viewpoints and apply pho-

1.3.3 3D Model Integration
This module integrates the 3D models of the face and glasses to create a cohesive mesh.
1. Feature Extraction: Identify common features in both the glasses and face to en-
sure consistency across different types.
2. Feature Mapping: Map the features of the glasses to the features of the face to
generate a unified 3D mesh.
3. Model Integration: Connect and assimilate the glasses mesh with the face mesh
to create a joint model of the face wearing glasses.
1.3.4 Glasses Customization
This module allows users to customize the appearance of the glasses.
1. Color: Choose from a variety of colors to personalize the frame of the glasses.
2. Size: Adjust the size of the glasses to meet individual preferences.
1.3.5 API Development
Create an Application Programming Interface (API) to connect the main program with

virtual glasses try-on system.

ensure authorized access.

users.

Chapter 3
System Overview
Functionality:
OptiFit is a software solution that enables users to virtually try on eye wear (glass frames)
using their device’s camera. It uses advanced facial recognition and 3D modeling tech-
niques to superimpose 3D models of glass frames onto the user’s face in real-time. The
solution includes several key features like Frame selection, Virtual Try-on, Customiza-
tion, Glasses 3D model creation, face 3D model creation, integration with retailers, and
3D previewer.
Context:
OptiFit addresses the challenges faced by eyewear retailers and customers when it comes
to purchasing glasses online. In a traditional retail environment, customers try on physical
frames to ensure they match their face shape and style. However, the rise of online shop-
ping means customers are increasingly making eyewear purchases without being able to
try them on.
To bridge this gap, OptiFit provides a virtual try-on experience that allows customers
to make informed decisions from the comfort of their homes. This increases customer
satisfaction and reduces the chances of returns or dissatisfaction after purchase.
Design:
The project is divided into several components and layers:
1. Application Layer (Frontend)
• Handles user interactions, including the selection of glasses, camera access, and
virtual try-on features.
• Provides an intuitive user interface that allows users to view glasses from different
angles, adjust sizes, and customize frames.
29

• Manages core business logic, including facial recognition, 3D model generation,
and API integration.
• Processes data from the user’s face and the glasses 3D models and merges them in

• Handles frame customization requests and stores frame catalogs, user data, and
transaction details.
3. Data Layer
• Consists of databases that store user information, glasses frame details, 3D models,
and transaction records.
• Supports non-relational data storage to manage various formats (e.g., user prefer-

The information domain of our project is transformed into structured data such as image
