1.2 Scope
The project focuses on building AI-generated poster and video ads for furniture indus-
try. Initially, a product link is given to our application, which will automatically scrape
product descriptions from the web, preprocess them, and feed it to an LLM, which will
generate the poster advertisement description. The description will be given to an image
generator model, which will generate the poster. customer can further tweak their adver-
tisement by giving prompt to their model regarding changes in the poster. For video ads,
a description from an LLM will be created and fed into the video generator model, which
will generate a video . The video will have a human avatar with enhanced emotions and
gestures explaining the product description. There will be both a male as well as a female
avatar trained for the advertisements. Our application will be web based and will be de-
ployed on cloud (AWS) for reliability and availability. However, the app does not scrap
data from all websites but rather a selected few, the video ads created will not be dynamic;
instead, templates will be used, however, the content of the videos will be dynamic and
change with each product

User class Description
End User An individual or company trying to create advertisements for their goods or
services is the end user. Product URLs will be entered, preferences will be
chosen, and the created advertising will be downloaded or shared. Because
users might not be highly proficient in technology, the system must be easy
to use. The users, who may come from a variety of industries like retail or
e-commerce, demand a fast and easy ad generating procedure.
Admin The administrator oversees the platform’s operation, handles user manage-
ment, keeps an eye on system performance, and makes sure the AI tool for

3.1 Architectural Design
The system is structured into four main layers:
3.1.1 Presentation Layer
• This layer includes the user interface where users interact with the system.
• Users provide input (e.g., URL for scraping, image edits, video requests).
• Flow of data occurs from this to the Application Layer.
3.1.2 Application Layer
• It is the middle layer between communication between the presentation and busi-
ness logic layers.
• Receives input from the Presentation Layer (e.g., URLs, prompt editing requests).
• Passes requests to the Business Logic Layer for processing (e.g., scraping, im-
age/video generation).
• Returns results to the Presentation Layer for display.
3.1.3 Business Logic Layer
• This layer has the core functionality of the application.
20
3.1 Architectural Design
• It has all main processes such as scraping the web, LLM prompt generation, image
generation, prompt-based image editing, and video generation.
• All necessary services like Scraping Service, LLM-based Text Generation, Im-
age/Video Generators, and Prompt Editor.
• It also interacts with the Data Layer to fetch or store information (e.g., scraped data,
generated media).
3.1.4 Data Layer
• It is responsible for data storage and retrieval.
• It stores scraped content, generated prompts, images, and videos.
• It works with databases or file systems to manage project data.
• It communicates with the Business Logic Layer to provide data required for image
generation and video creation.
3.1.5 Flow of Information
• Data and commands flow downward from the Presentation Layer to the Data Layer
for processing.
• Results and responses flow upward from the Data Layer to the Presentation Layer
for user interaction.

saved. The generated image is saved after processing, and the generated image path is
updated. This allows the user to access previous projects and even make changes.
