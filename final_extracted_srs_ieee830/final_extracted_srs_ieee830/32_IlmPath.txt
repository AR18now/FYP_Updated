1. INTRODUCTION
1.1 Problem Statement
There is no shortage of Quran apps, but many are outdated, outsized, with poor user interface, invasive ads, and unethical privacy practices. There is need for a quality app that facilitates the next generation in Learning and seeking answers in a modern, digital world.

1.2 Scope
IlmPath is a platform where users can learn to read Holy Quran with tajweed, get insights about the Holy book, interact with ulama and Scholars, use Quranic chatbot, recite verses to identify Surah/Ayah, bookmark verses, take notes, and track learning performance.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- NLP: Natural Language Processing
- Q&A: Question and Answer

2. OVERALL DESCRIPTION
2.1 Product Perspective
IlmPath integrates technology with religious education to provide immersive and comprehensive learning experience for Muslims seeking to improve Quranic knowledge with limited access to qualified educators.

2.2 Product Functions (Summary)
- Quranic learning with tajweed from Qaidah
- Real-time audio feedback on pronunciation errors
- Progress tracking
- Quranic chatbot answering queries based on Quran
- Surah/Ayah finder via voice recitation
- Root meaning of words
- Note-taking on Surahs
- Bookmarking Surahs
- Ulama registration and verification
- Ulama-user chat system
- Payment-based appointments with Ulama

2.3 User Classes and Characteristics
- Student/Learner: Primary user responsible for learning Quranic lessons, finding Surahs/Ayahs, making appointments with Ulama
- Ulama/Scholar: Authorized users offering Quranic teachings, answering inquiries, providing advice
- Payment Service Provider: External system actor for financial services
- Guest: Individuals accessing features without registration (Qirat test, Quranic chatbox, Surah finder)

2.4 Operating Environment
Web application, React Native frontend, cloud database.

3. FUNCTIONAL REQUIREMENTS
3.1 Module 1 - Quranic Learning and Recitation
FR1.1: The system shall display Qaidah as initial learning resource for student.
FR1.2: The system shall automatically promote user to next lesson after successful completion.
FR1.3: The system shall allow user to browse or search for specific Surahs/Ayahs before starting recitation test.
FR1.4: The system shall prepare for voice input once Surah/Ayah selection is made.
FR1.5: The system shall allow users to initiate voice-based recitation test upon request.
FR1.6: The system shall record user recitation and begin analysis for pronunciation errors.
FR1.7: The system shall detect pronunciation errors or Tajweed mistakes.
FR1.8: The system shall mark detected mistakes for live feedback.
FR1.9: The system shall provide voice feedback based on detected errors.
FR1.10: The system shall offer voice assistance suggesting correct pronunciation.
FR1.11: The system shall allow students to practice Quranic recitation daily.
FR1.12: The system shall enable users to listen to recitation while reading along.
FR1.13: The system shall display text of selected Ayah with translation.
FR1.14: The system shall provide background information and context related to Ayah.
FR1.15: The system shall track daily listening time.
FR1.16: The system shall provide daily recitation progress tracking.
FR1.17: The system shall allow users to long-press Arabic word to search root meaning.
FR1.18: The system shall identify selected word and retrieve relevant linguistic data.
FR1.19: The system shall display root meaning and detailed definition.
FR1.20: The system shall provide context for both Arabic words and complex English terms.

3.2 Module 2 - Quranic Interaction and Search
FR2.1: The system shall verify user intent and prompt for Quranic-related queries.
FR2.2: The system shall log conversation and ensure appropriate Quranic references.
FR2.3: The system shall retrieve relevant Quranic information based on user query.
FR2.4: The system shall use database of Quranic texts and Tafsir for accurate responses.
FR2.5: The system shall provide accurate response with Quranic references.
FR2.6: The system shall offer follow-up questions or clarification options.
FR2.7: The system shall allow users to input Surah or Ayah verses by recording for search.
FR2.8: The system shall display relevant search results including matching Surahs and Ayahs.

3.3 Module 3 - User Engagement and Customization
FR3.1: The system shall allow users to add personalized notes to specific Ayahs.
FR3.2: The system shall save user notes for future reference.
FR3.3: The system shall allow users to bookmark specific Ayahs for easy access.
FR3.4: The system shall display bookmarked Ayahs in user study dashboard.

3.4 Module 4 - Ullama-Student Interaction
FR4.1: The system shall provide users option to book paid appointments with registered scholars.
FR4.2: The system shall offer chat option for connecting with scholars.
FR4.3: The system shall allow scholars to create profile and submit certification documents.
FR4.4: The system shall approve or reject registration based on background checks.
FR4.5: The system shall allow scholars to respond to user queries privately with Quranic guidance.
FR4.6: The system shall notify users when scholar has responded.
FR4.7: The system shall allow scholars to set time slots for paid consultations.
FR4.8: The system shall process payments through secure transactions before confirming appointments.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: MTBF ≥ 500 hours.
NFR2: Real-time processing for Qirat test with fallback mechanisms.

4.2 Usability
NFR3: User interface simple and intuitive for all ages.
NFR4: Clear audio feedback and easy-to-navigate text for recitations.

4.3 Performance
NFR5: Real-time feedback for voice-based features with response time ≤ 10 seconds.
NFR6: Payment transactions smooth without errors.
NFR7: Data retrieval for progress tracking and analytics ≤ 5 seconds.

4.4 Security
NFR8: Only logged-in and authorized individuals permitted access to notes and bookmarking features.
NFR9: Complete transparency for chats (no user or Ulama can see others' chats).
