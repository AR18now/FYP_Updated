1.2 Scope
The scope of our project is to develop a comprehensive road surface index calculation sys-
tem that leverages lidar sensors and crowd sensing technology for efficient and accurate
road condition monitoring. The primary functionality of the system includes detecting
road surface irregularities such as ruts, cracks, and potholes, and calculating a detailed
road surface index. This index will be used to prioritize repairs and schedule maintenance
activities. The project will incorporate lidar sensors mounted on vehicles to gather real-
time data on road surfaces, ensuring precise and continuous monitoring. In addition, we
will integrate crowd sensing, allowing vehicles equipped with sensors to contribute in col-
lections of road condition data, thereby increasing the system’s coverage and enhancing
data integrity. All collected data will be geotagged to ensure location accuracy, enabling
authorities to easily identify and access specific problem areas on a map. The system
will feature a user-friendly interface for road management authorities to visualize road
conditions, generate reports, and manage maintenance priorities. The project will focus
on developing algorithms for data processing, road condition classification, and integra-
tion of data collected through crowd-sensing to create a robust solution. The scope also
includes testing and validating the system to ensure its reliability, scalability, and ease of
deployment for large-scale road monitoring.

2.1 Functional Requirements
This section describes the functional requirements of the system.
2.1.1 Hardware setup
Following are the functional requirements for the hardware setup module:
1. Sensor Mounting: The system must mount lidar sensors securely on the vehicle to
ensure stable data collection.
2. Sensor Calibration: The system must calibrate the lidar sensors before data col-
lection to account for terrain variations.
3. Weatherproofing: The system must ensure that all hardware components are weath-
erproof and capable of functioning in varying environmental conditions.
2.1.2 Data Collection
Following are the functional requirements for the data collection module:
1. Real-Time Data Capture : The system must collect real-time road surface data
using lidar sensors.
2. Geotagging: The system must geotag all collected data to ensure precise location
tracking of road conditions.

2.2 Non-Functional Requirements
Following are the non-functional requirements for the system:
2.2.1 Reliability
The system must maintain a high level of reliability, with a target Mean Time Between
Failures (MTBF) of 500 hours. A failure is defined as any instance where the system is
unable to process or collect road surface data as intended. To protect against failures, the
system will incorporate redundancy measures, such as backup sensors and data storage.
An error detection strategy will involve real-time monitoring of system performance, and
the correction strategy will include automated recovery procedures to restore functionality
within 10 minutes of a failure.
2.2.2 Usability
The system shall be designed for ease of use, allowing users to complete primary tasks,
such as data collection and analysis, with minimal training. For example, users should be
able to retrieve previous road condition reports with a single interaction. Example:
USE-1: the interface must provide clear instructions and feedback during user interactions
to enhance the overall user experience and minimize errors.
7
2. Project Requirements
2.2.3 Performance
The system must prioritize accuracy in data processing and analysis over speed. The ma-
chine learning model should be designed to achieve a minimum accuracy rate of 90%
in road surface index calculations and anomaly detection. While the time taken for pro-
cessing will be monitored, it is acceptable for the model to require up to 30 seconds
for thorough analysis, ensuring that the results are reliable and precise. The system will
provide periodic updates on performance metrics to ensure that accuracy standards are
consistently met.
2.2.4 Security
The system must ensure the protection of collected data against unauthorized access and
breaches. Security requirements include limiting access to authorized personnel only,
ensuring data encryption during transmission, and maintaining an audit trail for all data
access and modifications. The system must also be resilient against common cybersecu-
rity threats, such as SQL injection and denial-of-service attacks.
2.2.5 Scalability
The system must be designed to handle increasing amounts of data without a decline
in performance. It should support the addition of more lidar sensors and crowd-sensing
devices seamlessly, accommodating up to 100 vehicles collecting data simultaneously.
2.2.6 Maintainability
The system must be designed for easy maintenance and updates, with a goal of reducing
downtime for software updates to less than 1 hour per month. The architecture should
support modular upgrades to specific components without affecting overall system func-
tionality.
