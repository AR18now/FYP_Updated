1.1 Problem Statement
In traditional educational settings, teachers often lack the flexibility to tailor learning ma-
terials to individual student needs. They are constrained by standardized resources that
do not account for the diverse learning paces, skill levels, or familiarity students have
with the content. As a result, students who struggle with certain topics risk falling be-
hind, while those who excel may not be sufficiently challenged. These limitations hinder
personalized learning and engagement, creating a one-size-fits-all approach that fails to
address unique student needs. ProdigyPal is being developed to solve this problem by
providing a platform that generates customized, dynamic worksheets. This software will
also allow teachers to offer personalized learning experiences, track student performance,
and adjust to individual strengths and weaknesses. By creating an adaptive learning envi-
ronment ProdigyPal aims to uplift struggling students while challenging high achievers,
in turn fostering a more personalized and effective learning process.
1.2 Scope
The scope of ProdigyPal includes the development of a web-based application designed
to generate personalized, dynamic worksheets for students in grades 1 to 6, focusing on
mathematics. The system will use machine learning models to create adaptive questions
tailored to individual student performance. It will track student progress and offer teach-
ers detailed performance analytics, helping them identify strengths and areas for improve-
ment. The application will feature gamification elements like leaderboards and achieve-
ments to keep students engaged and motivated. Teachers and students will have different
access roles, with features customized to their needs. ProdigyPal aims to provide a scal-
able and user-friendly platform that enhances the learning experience by offering tailored
educational content and real-time assistance. The project covers the full range of work-
sheet generation, from data processing to final delivery, within a reliable and adaptive
system.

1.3 Modules
1.3.1 Module 1: Dataset and Data Pipeline
This module focuses on building and maintaining reliable datasets that the Machine Learn-
ing models can use for generating content. It also handles the flow of data from the
database through the different models and into the web application, ensuring seamless
data processing.
1. Dataset Collection and Preparation: Collects and processes raw data needed for
training the models.
2. Data Pipeline Management: Manages the flow of data from the database to the
ML models and ensures its availability for real-time processing.
1.3.2 Module 2: Machine Learning Models
This module is dedicated to the design, development, and training of the machine learning
models that power ProdigyPal’s personalized learning experience.
1. Question Generation Model: Dynamically generates adaptive questions based on
students’ past performance and current difficulty levels.
2. Image Generation Model: Produces relevant visual aids, such as diagrams or il-
lustrations, to accompany the generated questions.
3. Answer Explanation Model: Provides step-by-step solutions to the generated
questions, helping students understand how to arrive at the correct answers.
1.3.3 Module 3: Worksheet Assembly and Integration
This module oversees the integration of the different ML-generated components, creat-
ing a cohesive and dynamic worksheet that can be accessed by users through the web
application.
1. Content Organization: Organizes the questions, images, and answers generated
by the models, labeling them as worksheet assets.
2. Worksheet Compilation: Combines all assets (questions, images, solutions) into
a final, structured worksheet ready for student interaction.

2.3 Non-Functional Requirements
This section specifies nonfunctional requirements. These quality requirements should be
specific, quantitative, and verifiable. The following are some examples of documenting
guidelines.
2.3.1 Reliability
REL-1: The system shall achieve an MTBF of at least 1,000 operational hours under normal
usage conditions. A failure is defined as any incident where a user cannot gener-
ate worksheets, track student performance, or access essential features for over 30
minutes.
REL-2: In case of a failure, no more than 5% of user-generated data shall be lost or cor-
rupted. The system shall maintain transactional integrity, ensuring that either all or
none of the data associated with an operation is saved.
REL-3: In the event of a failure, the system shall recover and resume normal operations
within 10 minutes, including restoring access to all core features.
2.3.2 Usability
USE-1: The system shall ensure that new users can learn to navigate the core features within
15 minutes of their first interaction, without external assistance.
USE-2: At least 90% of tasks shall be completed by users without needing more than 5
clicks or interactions per task.
USE-3: The system should provide contextual tooltips and inline validations to prevent
users from making common errors
USE-4: The system shall provide clear and actionable error messages that include descrip-
tions of what went wrong and how to fix it. Users should be able to recover from
common errors (e.g. incorrect worksheet topic selection) within 1 minute.
USE-5: Teachers shall be able to generate a new personalized worksheet for a student or
class in less than 2 minutes, with minimal input required beyond topic selection
and difficulty adjustment.

System Overview
ProdigyPal is a personalized learning solution enhanced by AI, catering to both teachers
and students to enrich the educational journey. The objective is to tackle the deficien-
cies of conventional learning methods, which frequently lack flexibility and adaptability,
resulting in inconsistent student results. The main emphasis of the platform is on Mathe-
matics for students up to Grade 6, providing a flexible and interactive space for learning
and evaluation
ProdigyPal offers two main features for two primary users: For Teachers: It pro-
vides a smart chatbot with AI capabilities to help create interactive worksheets. These
worksheets are customized to cater to the unique requirements and pace of learning of
each student, enabling teachers to offer personalized learning resources. It consists of
resources for monitoring student achievements, providing beneficial information on each
student’s advancement, and aiding teachers in filling learning deficiencies.
For Students: It provides an interactive learning experience that enhances participation
and drive. By including leaderboards, point systems, and prompt feedback, students are
motivated to engage in their learning process. The system offers adaptive learning path-
ways, giving students challenges tailored to their skill levels, as well as providing hints
and feedback to support their progress.
ProdigyPal utilizes advanced technologies like machine learning models, dynamic content
generation, and real-time data analytics to create a personalized and interactive learning
environment for students, giving teachers more flexibility and control over educational
resources.

3. System Overview
3.1 Architectural Design
ProdigyPal’s architectural design adopts a modular strategy, dividing the system’s func-
tionality into separate but interconnected components. This design improves scalability
and maintainability, while also enabling separate development, testing, and deployment
of each module.
The system is divided into four main modules, with each one handling a particular aspect
of the platform’s functioning:
3.1.1 Dataset and Data Pipeline Module
This module is in charge of collecting, standardizing, and organizing the data needed to
train machine learning models. It establishes a data pipeline to guarantee smooth data
transition from capturing to analyzing.
The module serves as a base, as the information gathered here is used in the ML train-
ing module to guarantee that the worksheets created are precise and customized for the
students’ requirements.
3.1.2 Machine Learning (ML) Models Training Module
This module is responsible for the training and adjusting of machine learning models. It
is in charge of creating dynamic worksheets that range in difficulty levels according to
students’ skills.
Essential components like engineering and the Chatbot backend take inputs from teachers
and convert them into customized, AI-generated materials.
The result of this module seamlessly connects with the front-end to create worksheets in
real-time.
3.1.3 Client Web Application Module
This module serves as the interface that teachers and students engage with. Constructed
using ReactJS, which enables an interactive and adaptable user interface.
Teachers have the option to utilize the Custom Worksheet Generation Prompter for creat-
ing materials, as students engage with the gamified UI that features progress tracking and
engagement resources.
This component interacts with the back-end APIs, obtaining information produced by the
