1. Introduction
1.1 Problem Statement
Creating real human facial images from inputs, like sketches, partial photos, or text, is
a complex and unresolved problem. Despite advancements in artificial intelligence, ex-
isting solutions (reference 1) fall short in generating high quality real-images that truly
transform to human face. These limitations are particularly problematic for law enforce-
ment, where the accurate reconstruction of a face from a sketch can be critical to identify
the suspect. This project aims to address this gap by developing an AI-driven system that
not only generates realistic human faces from but also offers powerful editing capabilities
through a user-friendly application for agencies to modify image according to their needs.
1.2 Scope
Our proposed project aims to develop a AI-driven system capable to convert both hand-
drawn sketches and textual prompts into realistic human facial images. This makes are
system valuable for forensic investigation to catch suspect where CCTV footages are not
available they really on sketches to identify suspect. Moreover artist use to convert their
sketches to realistic looking facial images.
One of the key features of our system is manipulating and editing the generated im-
ages based on user input. Users can modify specific facial features, such as face shape,
hairstyle, hair color, eye color, and more, simply by providing prompts. This flexibility
allows users to generate images according to their specific needs and preferences. This
helps in forensic investigations.
Moreover, our system supports real-time conversion of sketches into images, allowing
users to see the image being created dynamically as they draw. This real-time interaction
enhances usability, and helps adjusting during the image creation process.
The boundaries of this project are focused on facial image generation and manipulation.
While it helps in creating and editing facial features, it is not extended to full-body genera-
tion or other types of images. Moreover, the system will be deployed as a web application,
ensuring accessibility to a wide range of users.

3.1 System Workflow
Following is the workflow of our system :
• The user is asked to generate a facial image of a sketch or a textual description.
• Firstly, if the user selects to generate the facial image through a sketch, then he can
either upload a sketch or draw by himself.
• The sketch is then denoised and passed to the Sketch-To-Face model which then
generates an upsampled image.
• Secondly, if the user enters a prompt, the prompt is converted into tokens and en-
coded.
• The Text-To-Face Model then generates the facial image.
• Lastly, the user can manipulate the facial attributes of the generated face.
3.2 Architectural Design
The system is based on a Client-Server architecture in which the user interacts with system
using a web interface.
11
3. System Overview
3.2.1 Client-Server Architecture
The communication between the web interface and the backend components is done
through an API Gateway. The backend consists of three modules which includes Sketch
to Face Module, Text to Face Module and Face Manipulation Module. The input is passed
through these individual modules. Each module performs a specific task. The output of
these modules is stored in a Database.
3.2.2 Client Side
The client is responsible for the communication between the system and the user. It
consists of the following two modules :
• Web Portal : The user interacts with the system using a Web Portal. This mod-
ule is responsible for all the front-end related tasks. It consists of the following
functionalities :
– Allow the user to draw or upload Hand-drawn sketches.
– Allow the user to enter textual prompts for facial image generation.
– Allow the user to manipulate facial attributes of a generated image.
– Display the generated output to the user.
• API Gateway : This module is used to make communication between the web por-
tal and the backend system possible using API Calls. It is responsible for handling
requests and responses. It is used to handle API Clients in an efficient manner.
3.2.3 Server Side
The server is responsible for handling all the backend components such as the Database.
It consists of all the services that are required by the system. Following modules are
involed in this:
• Sketch to Face Module : This module is responsible for the generation of facial
images through sketches. The module takes a sketch as an input through API and
uses it to generate a similar facial image.
• Text to Face Module : This module is responsible for the generation of facial im-
ages through textual prompts given by the user. The module takes a textual prompt
as an input and then generates a facial image based on the description provided.

3.4.2 Data Structures
The primary data structures in Sketch2Face include:
3.4.2.1 Sketch Data
• Input: Hand-drawn sketches uploaded or drawn on GUI by the user.
• Structure: Stored as image files (e.g., PNG or JPEG) in a temporary storage system
before processing.
20
3.4 Data Design
• Processing: The sketches are passed through an AI model, which converts them
into a facial image.
• Storage: The processed sketch is saved temporarily for manipulation or can be
saved directly in database if no manipulation is required.
3.4.2.2 Textual Prompt Data
• Input: Textual descriptions provided by the user (e.g., "male, short hair, blue
eyes").
• Structure: The data/text is passed through AI model for image generation.
• Processing: Each attribute (hair color, facial features, etc.) is mapped to specific
facial characteristics during the image generation process.
• Storage: Once converted to an image, it can be further manipulated or stored di-
rectly into database.
3.4.2.3 Generated Image Data
• Output: The AI model generates facial images from the user’s sketches or textual
prompts.
• Structure: Stored as image files (e.g., PNG, JPEG).
• Storage: Image can be further manipulated or stored directly into database.
3.4.2.4 Manipulation Data
• Input: User requests to manipulate specific features of the generated image (e.g.,
change hair color or adjust facial shape).
• Processing: The AI model updates the image based on the modifications, and the
new image is stored in the system.
• Output: Modified image.
• Storage: Image can be further manipulated or stored directly into database if user
is satisfied with modification.
21
3. System Overview
3.4.2.5 User Data
• Input: User profiles include login information, history of inputs (sketches, text
prompts), and saved images.
• Structure: Stored in a relational database with tables for user details, login creden-
tials, and historical actions.
3.4.3 Databases and Data Storage
The system utilizes a combination of file storage and database management systems to
handle large amounts of image data and user information.
3.4.3.1 Database (Relational Database)
• User Profiles: A relational database that stores user information, including login
details, history of actions, and saved images.
• Sketch/Prompt History: Tracks sketches and text prompts submitted by the user,
mapping each submission to the respective generated image.
• Manipulation History : Stores details of the changes made to the images (e.g.,
color, facial structure modifications).
3.4.4 Organization and Processing
3.4.4.1 Data Storage
• All generated and manipulated images are stored with unique identifiers that map
them to the original input (sketch or text prompt).
3.4.4.2 Data Processing
• Sketches and textual prompts are processed in real-time to generate facial images.
• Manipulation requests are processed by applying changes to the image and storing
the new version in the database, without overwriting the original.

• User can only access their own generated and manipulated images through a secure
