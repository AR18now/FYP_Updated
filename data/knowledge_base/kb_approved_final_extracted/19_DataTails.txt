1. INTRODUCTION
1.1 Problem Statement
Reddit hosts extensive user-generated content across wide topics. Deriving meaningful insights poses challenges due to unstructured format. Conventional data analysis tools aren't well-suited for complex, context-dependent Reddit discussions where topics evolve rapidly and user sentiments are complex and multifaceted.

1.2 Scope
DataTails transforms unstructured Reddit data into structured and actionable insights using Large Language Models (LLMs), Knowledge Graphs, and Visualization Recommendation Engine. Features include web scraping, Kafka streaming, MongoDB storage, knowledge graph construction, topic modeling, RAG-based LLM for contextual responses, and automatic visualization recommendations.

1.3 Definitions, Acronyms, Abbreviations
- LLM: Large Language Model
- RAG: Retrieval-Augmented Generation
- KG: Knowledge Graph
- NLP: Natural Language Processing

2. OVERALL DESCRIPTION
2.1 Product Perspective
DataTails processes Reddit's unstructured data through multiple stages, transforming it into structured and actionable insights, providing sophisticated query functionalities and user-friendly visual representations.

2.2 Product Functions (Summary)
- Real-time Reddit data extraction
- Kafka streaming for data ingestion
- MongoDB storage for unstructured data
- Knowledge graph construction
- Topic modeling for theme extraction
- RAG-based LLM for contextual responses
- Automatic visualization recommendation
- Sentiment analysis
- Trend identification

2.3 User Classes and Characteristics
- Developers: Configure system integrations, manage data extraction, tuning
- End Users: Access processed insights and reports
- Premium Users: Receive visualization recommendations, detailed knowledge graphs, personalized topic analysis
- Non-premium Users: Basic features (query submission, topic selection)
- System User: Overall maintenance, monitoring, configuration
- Data Source Component: Reddit for scraping unstructured data
- Kafka Integration: Streaming component for real-time data

2.4 Operating Environment
Web application, React.js frontend, Flask backend, MongoDB, Kafka.

3. FUNCTIONAL REQUIREMENTS
3.1 Data Scraping and Ingestion
FR1.1: The system shall scrape Reddit using PRAW, Selenium, or Beautiful Soup to extract posts, comments, user data.
FR1.2: The system shall use cron jobs to schedule scraping tasks at predefined intervals.
FR1.3: Kafka shall be used for real-time data ingestion from scraping module.
FR1.4: MongoDB shall store data with differentiation between premium and non-premium users.

3.2 Data Transformation and Knowledge Graph Construction
FR2.1: The system shall convert raw unstructured Reddit data to structured data using topic modeling and entity extraction.
FR2.2: The system shall search for and integrate suitable ontologies (SIOC, DCMI, FOAF) for KG structuring.
FR2.3: The system shall perform periodic Knowledge Graph reconstruction to update structure with new data.

3.3 Advanced Querying and Insights Generation
FR3.1: The system shall integrate RAG-based LLMs to respond to user queries extracting relevant knowledge from KG.
FR3.2: The system shall accept user queries via natural language interface and extract relevant information.
FR3.3: The system shall recommend appropriate visualizations based on user input and context.

3.4 System Integration and Testing
FR4.1: The system shall combine various APIs (data scraping, topic modeling, KG-based querying).
FR4.2: The system shall connect RAG-based LLMs with KG to handle user queries.
FR4.3: The system shall include visualization recommendation engine.
FR4.4: Premium users shall access expanded query limits, comprehensive visualizations, personalized topic modeling insights.
FR4.5: Non-premium users shall have basic functionality (limited queries, fewer visualizations, restricted KG access).
FR4.6: Continuous integration pipeline shall test updates and monitor performance.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: MTBF ≥ 99.95% for premium users.
NFR2: System shall detect scraping/ingestion failures and retry within 5 minutes.

4.2 Usability
NFR3: Users shall perform text-based search and receive insights with ≤ 3 interactions.
NFR4: Interface shall present recommended visualizations in drag-and-drop format.

4.3 Performance
NFR5: 95% of user queries generated within 3 seconds.
NFR6: Visualization loading ≤ 2 seconds for non-premium, ≤ 1 second for premium.
NFR7: Kafka shall ingest and process incoming data streams within 2 seconds.

4.4 Security
NFR8: Role-based access control (RBAC) and OAuth2.0 for authentication.
NFR9: HTTPS encryption to protect user data during transmission.
NFR10: All user interactions logged for auditing; suspicious activity reported in real-time.
