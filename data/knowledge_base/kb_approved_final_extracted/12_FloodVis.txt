Project: FloodVis - Deep Learning with Aerial Imagery for Flood Assessment

1. INTRODUCTION
1.1 Problem Statement
The 2022 floods in Pakistan left 1,739 dead and over 8 million displaced. Traditional ground-based and satellite remote sensing equipment faces challenges: optical satellite images hindered by cloud cover, and survivor identification is time-consuming and labor-intensive.

1.2 Scope
The system integrates deep learning with SAR satellite and UAV imagery for flood detection, mapping affected areas, and detecting humans and animals in flood zones. Features include semantic segmentation for flood exposure zones, custom object detection for affected individuals/livestock/damaged buildings, approximate coordinates for rescue teams, and a web dashboard with analytics.

1.3 Definitions, Acronyms, Abbreviations
- SAR: Synthetic Aperture Radar
- UAV: Unmanned Aerial Vehicle
- API: Application Programming Interface

2. OVERALL DESCRIPTION
2.1 Product Perspective
FloodVis aims to enhance flood disaster response by providing real-time data and improving detection accuracy and speed for flood detection, survivor identification, and damage assessment.

2.2 Product Functions (Summary)
- Flood detection and mapping using semantic segmentation
- Human and animal detection in flood zones
- Counting of critically affected individuals and cattle
- Submerged infrastructure detection
- Dashboard with flood area size and severity
- Pre- and post-flood image comparison
- Report export functionality

2.3 User Classes and Characteristics
- Emergency Responders: Acquire counts and locations of stranded humans/animals
- Disaster Managers: Make key decisions regarding resource allocation
- System Administrators: Maintain application, ensure security and data integrity

2.4 Operating Environment
Web application with backend data storage, model deployment, and cloud scalability.

3. FUNCTIONAL REQUIREMENTS
FR1: The system shall classify regions as flooded or non-flooded.
FR2: The system shall detect and count critically affected and non-critically affected humans.
FR3: The system shall detect and count critically affected and non-critically affected cattle.
FR4: The system shall detect number of submerged infrastructure.
FR5: The system shall provide quantitative data via dashboard (flood area size, severity).
FR6: The system shall allow users to export reports with flood analysis and detection summaries.
FR7: The system shall allow comparison of pre- and post-flood imagery.

4. NON-FUNCTIONAL REQUIREMENTS
NFR1: The system shall process images with varying levels of noise.
NFR2: The system shall support medium-resolution satellite and UAV images.
NFR3: The system shall process images captured from different angles.
NFR4: The system shall provide accurate flood segmentation and detection of cattle, humans, and infrastructure.
NFR5: The system shall process large amounts of data quickly for emergency insights.
NFR6: Interface shall be user-friendly for decision-makers and rescue teams.
NFR7: The system shall ensure confidentiality and integrity of satellite/UAV data and results.
