This research aims to bridge that gap by developing a \textbf{Real-Time Road Safety

for relevant dangers and update the safety index of different routes in real-time. By provid-
ing these functions, the system seeks to increase user comfort and safety while commuting
by enabling users to take preventative action and develop well-informed opinions.
1.2 Scope
The Real-Time Road Safety Alert System offers current information on road hazards re-
sulting from natural catastrophes to improve travel safety. The main job of the system is to
look for user-generated material about road conditions on different news sites and social
media platforms.The system will employ natural language processing (NLP) to look for
terms in postings that could be danger indicators. This will enable the system to assess
these risks using a predetermined range that spans from safe to critical. Because the safety
indices for different routes will be updated in real-time, users will always have access to
the most recent information on their travel routes.
Users will be able to view their routes and the associated safety index clearly using a
map-based interface. The technology will provide users with alerts when major threats
are detected, in addition to real-time updates, enabling them to plan their travel routes with
the necessary information.The program will use Google Maps for areas with dependable
internet access; offline users will get the most recent updates that are stored on their
devices. The program will provide users with the last safe paths in the event that internet
connectivity is not accessible, even if its main focus will be on external data sources.
Another part of the project’s scope is the integration of an intuitive interface that al-
lows passengers to customise their alerts and notifications to their preferences.The project
scope does not presently include full danger checking, but the system will broadcast high-
severity alarms to assure user safety. The objective is to offer a robust and workable
solution that improves passenger experience while lowering the risks associated with un-
foreseen road hazards. This comprehensive plan will let clients travel with confidence and
steer clear of dangerous situations.

2. Project Requirements
This chapter describes the functional and non-functional requirements of the project.
2.1 Functional Requirements
2.1.1 Feature 1: Safe Route Planning
Description: The first feature of SafeRouteExplorer will allow users to enter their
destination and receive the safest route based on real-time social media data.
(a) The system shall allow the user to enter their starting location and destination.
(b) The system shall show the safest route based on the data received and process-
ing done on it.
(c) The system shall dynamically update the map if a road hazard occurs along
the route.
(d) The system shall display the estimated time to reach the destination.
2.1.2 Feature 2: Real-time Social Media Monitoring for Poten-
tial Hazards
Description: This feature will monitor social media and news sources for potential
road hazards like accidents, crimes, natural disasters, etc.
(a) The system shall continuously fetch tweets, Facebook, and Instagram posts in
real-time based on specific keywords.
(b) The system shall process and filter the incoming data to extract relevant infor-
mation.
(c) The system shall identify the severity of the hazard detected.
(d) The system shall identify the location where the potential road hazard oc-
curred.
(e) The system shall classify the severity of the hazard and update the map to
show the safest route for the user’s destination.

3. System Overview
3.1 System Overview
The "Real-Time Safe Path Finder" system is designed to increase travel safety by
distributing real-time hazard notifications and route safety information. It uses data
collected from social media platforms such as Twitter and Instagram, and news
channels to inspect hazards; topics of potential blockage or natural disaster- in road.
Using NLP, the system filters and processes the relevant posts to create an Safety
index for various routes. Those services can be approached via web and mobile
applications like alert, map interactive, and customized safety preferences.
3.2 Architectural Design
3.2.1 Box and Line Diagram
The initial system architecture can be visualized using a simple box and line dia-
gram. This diagram shows the different modules of the system and how they interact
with each other.
The modules include:
• User Interface: Allows users to access the system in order to view routes and
also receive alerts as well as safety preference settings.
• Data Collection: Decide the external APIs like social media, news etc. in
order to collect the information in real time.
• NLP Processing: Filters relevant posts and data to extract hazard-related in-
formation.
• Safety Index Calculation: Dynamically calculates the safety index for vari-
ous routes based on the processed data.
• Notification System: Sends real-time alerts to users about hazards along their
routes.
• Database: Maintaining list of user profiles, route information, and hazard
alerts.

• Feedback State: Users provide feedback on route safety.
3.4 Data Design
The system utilizes a real-time database to store, process, and retrieve key data en-
tities in the system. The following are major data structures currently being utilized
below:
• User Profiles: Maintains user-specific information, preferences, location, and
their preferred custom alert settings.
• Routes: Stores route information that includes start and end points, safety
index, and other hazard information.
• Alerts: Stores real-time hazard alerts along with time-stamped, location, and
hazard type.
• Social Media Posts : Stores social media posts that contain relevant hazard
data.
• Feedback: Store feedback from users on route safety and hazard accuracy.
All data is processed using Firebase NoSQL database to ensure real-time synchro-
nization and availability for users.
