1.2.3 Creating Ensemble Model
The improvement of object detection will be achieved by using an ensemble approach that
integrates YOLOv8 and Faster R-CNN models. Furthermore it is useful in complex data
such as the IP102 dataset with several pest classes and potential class imbalance issues.
The following explains why ensemble modeling is useful in this situation:
1.2.3.1 Improved Generalizations
Different models have different strengths and weaknesses. While Faster R-CNN focuses
on accuracy, YOLOv8 is said to be highly efficient in terms of speed and performance in
real-time applications. Combining these models allows us to take use of their features,
which improves overall performance on a heterogeneous dataset such as IP102.
1.2.3.2 Handling Class Imbalance
The IP102 dataset shows imbalances both intra- and interclass, or between and within a
class, indicating that certain pest classes are more common than others. By combining
predictions from several models, ensemble approaches might help lessen these problems
by balancing the biases of individual models toward particular classes. This can improve
the model’s capacity to identify less common classes that a single model might miss.

1.2.3.3 Enhanced Robustness
In general, ensemble models are less sensitive to the noise present in a dataset and vari-
ations in such a set. This is important especially for pest recognition because distinctly
different backgrounds, environmental conditions and image contrast and resolution alter
the appearance of a pest.
1.2.3.4 Reduction of Error
The ensemble model can reduce the errors created by different models by combining
the outputs of several models in a single one. In total, this can lead to better means of
prediction, especially in such a complex case as pest identification, whereas false positives
and false negatives can have a very negative outcome.
The IP102 dataset presents a number of issues, mostly related to inter-class and intra-class
imbalances. These challenges can in fact be ably managed if YOLOv8 is implemented
in conjunction with Faster R-CNN in an ensemble manner. These models’ capabilities of
learning can be quite useful in determining and classifying numerous pests accurately.

1.3.2 IP-102 Dataset Preprocessing
Roboflow would be used for the preprocessing of IP-102 dataset. Before the models are
trained, some preprocessing steps are required to get the best performance. The major
preprocessing techniques are:
1.3.2.1 Data Annotation
Images from the dataset would be annotated by manually drawing bounding boxes around
objects required for detection and classifying them into classes. Corresponding testing,
training and validation data would be used for model training.
1.3.2.2 Data Augmentation
Rotation: Facilitates object detection by helping models learn from different angles.Flip-
ping: Facilitates model’s robustness against variations in object detection by flipping
images vertically and horizontally. Scaling and resizing: Facilitates model in better
detecting objects by randomly resizing images Color Jitter: Facilitates models to be-
come invariant to lighting conditions by adjusting brightness, saturation and contrast of
images.
1.3.2.3 Data balancing:
Since the dataset has a class imbalance, oversampling and undersampling will be consid-
ered to ensure the models are trained on a balanced dataset.
We will experiment with different combinations of these preprocessing steps to increase
the quality of the training data and to find models that perform the best.

System Overview
PestInsight is a web-based interface dedicated to the recognition of pests in rice, wheat
and corn crops based on real-time deep learning models (YOLOv8, Faster R-CNN). It
generates reports, estimates severity and provides advice on pest management. The sys-
tem consists of a React frontend, Django backend and MySQL for database management.
PestInsight tackles problems such as class imbalance, and similarities between pests by
firstly preparing the dataset and then enhancing image quality for better recognition.
