1.2 Scope
DataTails intends to create a powerful analytics platform developed exclusively for large-
scale, unstructured Reddit datasets. The platform enables users to make natural language
inquiries, relying on integrated Large Language Models (LLMs) to give contextually ac-
curate insights that are reinforced by Knowledge Graphs. One of its most notable features
is the Automatic Visualization Recommendation Engine, which recommends ideal ren-
derings based on the examined data. The platform includes crucial functionalities for data
analysts, organizations, and researchers, such as sentiment analysis, trend identification,
and subject modeling.
DataTails will be accessible via a web application, with premium users benefiting from
advanced features. Premium users will not only access enhanced capabilities but also gain
in-time data insights through live scraping of Reddit at designated intervals. Initially, the
platform’s scope will focus on Reddit data, with the potential for future expansion to other
data sources.

1. Reddit Scraping - The system shall scrape Reddit using PRAW, Selenium, or

2. Logic for Choosing Subreddits - To effectively choose subreddits to collect data
from, we selected subreddits with high context, high volume, and high traffic. We
gathered a list of subreddits from various topics, including: Pakistan, Technology,
News, Politics, Travel, Relationships, etc.
Once these subreddits were listed, we employed a method to determine which ones
to choose for data collection. Each subreddit has a count of subscribers (followers)
and each post has a number of upvotes. To evaluate suitability, we calculated the
average number of upvotes from 2000 posts in each subreddit, along with the max-
imum and minimum number of upvotes. Additionally, we determined subreddits
with maximum and minimum subscribers.
To normalize these values, min-max normalization was applied to both upvotes and
subscriber counts for each subreddit. With the normalized data, we calculated a

3.1 Architectural Design
DataTails’ architectural design takes a modular approach, integrating multiple technolo-
gies to deliver full data analysis and visualization capabilities. The architecture is or-
ganized into distinct modules, each responsible for a certain task, enabling the system’s
scalability, maintainability, and extensibility. The following is a high-level summary of
how the system’s modules interact to provide the desired functionality.
The architecture of DataTails includes the following major components:
• Frontend (React.js): The frontend is built with React.js and serves as the primary
interface for users. It enables both premium and non-paid users to submit searches,
analyze visualizations, and engage with data. The frontend makes use of visual-
ization libraries like Plotly.js, d3.js, and Chart.js to create interactive and visually
appealing data representations.
• Backend (Flask): The backend uses Flask to handle user queries, manage data
flow between the frontend and other system components, and link with other data
sources like Reddit via the Reddit API. It regulates communication between the
system’s many modules.
• Web Scrapper: The web scraper is in charge of collecting data from Reddit via the
Reddit API. It collects and stores unstructured data in MongoDB for later process-
ing.
• MongoDB: MongoDB serves as a fundamental database for storing the scraped
data. It manages unstructured data in a dependable and scalable manner, allowing
other modules to access it more easily.
18
3.1 Architectural Design
• Topic Modelling Module: This module extracts key themes from MongoDB data
and organizes the information sensibly. The extracted topics are then utilized to
construct a Knowledge Graph and provide inputs to the RAG-based LLM.
• Knowledge Graph and RAG-based LLM: The Knowledge Graph visualizes the
relationships between various subjects and entities retrieved from the data. The
RAG-based LLM gives context-aware replies to user questions based on processed
data, boosting information quality.
• Visual Recommendation System: This module recommends appropriate visual-
izations to users based on insights gained from the knowledge graph and the RAG-
based LLM, so improving the overall user experience and increasing access to data
insights.
• Kafka Stream and Cron Job Scheduler : The Kafka Stream manages in time
data streaming, ensuring users have access to the most recent information. The
Cron Job Scheduler manages data scraping processes and keeps the data up to
date without operator intervention.
3.1.1 Relationships Between Modules
The modules are interconnected to create an efficient data analysis pipeline:
• Frontend (React.js) interacts with the Backend (Flask) to handle user requests
and display visualizations using Plotly.js, d3.js, and Chart.js.
• Backend uses the Reddit API to direct the Web Scrapper to collect data, which
is stored in MongoDB.
• Topic Modelling Module processes the stored data to derive topics and creates a
Knowledge Graph and interacts with theRAG-based LLMfor advanced analysis.
• Visual Recommendation System uses the generated insights to recommend ap-
propriate visualizations.
• Kafka Stream ensures in-time data access, while the Cron Job Scheduler auto-
mates the data scraping process.
