SRS Knowledge Base Metrics: Methodology

This document explains how KB TXT files are evaluated in the project using rag/evaluate_srs_kb.py.

Scoring Scale

All metric values are normalized between 0 and 1.

  • 1.0 = best quality
  • 0.0 = poor quality

Overall Score Formula (Weighted)

overall_score = 0.12*completeness + 0.08*fr_coverage + 0.08*nfr_coverage + 0.07*clarity_readability + 0.06*ambiguity_quality + 0.05*extraction_cleanliness + 0.05*content_adequacy + 0.05*duplication_quality + 0.06*structure_compliance + 0.05*numeric_specificity + 0.03*lexical_richness + 0.05*weak_phrase + 0.05*optionality + 0.05*atomicity + 0.06*consistency + 0.08*verifiability + 0.05*requirement_uniqueness + 0.04*requirement_id_coverage + 0.04*readability + 0.03*requirement_density

Metric Definitions and Calculations

Metric What It Checks How It Is Calculated (Surface Formula)
Completeness Metric Whether core IEEE SRS sections exist. hits / total_expected_sections for Introduction, Overall Description, Specific Requirements markers.
Structure Compliance Metric Heading quality and expected section structure. heading_hits / expected_heading_count using regex checks for key headings.
Clarity Metric Vagueness level in text (ambiguity-based). 1 - (ambiguous_word_count / total_words).
Weak Phrase Metric Presence of weak/vague phrases (fast, efficient, user-friendly, etc.). 1 - (weak_phrase_hits / total_words).
Optionality Metric How much requirements use optional language (may/might/could/optional). 1 - (optional_requirement_lines / requirement_lines).
Atomicity Metric If one requirement contains too many actions at once. 1 - (multi_action_requirements / total_requirements).
Consistency Metric Conflicting terms in same document (e.g., always vs never). 1 - (conflict_pairs_found / total_conflict_pairs_checked).
Verifiability Metric Whether requirements are measurable/testable. measurable_requirements / total_requirements (numbers, thresholds, %, etc.).
Requirement Uniqueness Metric Duplicate/near-duplicate requirements. 1 - (duplicate_pairs / total_requirement_pairs) using text similarity.
Requirement ID Coverage Metric Use of IDs such as FR-01 / NFR-01. requirements_with_ids / total_requirements.
Readability Metric Average sentence length quality band. Score by average words per sentence (best around 12-20 words).
Requirement Density Metric Requirement concentration per text length. Compute requirements per 1000 words, then map to target quality bands.

Additional Support Metrics (also in report)

  • FR Coverage: functional requirement signal quality.
  • NFR Coverage: non-functional signal quality.
  • Extraction Cleanliness: OCR/page/table noise penalty.
  • Content Adequacy: enough content length for reliable retrieval.
  • Duplication Quality: repeated-line ratio.
  • Numeric Specificity: measurable constraints presence in quality-related lines.
  • Lexical Richness: vocabulary diversity proxy.

Structural wording checks

The evaluator includes heuristics for imperative phrasing, weak terms, optionality, continuance phrases, figure/table references, incomplete placeholders, and vague adjectives. These are reported with arm_* field names for compatibility.

Metric field Meaning Surface formula
arm_imperative_quality How many requirement lines use imperative language (shall/must). imperative_requirement_lines / total_requirement_lines
arm_weak_phrase_quality Penalty for weak/soft phrases. 1 - (weak_phrase_count / total_requirement_lines)
arm_optionality_quality Penalty for optional words (may/could/can/optional). 1 - (option_count / total_requirement_lines)
arm_continuance_quality Penalty for continuance terms (following/as follows/listed below). 1 - (continuance_count / total_requirement_lines)
arm_directive_quality Penalty for directive references (figure/table/note). 1 - (directive_count / total_requirement_lines)
arm_incomplete_quality Penalty for incomplete placeholders (TBD/TBS/XXX). 1 - (incomplete_count / total_requirement_lines)
arm_ambiguity_quality Penalty for known ambiguous terms. 1 - (ambiguity_count / total_requirement_lines)
arm_overall_score Weighted combination of all structural wording checks. Weighted sum of the sub-scores above.

Pass/Fail Rule Used

A document is marked passed when:

  • overall_score >= pass_threshold (default 0.62), and
  • ieee_section_coverage >= 0.66.
