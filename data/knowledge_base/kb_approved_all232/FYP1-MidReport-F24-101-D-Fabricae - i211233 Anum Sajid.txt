Chapter 1 
 
 
Introduction 
 
 In today’s technology -centric world, emerging fashion designers face a growing need to 
maximize their resources, achieve valuable outcomes and market their skills. Recognizing these 
issues, our web application, Fabricae, will provide a complete platform to help independent and 
small-scale fashion designers overcome this resource constraints and simplify their design 
process. Our project helps designers create and showcase their de- signs in easy to follow steps. 
Firstly, designers can either generate a textile motif from scratch by using AI prompts or they 
can upload their own sketches which would be con- verted to 2D motifs. Using these motifs, they 
would create and customize their design pattern. To view this design pattern on actual garments, 
the designers can translate them onto custom 3D mock ups where further 3D embellishments can 
also be added. The pro fessional portfolio creation feature, which offers customizable templates, 
allows designers to showcase their creativity effectively. By displaying their portfolios on the 
platform, designers can connect with larger companies, opening doors to new opportunities and 
help in the advancement of their careers. 
 
 
1.1 Problem Statement 
 
Our platform enables designers to create unique and high-quality design patterns. 
Designers can create these patterns either by using descriptive prompts or by 
digitalizing their own pencil sketches without needing any expensive tools or a lot of 
time. Once the user will have the patterns, he can customize a pattern of his choice 
using our customization tool. Additionally, we simplify the design process by offering 
3D mock ups where de- signers can visualize their final product on and would be able 
to add 3D embellishments on top (such as buttons and laces ). This allows them to see 
the potential in their final designs without having to commit to any more resources. 
Beyond design, our platform also acts as a networking place where users can build and 
showcase their portfolios using 
5 
our customizable templates. This feature will help them connect with the industry 
professionals and bridge the gap between the big fashion companies and independent 
designers, opening doors for new opportunities and collaborations. 
 
1.2 Scope 
 
Fabricae is a web application that will enable designers to create unique textile patterns. 
The user will begin by signing into their account. They will then have two options: 
create patterns through descriptive prompts or create patterns by changing their 
uploaded pencil sketches into digital images. Next, the user can use these patterns to 
customize it from scratch using our editing tool. This tool will enable the user to 
customize their pattern using the options to add filters, apply color changes, adjust 
alignment and rotate the motifs to create a pleasing pattern. When the pattern has been 
created, the user can then check their final output by displaying it on a few custom 3D 
clothing mock-ups (both eastern and western i.e. shirt, pant, dress, shalwar kameez, 
long shirt). Furthermore, the user can add 3D embellishments (buttons and laces) to the 
model. The user may have the option to save their created patterns in our database and 
then in future use these patterns while creating their portfolios. Moreover, the user can 
create their own portfolio using our 2 -3 portfolio templates and then publish them on 
our platform. If a designer brand wants to approach the user because of their portfolio, 
they can do so using our chat feature. 
 
1.2.1 Prompt-based Textile Pattern Generation 
 
This module allows users to type in descriptive text prompts to generate design 
patterns. Users can then select more than one pattern for further editing. The 
combination of the selected patterns will then be used in our editing tool where he 
can create a final pattern of his choice. 
Features: 
 
1. Dataset creation (collecting and captioning the images) 
 
2. Pattern generation using textual descriptive prompts. 
 
3. History feature to view the previous prompt-based motif conversions 
 
 
 
 
 
 
 
 
 
6 
 
1.2.2 Sketch-to-2D Motif Conversion 
 
In this module, users can upload sketches of their textile patterns which will be 
converted to 2D digital design. After this, the user can select it for further 
customization and editing. 
 Features: 
 
1. Dataset creation 
2. Image upload functionality for sketches 
3. Pencil sketch to digital image conversion 
4. History feature to view the previous sketch-to-2D motif conversions 
 
1.2.3 Pattern Creation and Editing 
 
This module provides tools for designers to customize their textile patterns. The 
user will be given the option to drag and drop the different patterns he selected 
from the pattern generation module onto a blank white canvas. He will then create 
a final pattern using the motifs and the customization tool which includes options 
to apply filters, color changes and alignment adjustment. 
Features: 
 
1. Converting generated motifs to editable objects 
2. Drag, drop, rotate and scale your selected motifs 
3. Change pattern colors 
4. Apply filters on patterns 
 
1.2.4 3D Visualization and Embellishment 
 
Users can wrap their created patterns onto 3D models of clothing items and apply 
embellishments like buttons and laces on top of it. In this module, the user would 
have the option to view his design patterns on a 3D model and view it from all sides. 
Features: 
 
1. Create 3D clothing mock-ups on blender 
2. Drag and position the 3D embellishments on the 3D mock-ups 
3. 360-degree view of 3D mock-ups 
7 
 
4. Applying created patterns to 3D clothing models 
5. Export designs in format such as PNG, glTF and OBJ 
6. Add watermark to protect ownership of designs 
 
 
1.2.5 Portfolio Creation 
 
This module will allow users to create fashion portfolios using our pre-defined 
templates. The users can upload their created designs, add descriptions, images and 
videos so that they can have a personalized portfolio creation experience. 
Features: 
 
1. Customizing the pre-designed Portfolio templates 
 
2. Upload functionality for designs on the templates 
 
1.2.6 Portfolio Publishing and Networking 
 
Users can publish their portfolios on our platform. This way the big fashion 
companies or the industry professionals can view these portfolios and contact them 
through a chat box. Over here the users can also share their portfolios with social 
media options for further networking and recognition. 
Features: 
 
1. Portfolio Publishing 
 
2. Messaging through chat box 
 
3. Portfolio sharing on media platforms 
 
4. Message alerts on portfolio engagement

1.3 User Classes and Characteristics 
 
 
 
User class Description 
 
Fashion Designer 
 
A fashion designer will use this app to generate different 
textile patterns either by using descriptive prompts or by 
digitalizing their pencil sketches. He may have limited 
technical skills but he has great ideas to implement. The 
designer has the ability to make clear and pleasing penci l 
sketches. The designer likes to work independently or in a 
small team. He can generate as many patterns as he wants and 
then save them as a history for future use. He is also looking 
forward to creating a strong portfolio that could gain the 
attention of industry 
Industry 
Professional 
The industry professional may not be interested in the 
entire design process but he may want to look at the 
final design outputs of different designers. He wants to 
easily navigate through the design feed and find the 
best portfolios and designs he is looking for. He may 
also want to chat with the designer in order to further 
collaborate on a future project or hire him.

2.1 Functional requirements 
 
This section describes the functional requirements of the system. These requirements are 
organized by features. 
 
2.2.1 Module 1 
 
Following are the functional requirements for Module 1: 
 
1. FR1: The system shall allow designers to generate textile patterns by entering 
text descriptions. 
2. FR2: The system shall allow designers to save their generated patterns. 
 
3. FR3: The system shall allow designers to view their generated patterns. 
 
2.2.2 Module 2 
 
Following are the functional requirements for Module 2: 
 
1. FR1: The system shall allow designers to upload pencil sketches from their local 
machine. 
2. FR2: The system shall allow designers to convert their uploaded pencil sketches 
into images. 
3. FR3: The system shall allow designers to save the generated digital pattern in his- 
tory. 
4. FR4: The system shall allow designers to view the generated digital pattern from 
history. 
 
2.2.3 Module 3 
 
Following are the functional requirements for Module 3: 
 
38 
 
1. FR1: The system shall allow designers to edit the pattern by changing its colors. 
 
2. FR2: The system shall allow designers to edit the pattern by changing the alignment 
such as rotate, scale motifs inside a pattern. 
 
3. FR3: The system shall allow the designers to crop a subpart of the pattern. 
4. FR4: The system shall allow designers to add filters on the edited patterns. 
 
2.2.4 Module 4 
 
Following are the functional requirements for Module 4: 
 
1. FR1: The system shall allow designers to select 3D clothing models from the 
given options to test their patterns. 
2. FR2: The system shall allow designers to drag and position the embellishments 
such as buttons and laces on top of the 3D model. 
3. FR3: The system shall allow designers to view the 3D mock-up in 360-degree view. 
 
4. FR4: The system shall allow designers to apply their customized patterns on the 
3D clothing model. 
5. FR5: The system shall allow designers to export their 3D designs in PNG, glTF 
and OBJ format. 
6. FR6: The system shall allow designers to protect their patterns by applying water- 
mark over them. 
 
2.2.5 Module 5 
 
Following are the functional requirements for Module 5: 
 
1. FR1: The system shall provide the designer with the option to choose any one 
template from an option of two. 
2. FR2: The system shall allow the designer to edit the portfolio template by adding 
text of his/her own choice. 
3. FR2: The system shall allow the designer to edit the portfolio template by 
uploading pattern and model images from his saved patterns and models. 
4. FR2: The system shall allow the designer to edit the text inside the portfolio 
template using bold, italic and alignment options. 
39 
 
 
2.2.6 Module 6 
 
Following are the functional requirements for Module 6: 
 
1. FR1: The system shall allow the designer to publish his/her portfolio on the plat- 
form. 
2. FR2: The system shall allow the designer to edit his profile image and name. 
3. FR3: The system shall allow the designer to add his generated and edited patterns 
in the design gallery. 
4. FR4: The system shall allow the designer to share his portfolio via WhatsApp, 
LinkedIn and Gmail. 
5. FR5: The system shall allow the designer to view notifications for message alerts. 
6. FR6: The system shall allow the Industry professional to view the design gallery 
of the designers. 
7. FR7: The system shall allow the Industry professional to view the portfolio of 
the designers. 
8. FR8: The system shall allow the Industry professional to send text messages to 
the designer. 
9. FR9: The system shall allow the designer to send text messages to the Industry 
professional once the Industry professional has initiated a chat.

Chapter 3 
System Overview 
Fabricae is a web app that is designed to ease the process of pattern creation for the fashion 
designers. Moreover, it also provides an opportunity for these designers to connect with industry 
professionals by publishing their portfolios on our platform. The project is targeting the 
independent designers as well as small-scale fashion designers who are creative enough to 
transform their thoughts and sketches into realistic design patterns. Moreover, they are also 
given the opportunity to further edit and visualize their patterns on 3D mockups. 
The platform’s architecture is a 3-Tier architecture for scalability and separation of concerns.
