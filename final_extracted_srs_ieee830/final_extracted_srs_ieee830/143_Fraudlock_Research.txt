1. INTRODUCTION
1.1 Purpose
This document outlines the requirements for FraudLock, a research project focused on developing a robust deepfake detection system using a multi-modal approach that synchronizes eye, lip, audio, and cheek muscle movements to identify manipulations in facial videos.

1.2 Scope
The research aims to create a deepfake detection system that analyzes inconsistencies between lip movements and audio, as well as the correlation between lip and cheek muscle dynamics. The proposed method uses CNNs for feature extraction, attention mechanisms for synchronization, and an MLP for classification. The goal is a cross-platform optimized app for mobile devices.

1.3 Definitions, Acronyms, Abbreviations
- CNN: Convolutional Neural Network
- MLP: Multi-Layer Perceptron
- DCNN: Deep Convolutional Neural Network
- MCMC: Markov Chain Monte Carlo
- DAG: Directed Acyclic Graph

2. OVERALL DESCRIPTION
2.1 Product Perspective
FraudLock addresses the limitations of existing deepfake detection methods (which often focus on isolated features) by using a multi-modal approach that correlates multiple facial and audio cues, aiming to reduce false positives and improve reliability.

2.2 Product Functions (Summary)
- Input Video Stream: Accept video with audio.
- Pre-Processing: Face detection (DLIB), feature localization (mouth, cheeks), audio extraction (MFCC, Mel Spectrogram), frame alignment.
- CNN for Lip Movement Feature Extraction: Extract lip movement features.
- Attention Mechanisms for Lip-Audio Synchronization: Self-attention and cross-attention to identify audio-lip mismatches.
- CNN and Cross-Attention for Lip-Cheek Muscle Correspondence: Correlate lip and cheek movements.
- Multi-Layer Perceptron (MLP) for Classification and Feature Fusion: Combine outputs from modules for final real/fake classification.

2.3 User Classes and Characteristics
- (The primary "user" is the system itself, designed to be integrated into applications. The research paper outlines the methodology and does not define specific user classes for a final application.)

3. FUNCTIONAL REQUIREMENTS
3.1 Proposed Methodology (Functional steps)
FR1: The system shall accept an input video stream with audio.
FR2: The system shall perform pre-processing including face detection (DLIB), feature localization (mouth, cheeks), audio extraction (MFCC, Mel Spectrogram), and frame alignment.
FR3: The system shall use a Convolutional Neural Network (CNN) to extract lip movement features from the pre-processed video data.
FR4: The system shall employ attention mechanisms (self-attention, audio self-attention, cross-attention) to analyze lip-audio synchronization and detect inconsistencies.
FR5: The system shall use a separate CNN and cross-attention layers to correlate lip movement features with cheek muscle movement features.
FR6: The system shall use Multi-Layer Perceptrons (MLPs) for classification from both the lip-audio and lip-cheek modules.
FR7: The system shall fuse the outputs from both MLPs to generate the final classification decision (real or fake).

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance (implied from methodology)
NFR1: The model should be efficient enough for real-time detection on mobile devices.
NFR2: The system should have high accuracy in detecting deepfake manipulations, with low false positive and false negative rates.
NFR3: The system should be robust to varying video conditions (lighting, camera angles, compression).

4.2 Security (implied)
NFR4: The system should be resilient against adversarial attacks designed to bypass detection.
