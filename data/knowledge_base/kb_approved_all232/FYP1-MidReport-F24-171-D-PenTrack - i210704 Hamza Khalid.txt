1.2 Scope
The scope of the project is to develop software that would help football players en-
hance their penalties. It would do this by analyzing body pose during the kick and the
dummy keepers’ reaction. The main functionalities of the system include pose estima-
tion, shot prediction, and goalkeeper simulation. Pose estimation tools like OpenPose[ 11]
or MediaPipe[22] will be used to extract key body points from the penalty footage. After
that the spatial features will be passed by an LSTM network to understand the sequence
of movements occurring during a penalty kick. The model would predict outcomes such
as direction and power of the shot. The direction would be classified as top left, top right,
bottom left, etc. Also some might wonder why not use the data of direction and power
from the users footage, well for it to be realistic the model should get the same amount
of information as what a keeper would get during a penalty. That amount of information
is of a very short time-frame in which a keeper only registers information till the player
strikes the ball.
The proposed system would also include a simulation in which a dummy keeper would
react to the predicted shot outcome. Based on the balls trajectory and speed, the goalkeep-
ers reaction would be classified into certain regions such as bottom left, bottom right, top
mid etc. For example, if the model predicts the ball to go to the bottom right, then a pre-
trained animation of a goalkeeper would perform a saving action towards that direction.
This simulation would provide a close to realistic training environment. This would en-
able players to make adjustments helping them improve their performance. The proposed
software would also include a user-friendly interface where players and coaches would be
able to upload videos and analyze the results. The system would prove to be a valuable
tool for players and coaches, especially in regions where access to high-level training and
keepers are limited.
Overall, the project’s scope highlights the development of a platform that would help
bridge the gap between traditional training and advanced technological analysis. Hence,
this will contribute to more effective and targeted practice sessions.

3.1 Architectural Design
The system will be divided into several interconnected modules that work together to
achieve the functionality of predicting penalty shot outcomes and simulating goalkeeper
reactions based on body pose analysis. The primary modules include:
1. Video Upload & Preprocessing : This module handles video uploads from users
and preprocesses the footage to extract relevant frames.
2. Pose Estimation Module : Uses MediaPipe to extract body pose keypoints from
the video frames.
3. Prediction Module: This module contains the machine learning model trained to
predict shot outcomes based on pose data.
4. Simulation Module: Simulates the interaction between the penalty shot and a goal-
keeper model that reacts based on the predicted outcome.
5. UI/UX Module: Provides a user interface for players and coaches to upload videos,
view simulation results, and interact with the simulation environment.

3.3 Data Design
In the Data Design section, we will outline how data will be handled and stored within
the system.
Data Structures:
1. Pose Data: This consists of 3D coordinates (x, y, z) for each key body point ex-
tracted from videos.
2. Shot Outcome Data : Stores shot characteristics like direction (top left, bottom
right, etc.) and power (low, medium, high).
3. Video Data: Raw videos uploaded by users are stored temporarily for processing,
with extracted pose data stored for long-term usage.
4. Simulation Data: Stores the results of each simulation, including the goalkeeper’s
movements and the outcome of the penalty.
Storage:
• Database: The system will use a relational database (e.g., PostgreSQL) to store
user information, processed pose data, and simulation results.
• File Storage: Videos will be stored in a cloud-based solution or file server for fast
retrieval during analysis and simulation.
