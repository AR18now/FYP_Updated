Project: SkillSpark - Gamified Learning Platform for Autistic Children
================================================================================

1. INTRODUCTION
1.1 Purpose
SkillSpark is a platform for autistic children that improves learning and emotional control through gamification, personalized according to specific needs and obsessions using AI-driven technologies with real-time video feedback and emotional analysis.

1.2 Scope
SkillSpark provides a gamified learning environment customized to each child's unique interests and obsessions, using dynamic learning exercises, real-time video feedback, emotional analysis, and sensory-friendly designs to keep children engaged and motivated.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- MLP: Multi-Layer Perceptron
- UI/UX: User Interface/User Experience

2. OVERALL DESCRIPTION
2.1 Product Perspective
SkillSpark bridges the gap between traditional education methods and the needs of autistic children, supporting development of both academic progress and emotional health.

2.2 Product Functions (Summary)
- Interactive games (Virtual Pet Game, Road Race Game)
- MLP-based character and UI/UX customization
- Caregiver dashboard for progress monitoring
- Emotional analysis via video
- Learning modules (Math, Alphabet, Social Skills, Road Safety)
- Real-time personalization based on behavior and emotions

2.3 User Classes and Characteristics
- Caregiver: Set preferences, monitor child's activity and progress
- Child (Autistic): Play integrated games, learn social and educational skills

2.4 Operating Environment
Mobile application (Unity-based), Firebase for data storage.

3. FUNCTIONAL REQUIREMENTS
3.1 Module 1 - SkillSpark Mobile App
FR1.1: The system shall allow child to choose between integrated games.
FR1.2: The system shall customize game content based on caregiver preferences.
FR1.3: The system shall provide interactive environment with rewards and encouraging comments.
FR1.4: The system shall analyze child's interactions to make real-time modifications reducing sensory load.
FR1.5: The system shall provide default sensory-friendly settings customizable by caregiver or child.
FR1.6: The system shall use scenarios within games to teach social responses.
FR1.7: The system shall analyze child's emotions by monitoring video and audio.

3.2 Module 2 - Virtual Pet Game
FR2.1: The system shall allow caregiver to select virtual pet according to preferences.
FR2.2: The system shall provide tasks for child to complete with virtual pet.
FR2.3: The system shall give positive feedback when child chooses correct option.
FR2.4: The system shall adjust task complexity based on child's reaction time.
FR2.5: The system shall integrate alphabet learning where child identifies letters or words.

3.3 Module 3 - Road Race Game
FR3.1: The system shall integrate math problems into the game.
FR3.2: The system shall randomly select car or select based on caregiver preferences, changing based on emotion analysis.
FR3.3: The system shall include road safety tasks (stopping at red lights).
FR3.4: The system shall provide immediate feedback on child's answers during race.
FR3.5: The system shall monitor child's performance in road safety tasks and provide feedback.
FR3.6: The system shall make changes to car in real-time based on child's emotions.

3.4 Module 4 - Caregiver's Dashboard
FR4.1: The system shall allow caregivers to monitor child's progress through dashboard.
FR4.2: The system shall allow caregiver to enter child preferences at game start for customization.
FR4.3: The system shall display child's performance details (game performance, response times, completed tasks).
FR4.4: The system shall analyze and identify emotional triggers detected during gameplay.
FR4.5: The system shall allow caregivers to adjust game complexity level through dashboard.
FR4.6: The system shall allow caregiver to make changes to user interface to reduce sensory overload.
FR4.7: The system shall apply caregiver-modified settings in real-time.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Restoration of Game
NFR1: System shall restore accurate last saved progress upon reloading.

4.2 Usability
NFR2: User-friendly interface with clear instructions and minimal helping screens.

4.3 Performance
NFR3: Games shall load within 15 seconds on devices with ≥ 4GB RAM.
NFR4: System shall respond to user inputs within seconds.

4.4 Security
NFR5: Real-time videos shall not be stored in any database to prevent unauthorized access.
NFR6: System shall notify caregiver and make immediate modifications without storing sensitive information.

4.5 Scalability
NFR7: Game shall run on all Android devices (smartphones, tablets) with adjustable graphic settings.
NFR8: Game shall handle 500 concurrent users without problems.
