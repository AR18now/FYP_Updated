Functional Requirements
FR-01: Search Multi-City Itinerary
Input: User enters origin, destination cities, dates, number of passengers, preferred airlines/hotels.
Processing: System retrieves relevant flight and hotel details from third-party sources.
Output: Displayed results include available itineraries with corresponding costs, baggage allowances, and refund policies.
Priority: High
FR-02: Compare Fares and Policies
Input: Selected itineraries from previous step.
Processing: System compares pricing and policy differences among chosen options.
Output: Results show detailed comparisons highlighting cost variations and specific terms.
Priority: Medium
FR-03: Reserve Seats and Add Insurance
Input: Chosen itinerary details.
Processing: System reserves selected seats and adds optional insurance coverage if requested.
Output: Confirmation message includes seat numbers, reservation ID, and insurance details.
Priority: High
FR-04: Receive E-Ticket and Store Passenger Profile
Input: Reservation confirmation.
Processing: System generates electronic tickets and stores passenger profile data securely.
Output: User receives email containing digital ticket and access to stored profile for future reference.
Priority: Low
FR-05: Handle Voucher Redemption
Input: Valid voucher code provided by user.
Processing: System verifies voucher validity and applies discount to current booking.
Output: Updated total amount displayed along with applicable discounts.
Priority: Medium
FR-06: Manage Fare Changes and Alternative Options
Input: Current booking status and updated preferences.
Processing: System evaluates new options based on changed criteria and presents alternatives.
Output: New set of possible itineraries and associated costs presented to user.
Priority: High
