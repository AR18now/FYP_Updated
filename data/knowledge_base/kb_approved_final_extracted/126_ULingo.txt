SOFTWARE REQUIREMENTS SPECIFICATION (IEEE 830)
Project: ULingo - Voice Automated Conversational Assistant for Students (Urdu)
================================================================================

1. INTRODUCTION
1.1 Purpose
This document outlines the requirements for ULingo, a personalized Urdu voice assistant mobile app for students, leveraging Large Language Models (LLMs) to provide meaningful and contextually rich interactions for accessing opportunities and managing finances.

1.2 Scope
The app provides voice-based interaction in Urdu. Key features include integration with LinkedIn for personalized job/internship opportunities, Namaz reminders based on location (Google Maps API), student finance budgeting, and a user-friendly mobile interface (Flutter). The system involves data collection, fine-tuning the LLaMA 3.1 model for Urdu, and testing.

1.3 Definitions, Acronyms, Abbreviations
- LLM: Large Language Model
- API: Application Programming Interface
- NLP: Natural Language Processing
- Flutter: Google's UI toolkit for building natively compiled applications

2. OVERALL DESCRIPTION
2.1 Product Perspective
ULingo addresses the language barrier faced by Urdu-speaking students in accessing digital tools and opportunities by providing a voice assistant that understands and responds in their native language.

2.2 Product Functions (Summary)
- Data Collection: Gather Urdu datasets and user requirements.
- Model Fine-Tuning: Fine-tune LLaMA 3.1 for Urdu language processing.
- Personalized Opportunities: Integrate LinkedIn API for career listings, Google Maps API for location.
- Mobile App Development: Flutter-based app with voice interaction, dark/light modes, and text display.
- Testing: Unit, integration, and user acceptance testing.

2.3 User Classes and Characteristics
- Undergraduate Students: Tech-savvy, looking for academic/career opportunities.
- Graduate Students: Focused on advanced career opportunities and research.
- Educational Administrators: Oversee integration of university email systems.
- Career Counsellors: Guide students using the app's resources.

3. FUNCTIONAL REQUIREMENTS
3.1 Data Retrieval
FR1: The system shall collect data from user inputs through the mobile app.
FR2: The system shall fetch tailored opportunities from LinkedIn based on user preferences.
FR3: The system shall retrieve the current location of the user through Google Maps API.

3.2 Model Finetuning
FR1: The system shall fine-tune the LLaMA 3.1 model to improve Urdu language understanding.
FR2: The system shall process user inputs and queries in Urdu, generating accurate responses based on the fine-tuned model.

3.3 Mobile App Development
FR1: The system shall authenticate the user during login.
FR2: The system shall converse with the user in Urdu (Nastaliq).
FR3: The system shall interact with the user via voice and display chat in text.
FR4: The system shall provide dark and light modes in the app.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: The system shall have a high degree of reliability, with logging and fallback mechanisms for failures.

4.2 Usability
NFR2: The app's usability will focus on ease of learning and interaction, especially for non-technical students.
NFR3: The UI/UX design will prioritize accessibility for Urdu-speaking users.

4.3 Performance
NFR4: 95% of responses shall be delivered within 3 seconds of user input, regardless of mobile platform.

4.4 Security
NFR5: Security measures will protect user data (LinkedIn credentials, email).
NFR6: The system must resist unauthorized access, with encryption used to secure sensitive data.
NFR7: The app will ensure proper API authentication and user session management.
