3.3.2 Database
The Smart-Gallery system uses SQLite and Milvus as its database management system.
SQLite is a lightweight, file-based database that provides efficient storage and retrieval
of data. Milvus is also a lightweight, vector database that provides efficient storage and
retrieval of vector data .
Both databases are suitable for applications where simplicity and minimal setup are re-
quired.
The key features of SQLite include:
• Self-contained: No external dependencies or server setup is needed.
• Serverless: Operates directly on the file system without requiring a separate server
process
20
3.3 Data Design
• Zero Configuration: No configuration is necessary, making it easy to integrate and
use.
• Cross-platform: Works consistently across various operating systems.
• Efficient: Optimized for read-heavy operations with low memory overhead.
The key features of Milvus include:
• Vector Database: it is specifically designed to handle vector data, which is used in
AI applications like image, video and natural language processing.
• Scalability: It supports horizontal scalability, allowing to handle large-scale data
efficiently .
• High Performance: Optimized for fast retrieval of vectors, it offers low-latency
search even with billions of vector data points.
• Open Source: Milvus is open-source, with a large community and frequent updates,
making it a reliable option for long-term projects.
The combination of SQLite and Milvus is used to store image data and the embeddings
generated by the processing models used within our application such as text based search-
ing and editing ,Caption generation and image classification tags.
which are crucial for maintaining consistent states and effiecient processing within the
Smart-gallery system.
