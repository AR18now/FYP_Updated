1. INTRODUCTION
1.1 Problem Statement
Businesses face critical lack of business intelligence regarding consumer sentiments, leading to ill-informed decisions and inefficient resource usage. Comprehensive market surveys are time-consuming and resource-intensive, especially for small-scale businesses and start-ups.

1.2 Scope
DataKaleidos is a cloud-based data platform providing real-time, keyword-driven insights into market trends, consumer behavior, and competitor activities for the Consumer Packaged Goods (CPG) industry, specifically Personal Care and Beauty. Features include automated data collection from social media, e-commerce sites, blogs, forums, articles; sentiment analysis (fine-grained, aspect-based, intent-based, emotional detection); data visualization; report generation; and AI-powered chatbot for natural language queries.

1.3 Definitions, Acronyms, Abbreviations
- CPG: Consumer Packaged Goods
- NLP: Natural Language Processing
- LLM: Large Language Model
- ETL: Extract, Transform, Load

2. OVERALL DESCRIPTION
2.1 Product Perspective
DataKaleidos automates gathering and cataloguing of online data to provide market insights, eliminating need for manual surveys and providing quick access to real-time query-able insights.

2.2 Product Functions (Summary)
- Web application with registration and authentication
- Data collection from social media, e-commerce, blogs, forums, articles
- Data pre-processing (text cleaning, tokenization, stemming, lemmatization)
- Sentiment analysis (fine-grained, aspect-based, intent-based, emotional detection)
- Data visualization (line graphs, pie charts, heatmaps)
- Report generation (PDF export)
- AI-powered chatbot for natural language queries
- Cloud integration with ETL pipeline

2.3 User Classes and Characteristics
- Business Owners: Acquire real-time consumer insights, use chatbot interface
- Marketing Team: Monitor product effectiveness, assess campaign performance
- Investors: Assess market trends for informed investment decisions
- Data Analysts: Gather, clean, interpret data sets
- Industry Researchers: Conduct market research using trend analysis
- Consumers: Query system for suggestions before purchase decisions

2.4 Operating Environment
Web application, microservices architecture, cloud storage.

3. FUNCTIONAL REQUIREMENTS
3.1 Module 1 - Web Application
FR1.1: The system shall allow users to register with email, username, and password.
FR1.2: The system shall authenticate users based on login credentials before granting access.
FR1.3: The system shall provide search interface for data queries using keywords.
FR1.4: The system shall display visualizations of queried data in graphs and charts.
FR1.5: The system shall maintain query history for each user.

3.2 Module 2 - Data Collection
FR2.1: The system shall collect product reviews from online sources.
FR2.2: The system shall catalog collected data by brand and product type in Personal Care and Beauty sector.
FR2.3: The system shall update dataset regularly by running collection process ≥ every 24 hours.

3.3 Module 3 - Data Pre-processing
FR3.1: The system shall clean raw data by removing punctuation, fixing grammatical errors, normalizing text.
FR3.2: The system shall tokenize text data for sentiment analysis.
FR3.3: The system shall apply stemming and lemmatization to reduce words to root forms.
FR3.4: The system shall handle irregular data elements (abbreviations, slang, emoticons).
FR3.5: The system shall discard duplicate entries, spam, and non-English content.

3.4 Module 4 - Sentiment Analysis
FR4.1: The system shall perform sentiment analysis (score negative and positive sentiments) for each text entry.
FR4.2: The system shall identify specific product features mentioned in user comments.
FR4.3: The system shall extract topics and keywords related to products from user comments and reviews.

3.5 Module 5 - Data Visualization
FR5.1: The system shall generate line graphs showing average sentiment trends over selected time period.
FR5.2: The system shall display pie charts representing proportions of positive, negative, and neutral sentiments.
FR5.3: The system shall provide heatmaps highlighting key topics and phrases.

3.6 Module 6 - Report Generation
FR6.1: The system shall generate reports summarizing key statistics, trends, and insights based on user queries.
FR6.2: The system shall allow users to export reports as PDF documents.

3.7 Module 7 - Chat-bot Development
FR7.1: The system shall allow users to interact with chat-bot using natural language queries.
FR7.2: The system shall understand and process user queries to return relevant data and insights.
FR7.3: The system shall support multi-turn conversations with follow-up queries.

3.8 Module 8 - Cloud Integration
FR8.1: The system shall store all collected and processed data on secure cloud platform.
FR8.2: The system shall implement ETL (Extract, Transform, Load) process to automate data flow.
FR8.3: The system shall deploy platform using cloud containers for scalability and availability.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: Automated logging to detect anomalies.
NFR2: Data processing resumption from last successful step within 10 minutes after recovery.
NFR3: Graceful error handling with clear messages and logging.
NFR4: Sentiment analysis module reliability ≥ 75-90%.

4.2 Usability
NFR5: Intuitive GUI design with simple navigation.
NFR6: Consistent design and interaction patterns.
NFR7: 90% of users understand and interact with dashboard within 10 minutes without training.
NFR8: Configure and generate analytics reports within 3 clicks.

4.3 Performance
NFR9: Dashboard load interactive charts and analytics data ≤ 5 seconds for ≤ 1,000 entries.
NFR10: User authentication ≤ 2 seconds.
NFR11: PDF reports generated and downloadable ≤ 5 seconds for ≤ 1,000 entries.
NFR12: User chat history load ≤ 10 seconds.

4.4 Security
NFR13: Robust user authentication and authorization to prevent unauthorized access.
NFR14: User data (emails, passwords) securely protected using encryption.

4.5 Scalability
NFR15: Process datasets up to 5,000 entries with consistent performance.
NFR16: Architecture support adding new modules with minimal changes.
NFR17: Efficiently manage datasets up to 0.5 million CPG records for smooth data fetch and visualization.
