Project: AdInsight - Ad Performance Analytics with Glance and Emotion Detection

1. INTRODUCTION
1.1 Problem Statement
Advertisers lack critical insights into how users interact with ads: where users look, level of engagement, emotional responses. This gap leads to missed opportunities for optimizing ad content, targeting the right audience, and improving ROI.

1.2 Scope
AdInsight is a software platform integrating with web browsers via extension to detect video ads, track user interactions, and gather metrics. Features include ad detection (EasyList), gaze detection (WebGazer.js), emotion and demographic analysis (DeepFace), object detection in ads (YOLO), and analytics dashboard.

1.3 Definitions, Acronyms, Abbreviations
- AR: Augmented Reality
- AI: Artificial Intelligence
- API: Application Programming Interface

2. OVERALL DESCRIPTION
2.1 Product Perspective
AdInsight provides companies with comprehensive analytics to optimize advertising strategies through facial recognition, gaze detection, demographic analysis, and object detection.

2.2 Product Functions (Summary)
- Ad detection and localization on web pages
- User gaze tracking
- User emotion detection (happiness, sadness, surprise, anger, neutral)
- Demographic analysis (age group, gender)
- Object detection in video ads
- Heatmap generation based on gaze data
- Analytics dashboard for ad performance
- Report generation

2.3 User Classes and Characteristics
- Advertising Companies/Marketers: Primary users interested in ad performance and user engagement analytics
- Data Analysts: Interpret raw data, identify trends, prepare performance reports
- Ad Creators/Content Designers: Design video ads based on insights
- End Users/Consumers: Individuals whose interactions are tracked

2.4 Operating Environment
Browser extension (Chrome, Opera, Edge), server module, analytics frontend.

3. FUNCTIONAL REQUIREMENTS
3.1 Ad Detection
FR1.1: The browser extension shall identify and localize video ads in real-time.
FR1.2: The system shall detect video ads with ≥ 90% accuracy.
FR1.3: Detected video ads shall be linked to gaze and emotion tracking data.
FR1.4: Correlation shall include timestamps and metadata.

3.2 Gaze Detection
FR2.1: The browser extension shall track user eye movements.
FR2.2: Gaze detection shall start within 100 milliseconds after video ad starts playing.
FR2.3: The system shall track gaze at ≥ 30 FPS.
FR2.4: Heatmaps shall highlight areas users focus on most within ad.
FR2.5: Heatmaps shall refresh every 500 milliseconds.

3.3 Emotion and Demographic Analysis
FR3.1: The browser extension shall detect user emotions during ad playback.
FR3.2: Emotion detection shall occur in real-time and categorize into happiness, sadness, surprise, anger, neutral.
FR3.3: The system shall maintain emotion recognition accuracy ≥ 80%.
FR3.4: The system shall categorize age range (18-24, 25-34) and gender of users.

3.4 Object Detection in Advertisements
FR4.1: The server module shall detect objects within ad frames sent by browser extension.
FR4.2: Detected objects shall be stored with timestamps for future correlation.
FR4.3: The system shall detect objects with ≥ 60% accuracy.
FR4.4: Objects users focus on for ≥ 2 seconds shall be flagged as "objects of interest."

3.5 Data Storage and Processing
FR5.1: Browser extension shall send captured frames and data to server for storage.
FR5.2: Data shall be stored compliant with privacy regulations with encryption at rest.
FR5.3: Server module shall compile data on gaze tracking, emotion detection, demographics, objects of interest into report.
FR5.4: Reports shall include visual representations (heatmaps, charts).
FR5.5: Reports shall be exportable in multiple formats (PDF, Excel).

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance
NFR1: Ad detection and analysis ≤ 500 milliseconds of page load.
NFR2: Gaze detection latency < 100 milliseconds.
NFR3: Emotion detection from video frames ≤ 500 milliseconds.
NFR4: Server-side processing ≤ 5 seconds per video ad.

4.2 Scalability
NFR5: Platform shall support concurrent users, handling multiple ad interactions.

4.3 Availability
NFR6: System uptime ≥ 99.95%.
NFR7: Server/network failures shall retry and process incomplete/delayed frames within 20 seconds.

4.4 Security
NFR8: Data exchanged between browser extension and server encrypted with ≥ AES-256.
NFR9: GDPR compliance; no PII leaves host machine.
NFR10: Multi-factor authentication for analytics dashboard access.

4.5 Usability
NFR11: Analytics dashboard shall render insights within 2 seconds.
NFR12: Extension shall function on ≥ 95% of modern chromium browsers.

4.6 Maintainability
NFR13: Modular architecture for new features and bug fixes within 2 hours.
NFR14: Documentation coverage ≥ 90%.

4.7 Data Integrity
NFR15: Graceful error handling with retries and logging within 5 seconds.

4.8 Interoperability
NFR16: Seamless integration with WebGazer.js, EasyList, DeepFace, YOLO without performance bottlenecks.
