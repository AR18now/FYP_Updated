Chapter 1 
1 Introduction 
The project detailed in this document is a Gamified Intelligent Tutoring System (ITS) called Structify. It 
is designed to help learners grasp complex data structure concepts like linked lists, stacks, and queues 
through an interactive and engaging learning platform. The ITS addresses common difficulties students 
face when learning these subjects, aiming to improve understanding and performance by blending 
educational content with game-based elements. This project will benefit multiple readers: team 
members involved in building the system, FYP Panel evaluating its progress and supervisor guiding and 
evaluating its progress. 
1.1 Problem Statement 
In computer science education, the course of data structures is fundamental but challenging for 
students. The concepts of data structures such as linked lists, stacks and queues are abstract which 
makes it difficult for students to link their theoretical understanding of the concept with the practical 
application. Due to lack of clarity in these foundational topics students often struggle in this course 
which results in poor academic results and reduces their ability of succeeding in more advanced courses. 
Moreover, in conventional teaching, the individual needs of students are not looked after due to which 
students who need extra attention and require more effort often feel overwhelmed and disengaged. 
Studies show that learning data structures is very challenging and less motivating for novice students 
(Danielsiek et al., 2012; Qian & Lehman, 2017; Silva et al., 2019; Su & Zhang, 2021). Due to the 
difficulties encountered by students in learning data structures courses, the course is reported to result 
in high failure and dropout rates in computer science education (Gárcia- Mateos & Fernán dez - Alemán, 
2009a; Pérez - sánchez & Morais, 2016). These difficulties in learning data structures should be 
addressed to improve student performance in this course. 
1.2 Problem Solution 
To overcome the obstacles encountered in learning data structures, this project proposes a Gamified 
Intelligent Tutoring System (ITS) that changes the conventional way of learning into an entertaining, 
custom-made, and interactive way. The system’s aim is to make learning fun for students by adopting 
gamified elements based on adaptive learning and reward mechanisms while teaching complex data 
structures principles very efficiently. 
The ITS will start with an initial assessment of the user which will be used to create a personalized 
learning plan for them designed to target their specific needs. The learning plan consists of games based 
on data structure concepts. When the user is playing a specific game, code of that particular concept is 
shown along with it and relevant sections are highlighted according to the user's actions. This code 
visualization will help users understand the underlying code implementation of the data structure 
concept. Then the user will also be assessed for that concept. 
The system will monitor student progress and by utilizing adaptive learning adjusts the learning plan to 
match the user’s needs and pace. The interactive games will have real-time feedback for user’s actions 
to reinforce learning. This combination of interactive games, code visualization, assessments and 
personalization will help students master data structure concepts and create a fun and engaging 
learning environment 
1.3 Scope 
The Gamified Intelligent Tutoring System (ITS) will teach the concepts of data structure in an interactive 
and personalized environment. When the user will login to our system, they will have an introductory 
session which explains the platform and then the system conducts an initial assessment to judge the 
knowledge level of the user about data structures. 
The initial assessment consists of questions from various data structure topics to assess the user's 
knowledge about each topic and based on this initial assessment the system creates a personalized 
learning plan for the user. The personalized learning plan will be more focused towards the areas 
identified in the result of initial assessment check and will begin with foundational concepts and then 
moves to more advanced and complex topics. 
Each main topic, such as Linked Lists, Stacks, and Queues, will be divided into sub-topics. For example, 
Linked Lists will include insertion, deletion, and traversal as sub-topics. Each subtopic will then be 
broken down into multiple levels, such as insertion at the front, insertion at the back, and insertion at 
the end. Furthermore, these levels will have three difficulty stages: easy, medium, and hard. 
For instance, when learning about Linked Lists: 
 Insertion will be divided into levels like insertion at different positions (front, back, end). 
 Each level will have a progressive learning structure across three stages (easy, medium, hard), 
ensuring users grasp the concept thoroughly before moving forward. 
While learning about each concept, users will interact with gamified lessons that involve realtime code 
implementation, allowing them to visualize the underlying principles. For example, during the "stack" 
lessons, users will engage in a game that simulates pushing and popping elements, with accompanying 
code that dynamically shows how the operations are executed. Tooltips will explain the concepts being 
applied during the game. 
As users move through these levels, the system will monitor their progress and adapt the difficulty of 
games accordingly. If a user encounters difficulty with a specific stage in a level, they would be offered a 
relatively easier stage to play. Users who show mastery will be given the opportunity to advance 
through levels more quickly. 
Gamified elements will continue to motivate users. Points and badges will keep users engaged and 
provide a sense of accomplishment. The system will also offer real-time feedback after each lesson, with 
explanations of errors to help reinforce learning. 
Every concept in the system will end with a final assessment and upon clearing that assessment the 
user’s will earn rewards which will boost their confidence and will motivate them to continue further. 
Along with rewards, users will earn shareable badges to showcase their achievement. 
A user profile will track their progress, achievements, and areas of difficulty over time. Users can return 
to the platform to revisit previous lessons, refresh their knowledge, or challenge themselves with new 
material. The ITS will serve as an ongoing resource to support users throughout their learning journey, 
from beginner to advanced levels.

1.4 Modules 
Our system would be divided into the following modules: 
1.4.1 Assessment and Diagnostics: 
 This module will be responsible for assessing the initial knowledge level of users through 
diagnostic tests. It will generate personalized learning plans based on the assessment 
results and continuously evaluate users' progress throughout the course. This module 
will also provide final assessments to gauge overall comprehension and readiness for 
real-world application. 
1.4.2 Personalized Learning Path: 
 Based on the diagnostics, this module will create and manage customized learning plans 
for each user. It will organize the learning content in a sequence that suits the user's 
needs, ensuring that foundational concepts are thoroughly understood before 
progressing to more advanced topics. The module will adapt the learning path in real-
time based on the user’s ongoing performance. 
1.4.3 Gamified Learning: 
 This core module will encompass the various data structure topics, including linked lists, 
stacks and queues through gamification. Each topic will be broken down into interactive 
games. The module will ensure that users engage with the material in a meaningful way, 
bridging the gap between theory and practical application. 
1.4.4 Reward mechanism and Engagement: 
 This module will incorporate game-like elements such as points and badges and 
achievement tracking to motivate users. It will manage the rewarding system, track user 
progress, and display achievements, creating an engaging learning environment. 
1.4.5 Real-Time Feedback and Support: 
 This module will provide instant feedback to users as they complete exercises and 
assessments. It will explain errors and guide users through the correct solutions, helping 
to reinforce learning. 
1.4.6 Dynamic Code Visualization: 
 This module will generate real-time code representations as users interact with the 
system’s exercises and games. As users manipulate data structures (e.g., adding an 
element to a stack or traversing a tree), the corresponding lines of code will be 
dynamically generated and displayed. The specific line of code being executed will be 
highlighted, allowing users to directly see the connection between their actions and the 
underlying code. This feature will enhance understanding by providing immediate, visual 
feedback on how data structure operations translate into actual code, bridging the gap 
between theory and practice. 
1.4.7 User Profile Management: 
 This module will handle all aspects of user management, including account creation, 
profile management, authentication, and access control. It will ensure that users can 
securely sign up, log in, and manage their profiles. Additionally, it will track user 
progress and maintain individual learning histories, allowing users to revisit their 
learning paths at any time. 
1.4.8 Performance Analytics: 
 This module will provide analytics based on user performance and engagement. 
Learners will have access to insights into how they are progressing, which areas are 
most challenging, and overall trends in learning outcomes. This data will be crucial for 
continuously improving the system and tailoring content to better meet the needs of 
users. 
1.4.9 Settings: 
 This module will allow users to customize the platform's appearance with themes like 
light or dark mode, adjust font size and style for better readability, and personalize the 
dashboard layout. They can also set their preferred language for the interface and 
content. Additionally, the module allows users to submit feedback, helping improve the 
platform. 
These modules will work together to create a cohesive and effective learning environment, ensuring 
that users can master data structures in an engaging, personalized, and supportive setting. 
1.5 User Classes and Characteristics 
The primary users of the system will include the following classes: 
 Learner: Student with a basic understanding of programming fundamentals who want to 
improve their knowledge of data structures. 
 Administrator: Who will manage the users and oversee the operations of the system.

This chapter provides a high-level understanding of our system's structure and design. It introduces the

3.1 Architecture Design 
We have composed a layered architecture for our system. The layered architecture diagram for our 
system is organized into four main layers, each responsible for specific functions: 
1. Presentation Layer: This topmost layer handles the user interface and user interactions. It 
consists of components such as: 
 Games: User-facing games as part of the learning experience. 
 Web UI: The user interface that allows interaction with the system. 
 Code Visualization: Provides visual representations of code to aid learning. 
 Progress Visualization: Displays the progress of the learner throughout the system. 
2. Application Layer: This layer contains the core business logic that governs the system's 
operations, including: 
 Adaptive Learning: Handles personalized learning paths based on user behavior. 
 Rewards: Manages the system's reward mechanism for learners. 
 Assessments: Administers and processes tests or quizzes for users. 
 Feedback: Collects and provides feedback to learners to aid their improvement. 
3. Data Access Layer: This layer manages communication between the application logic and the 
data storage. It contains: 
 MongoDB Queries: Interacts with the database to retrieve or update information. 
 RESTful APIs: Provides an interface for external systems or services to interact with the 
system's data. 
4. Database Layer: The bottom layer is responsible for data storage, specifically 
 Database Collections: Houses the system's structured data in collections (relevant to 
MongoDB). 
The architecture follows a clean separation of concerns, ensuring each layer is focused on its specific 
tasks while communicating with adjacent layers through defined interfaces.

3.2.4 Class Diagram: 
This class diagram provides a comprehensive view of the system's architecture, illustrating the key 
components and their relationships. The main components are broken down into several classes, each 
representing distinct entities in the system. 
1. User Interface Class: This serves as the base interface for users, containing attributes like 
userID, userName, email, and password, along with basic functions like register(), login(), 
logout(), and updateProfile(). It is extended by the Admin and Learner classes. 
2. Admin Class: Inherits from the User interface and manages users through the function 
manageUsers(). 
3. Learner Class: Inherits from the User interface and has access to additional functionalities such 
as viewing rewards, feedback, progress, and taking lessons or quizzes. It maintains relationships 
with lessons, quizzes, and games. 
4. Lesson Class: Stores details of each lesson, such as lessonID, title, and associated games. It has 
functions like startLesson() and completeLesson(). Lessons are part of a Learning Path. 

5. Reward Class: Represents the rewards that users can receive. It contains details like rewardID, 
title, description, and points. The function assignReward() allows assigning rewards to learners. 
6. Game Class: Contains attributes like gameID, score, and lives. It manages the gameplay through 
methods like playGame(), checkMove(), updateScore(), and endGame(). 
7. Quiz Class: Contains the list of questions, lives, and methods such as startQuiz(), submitQuiz(), 
and calculateScore(). Each quiz has multiple questions. 
8. Question Class: Represents individual questions in quizzes, storing the question text, options, 
and correct answers. 
9. Learning Path Class: Organizes lessons and assessments, providing functions like generatePath() 
and updatePath(). It helps learners follow a structured learning process. 
10. Feedback Class: Stores feedback related to lessons or quizzes, with methods to generate 
feedback for users. 
11. Assessment Class: Holds assessment details like assessmentID, associated questions, and 
results. It has functions like startAssessment(), submitAssessment(), and calculateScore(). 
12. Progress Class: Tracks a learner's progress, including completed lessons, total score, and 
remaining lives, with methods to update and view progress. 
13. Code Visualization Class: Handles the visualization of code, helping learners understand the 
structure and logic of their solutions. It includes functions to generate and update visualizations 
of code. 
14. Score Class: Manages the scores associated with quizzes and assessments, with methods like 
calculateScore() and updateScore(). 
 
Each of these classes interacts with others through associations, including one-to-one, one-to-many, and 
many-to-many relationships, creating a robust framework for the system's functionality. The diagram 
highlights the modular structure of the system, ensuring that each part operates independently but 
contributes to the overall user learning experience.
