Project: SmartStudyCompanion - AI-Powered Adaptive Learning Platform for Programming

1. INTRODUCTION
1.1 Purpose
This document outlines the requirements for SmartStudyCompanion, an AI-powered web application that enhances programming education through personalized learning experiences using knowledge graphs and Retrieval-Augmented Generation (RAG).

1.2 Scope
The system creates personalized knowledge graphs for learners, visualizing programming concepts and their relationships. It uses a RAG-powered chatbot for context-aware answers, an adaptive assessment system that adjusts difficulty based on performance, and a recommendation engine for personalized learning paths. The system initially focuses on fundamental programming concepts.

1.3 Definitions, Acronyms, Abbreviations
- AI: Artificial Intelligence
- RAG: Retrieval-Augmented Generation
- LLM: Large Language Model
- KG: Knowledge Graph

2. OVERALL DESCRIPTION
2.1 Product Perspective
SmartStudyCompanion addresses the challenges of traditional programming education by providing an adaptive, personalized learning journey that visualizes complex concept relationships and offers one-on-one tutoring-like support through AI.

2.2 Product Functions (Summary)
- Data collection and preprocessing: Collect and clean programming books, perform chunking.
- Model building (Knowledge graph): Extract entities and relationships, create hierarchical graph.
- Model building (RAG): Generate graph embeddings, implement similarity search.
- Information retrieval: Fetch relevant subgraphs, visualize graphs, provide concept details, chatbot using RAG.
- Quiz and Assessment: Adaptive testing, track user progress, suggest topics based on performance.
- User interface: Intuitive UI, interactive visualizations, chat interface, quiz interface.

2.3 User Classes and Characteristics
- Student: Primary user, typically a university student looking to understand programming concepts (20-30 test users).
- Educator: Project supervisors and faculty evaluating system effectiveness.
- Administrator: The three team members who develop and maintain the system.

3. FUNCTIONAL REQUIREMENTS
3.1 Data Collection and Preprocessing Module
FR1: Collect programming books.
FR2: The system shall clean the collected data by removing irrelevant information and formatting inconsistencies.
FR3: The system shall perform text chunking on the cleaned data.
FR4: The system shall identify and remove duplicate data.

3.2 Model Building (Knowledge Graph) Module
FR1: The system shall select or train a model capable of extracting knowledge from preprocessed text data.
FR2: The system shall create a knowledge graph structure by identifying entities and relationships.
FR3: The system shall organize the knowledge graph in a hierarchical manner.
FR4: The system shall store the created knowledge graph data in a database.

3.3 Model Building (RAG) Module
FR1: The system shall generate graph embeddings for efficient retrieval.
FR2: The system shall implement a similarity search function on the graph embeddings.
FR3: The system shall store the generated embeddings in a database.

3.4 Information Retrieval Module
FR1: The system shall fetch relevant subgraphs based on user expertise or skills.
FR2: The system shall provide a visualization of the retrieved subgraphs.
FR3: The system shall display detailed information about concepts when a user selects a specific node.
FR4: The system shall implement a chatbot using RAG to answer user queries related to programming.

3.5 Quiz and Assessment Module
FR1: The system shall provide a quiz after the completion of each learning topic.
FR2: The system shall implement adaptive testing that adjusts question difficulty based on the user's previous answer.
FR3: The system shall present quiz questions one at a time.
FR4: The system shall track the number of consecutive incorrect answers during a quiz.
FR5: The system shall terminate the quiz if the user provides a specified number of consecutive incorrect answers.
FR6: The system shall recommend that the user revisit the topic if the quiz is terminated.
FR7: The system shall store quiz results and use them to update the user's knowledge graph.
FR8: The system shall use quiz performance to suggest relevant topics for further study.

3.6 User Interface Module
FR1: The system shall provide an intuitive and user-friendly interface for all functionalities.
FR2: The system shall implement a user login system and store login information in a secure database.
FR3: The system shall create interactive visualizations for displaying knowledge graphs.
FR4: The system shall design and implement an interface for chat interactions with the RAG-powered chatbot.
FR5: The system shall provide a dedicated interface for quizzes, including section to display one question at a time, mechanism for user input, "Next" button, immediate feedback, and quiz termination screen.
FR6: The system shall display a recommendation to revisit the topic if a quiz is terminated.

3.7 Testing Module
FR1: The system shall include a mechanism for identifying and fixing bugs.
FR2: The system shall perform unit testing to evaluate individual components.
FR3: The system shall conduct integration testing.
FR4: The system shall provide a mechanism for gathering and incorporating user feedback.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: Our system should work without any major issues for at least 14 days straight.
NFR2: A major failure is defined as users unable to login for more than 5 minutes, knowledge graph not loading/updating properly, or chatbot not responding for more than 2 minutes.
NFR3: To protect from failures, we will automatically save quiz answers every minute and keep a daily backup of the knowledge graph.
NFR4: For error detection, we'll setup monitoring that alerts us if any page takes more than 60 seconds to load, chatbot fails to respond to more than 5 consecutive queries, or more than 10 users report the same issue within an hour.

4.2 Usability
NFR5: The system shall display an explanation of a concept on the side panel when a user clicks on a node in the knowledge graph.
NFR6: The knowledge graph visualization shall provide zoom capabilities.
NFR7: The system shall use color-coding for nodes to indicate familiarity (Red = doesn't know, Green = knows).
NFR8: The system shall provide clear error messages for incorrect quiz answers.
NFR9: The web interface shall be responsive across different desktop browser window sizes.
NFR10: The quiz interface shall allow users to easily navigate between questions and submit answers with minimal clicks.
NFR11: The chatbot interface shall be easily accessible from any page.

4.3 Performance
NFR12: The system shall retrieve and visualize relevant subgraphs within 10 seconds of a user request.
NFR13: The chatbot shall provide responses to user queries within 10 seconds for 90% of queries.
NFR14: Quiz questions shall be generated and displayed within 10 seconds of the previous question being answered.
NFR15: The system shall update the user's knowledge graph within 30 seconds of completing a quiz.
NFR16: All database operations (storing and retrieving user data) shall complete within 20 seconds.

4.4 Security
NFR17: The system shall ensure that each user can only access their own learning progress, quiz results, and personal information.
NFR18: The system shall implement access controls to prevent unauthorized users from viewing or modifying the knowledge graph and learning content.
