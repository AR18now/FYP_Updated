## 1. Introduction

1. Introduction
The app features six predefined scenarios, each focused on addressing three core chal-
lenges that introverts often face: overthinking and decision-making, public speaking, and
teamwork. These scenarios mimic real-world social settings. During the simulations,
users will interact with intelligent virtual characters using voice input, guided by a large
language model (LLM) for dialogue management. This LLM will generate personalized,
context-aware responses, creating a natural conversational flow based on the user’s input.
To enhance communication skills, the app will analyze users’ speech in real-time, assess-
ing pitch, frequency, pauses, and filler words through a dedicated voice analysis model.
This analysis will provide immediate feedback on the user’s speech patterns, helping them
improve their conversational effectiveness. After completing a simulation, users will be
assigned daily tasks to practice and apply the skills they’ve learned in real-life scenarios,
fostering independence and reducing reliance on the app.
Additionally, users will be able to monitor their progress over time, receiving insights
into their growth and areas needing improvement. To enhance engagement, gamification
techniques will be employed, rewarding users with points based on their performance
during simulations and tasks. The overall goal is to provide a seamless and personalized
user experience that supports continuous improvement in social confidence.

2.3 Non-Functional Requirements
This section specifies nonfunctional requirements for our project:
2.3.1 Reliability
The system should automatically save user progress (every 5 seconds) so that minimum
data is lost in case of a system failure. In case of failure, the system should be able to
recover all the user data and progress from the last saved state. The system uptime should
be above 90%.
16
2.4 Domain Model
2.3.2 Usability
The user interface should be intuitive and easy to use so that new users can easily navigate
through the app without any training. The design of the system should not be overly
complex, ensuring that the users can complete tasks with minimum effort.
2.3.3 Performance
The system should efficiently process the user input and respond within 10 seconds of
the interaction including the simulation and feedback. The system should load predefined
scenarios within 6-7 seconds.
2.3.4 Security
The system must implement robust security measures to protect user data. Along with
that, the system should also log all user interactions and maintain these logs.
