2. Project Requirements
2.2 Functional Requirements
Listed below are the functional requirements of our system, organized by modules.
2.2.1 Module 1 - Vision Model
Following are the requirements for module 1:
1. The system shall generate a detailed report after each session, summarizing user
performance, posture errors, and corrections made.
2. The system shall track user progress over time by comparing session reports.
3. The system shall provide visual graphs or data summaries on a weekly or monthly
basis to show improvements or regressions.
4. The system shall use a computer vision model to detect key body joints and posi-
tions in real-time during exercises.
5. The system shall analyze body positions and angles to detect deviations from cor-
rect posture and identify common mistakes (e.g., slouching, incorrect limb angles).
6. The system shall provide real-time feedback for incorrect posture via audio or on-
screen text.
7. The system shall offer specific guidance to correct posture errors (e.g., "Raise your
arms higher" or "Straighten your back").
2.2.2 Module 2 - Chat Bot
Following are the requirements for module 2:
1. The system shall allow users to input personal data (e.g., height, weight, arthritis
severity).
2. The system shall collect feedback from users on exercise difficulty, pain level, and
progress.
3. The system shall track exercise completion using data from the computer vision
model.
4. The system shall generate personalized physiotherapy routines based on user inputs
(e.g., arthritis severity, physical ability).

The architecture diagram illustrates a multi-layered structure comprising three primary

3.3 Data Design
This JSON schema defines key components essential for the system. The User component
stores personal details and session data related to physiotherapy routines. The Session
component details each routine, including timestamps and specific exercises performed.
The Exercise component outlines each activity, specifying repetitions and targeted joints.
The VisionModel assists in posture correction, while UserFeedback captures pain and dif-
ficulty ratings to personalize routines. ProgressTracking monitors user recovery over time,
and the Chatbot provides tailored exercise suggestions and nutrition plans. Lastly, the Re-
port component generates performance summaries, offering insights into user progress.
