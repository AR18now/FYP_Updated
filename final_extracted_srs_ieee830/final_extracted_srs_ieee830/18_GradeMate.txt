1. INTRODUCTION
1.1 Problem Statement
Evaluation of handwritten programming quizzes in C++ is time-consuming and error-prone. Traditional manual grading results in variability and vagueness, leading to unfair evaluation. Teachers spend excessive time collecting, checking, and grading submissions, leaving less time for interactive teaching.

1.2 Scope
GradeMate automates grading of handwritten C++ programming quizzes. Features include handwriting recognition to convert handwritten C++ code to digital format, evaluation using predefined grading rubrics, AI-generated solutions for multiple correct responses, secure cloud storage, batch processing, and plagiarism detection.

1.3 Definitions, Acronyms, Abbreviations
- OCR: Optical Character Recognition
- AI: Artificial Intelligence
- TA: Teaching Assistant
- UI: User Interface

2. OVERALL DESCRIPTION
2.1 Product Perspective
GradeMate automates grading, reduces manual effort, and provides fair evaluation by comparing student submissions against multiple correct solutions.

2.2 Product Functions (Summary)
- Handwritten quiz upload
- OCR conversion and text extraction
- Template matching
- Model training with hybrid datasets
- Automated solution generation
- Customizable grading rubrics
- Plagiarism detection
- Secure cloud storage
- Batch processing

2.3 User Classes and Characteristics
- Instructors: Define grading criteria, approve rubrics, oversee grading process
- Students: Submit handwritten quizzes, receive evaluations
- Teaching Assistants (TA): Manage quiz submissions, review automated grading, make manual adjustments

2.4 Operating Environment
Mobile application with dashboard, cloud storage integration.

3. FUNCTIONAL REQUIREMENTS
3.1 Collecting Dataset Module
FR1.1: Teachers and TAs shall upload handwritten quiz submissions in JPEG and PNG formats.
FR1.2: The system shall automatically extract text from handwritten submissions using OCR.
FR1.3: The system shall enable integration with external datasets (IAM, MNIST, InfityCBD-2) for model training.
FR1.4: The system shall track every quiz submission in database, associated with student ID and quiz number.

3.2 Model Training Module
FR2.1: The system shall enable use of hybrid datasets (EMNIST, student quizzes) to train machine learning models.
FR2.2: The system shall enable saving trained model for further use and upgrades.
FR2.3: The system shall allow modification of model parameters (learning rate, batch size, epochs).
FR2.4: The system shall produce performance metrics (accuracy, recall, F1-score) after training.
FR2.5: The system shall allow re-training using new datasets or student submissions.

3.3 Solution Generation Module
FR3.1: Teachers shall create custom grading guidelines and rubrics for specific questions.
FR3.2: The system shall compare student handwritten solution with model-generated solution using text similarity algorithm.
FR3.3: The system shall generate alternate proper solutions if students solved using different logic but reached correct answer.
FR3.4: The system shall record differences between student and generated solutions for later analysis.

3.4 Automated Grading Module
FR4.1: The system shall automatically evaluate each quiz submission using established rubrics.
FR4.2: The system shall enable teachers to grade assignments manually, modifying or overturning automatically generated grades.
FR4.3: The system shall link each grade to appropriate quiz submission in database.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Reliability
NFR1: MTBF ≥ 1,000 hours.
NFR2: System shall identify grading errors or mismatches with high accuracy.
NFR3: Teachers shall access student information and grading history from most recent checkpoint.
NFR4: Error logs shall record and notify system failures.

4.2 Usability
NFR5: UI shall be intuitive and easy to use for new users.
NFR6: Teachers shall upload quizzes within three clicks.
NFR7: Students shall view graded quizzes with one click on dashboard.
NFR8: Error alerts and help for common mistakes (e.g., wrong file format).

4.3 Performance
NFR9: Process handwritten quiz upload ≤ 60 seconds with faster internet.
NFR10: Automatically graded results for ≥ 20 submissions accessible within 10 minutes.
NFR11: System shall accommodate up to 100 students and 50 teachers concurrently without performance reduction.
NFR12: Each page/activity shall load < 2 seconds.

4.4 Security
NFR13: All quiz data, student submissions, and grades shall be encrypted.
NFR14: User authentication for instructors and TAs.
NFR15: User activity logs for teachers, TAs, and students kept for 90 days.
NFR16: Access control shall ensure only authorized individuals access/edit quizzes, grades, or student information. Account locked after 3 unsuccessful access attempts.
