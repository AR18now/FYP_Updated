## 1. Introduction

1. Introduction
1.2 Scope
The scope of this project is to develop a real-time software solution, SIGNFUSION, which
will enable the conversion of speech and text into animated sign language for multimedia
content. This system is designed to provide accessibility for the deaf and hearing-impaired
community, particularly in relation to video content on platforms like YouTube. The
primary functionality of SIGNFUSION is to overlay animated sign language on top of
videos, ensuring that users can access and engage with the content without relying on
subtitles alone.
One of the key features of this project is its ability to perform real-time speech-to-sign
translation, which translates spoken language into animated American Sign Language
(ASL) gestures with low latency and high accuracy. Additionally, the system will allow
text-to-sign translation for scenarios where pre-existing subtitles or text are available. An-
other important aspect is the customization of the avatar that performs the sign language
gestures, providing users with the ability to personalize their experience.
The project will be delivered as a Chrome extension, making it easy for users to install
and use on YouTube or other supported platforms. By integrating with Chrome’s APIs,
the extension will capture audio from videos, process the speech or text, and map it to
corresponding ASL gestures in real time. The system will focus on minimizing latency
and ensuring smooth, high-quality animations that align with the spoken words.
This solution is intended to bridge the accessibility gap for the deaf and hearing-impaired,
allowing them to consume digital content in a more natural and engaging way. The scope
of the project also includes testing and optimization of the system across different environ-
ments to ensure consistent performance. The goal is to create a scalable and customizable
platform that can be expanded to other video platforms in the future.

The SIGNFUSION system is designed to provide a real-time translation of speech and

The key functionalities of SIGNFUSION include converting spoken words from video

such as adjusting the size and position of the animated avatar on the screen. The system
also allows content creators to test the accuracy of ASL translations for their videos and
administrators to update the gesture datasets, ensuring accuracy and comprehensiveness
over time.

3.4.1 Modular Program Structure
The system’s architecture is divided into the following core modules, each responsible for
distinct functionality:
• Speech-to-Text (STT) Module: This module is responsible for capturing the audio
from video content and converting it into text. It uses a combination of Chrome’s
Web Speech API and Google’s speech recognition technologies. The Web Speech
API allows the system to extract spoken language from the video, while Google’s
automatic speech recognition (ASR) processes the audio input in real-time, trans-
forming it into accurate text output. The STT module employs techniques such as
noise filtering and normalization to handle various speech patterns, accents, and
background noise, ensuring that the transcription remains clear and accurate.
• Text-to-Gesture Mapping Module: The text generated by the STT module is then
passed to the Text-to-Gesture Mapping module. This component maps recognized
words and phrases to their corresponding ASL gestures using a pre-built dataset
of gestures. The system relies on a sophisticated machine learning model, trained
with a large dataset of ASL signs, to ensure the text is contextually mapped to
appropriate ASL signs. For instance, the module accounts for different meanings
of the same word in various contexts, ensuring the right gesture is used based on
the surrounding words. The system also uses techniques such as Natural Language
Processing (NLP) to ensure the translated text maintains its semantic meaning when
mapped to gestures.
• Animation Module: Once the ASL gestures are mapped, this module is respon-
sible for rendering them as animations that are overlaid on the video. The module
uses a custom animation engine that retrieves pre-animated ASL signs and seam-
lessly synchronizes them with the speech or text content of the video. The system
ensures that the avatar used for these animations performs the signs with smooth
transitions and lifelike motion, closely following the timing of the spoken words.
The animation module also allows for customization, enabling users to adjust the
appearance of the avatar and the position/size of the animation window on the video
screen.
• Backend Communication Module: The Backend Communication module han-
dles the interaction between the Chrome extension on the client-side and the back-
end server. The backend is responsible for managing the storage of ASL gesture
data, processing user preferences, and supporting real-time data communication.
This module ensures that the Chrome extension can fetch the necessary gesture data
from the backend without significant latency. It also manages any custom configu-

The system primarily handles the following data structures:

speech inputs. Each gesture entry contains:
– Gesture ID (Primary Key)
– Gesture Name
– Gesture Animation File
– Corresponding Text and Speech Phrases

avatar appearance, size, and position for the sign language animations. Each user
entry includes:
– User ID (Primary Key)
– Avatar Settings (Position, Size)
– Interaction Logs (For performance monitoring and personalization)

Conclusion
The SIGNFUSION project set out with the ambitious goal of bridging the accessibility
gap for the deaf and hearing-impaired community by providing real-time American Sign
Language (ASL) translations for video content. Through the development of a sophisti-
cated Chrome extension, SIGNFUSION successfully addresses the challenge of making
multimedia content more inclusive by overlaying animated ASL gestures on videos, par-
ticularly on platforms such as YouTube.
Throughout the course of this project, we explored and implemented cutting-edge tech-
nologies to ensure a seamless and accurate user experience. SIGNFUSION integrates
real-time speech-to-text conversion, advanced gesture mapping, and a robust backend to
deliver smooth and efficient ASL translations. The project was structured using a multi-
tiered client-server architecture, which divides the system into key layers—presentation,
application, and data—ensuring efficient communication between the Chrome extension
and the backend. This design provides modularity, scalability, and maintainability, allow-
ing the system to evolve and expand as needed.
Key components such as the Speech-to-Text (STT) module, Gesture Mapping module,
and Backend Communication module were designed to operate efficiently in concert.
These modules were tested rigorously to ensure accuracy and performance across dif-
ferent environments. The Avatar Customizer feature provides a personalized user expe-
rience, enhancing engagement by allowing users to adjust the size and position of the
animated avatar, thereby ensuring an inclusive interface for all types of users.
