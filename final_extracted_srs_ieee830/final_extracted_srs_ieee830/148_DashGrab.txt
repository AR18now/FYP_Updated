SOFTWARE REQUIREMENTS SPECIFICATION (IEEE 830)
Project: DashGrab - Automated Shopping System for Retail Clothing Stores
================================================================================

1. INTRODUCTION
1.1 Purpose
This document outlines the requirements for DashGrab, an innovative project that automates the shopping experience in retail clothing stores by integrating Computer Vision, RFID technology, and automated billing.

1.2 Scope
The system integrates with existing CCTV infrastructure to track customers' shopping behavior (picking up/returning items). It uses RFID tags for seamless item tracking and associates products with customers automatically. Features include automated billing upon exit and security monitoring to detect shoplifting or fraud.

1.3 Definitions, Acronyms, Abbreviations
- CCTV: Closed-Circuit Television
- RFID: Radio-Frequency Identification
- AI: Artificial Intelligence

2. OVERALL DESCRIPTION
2.1 Product Perspective
DashGrab aims to streamline retail operations by eliminating manual checkouts and reducing security risks through real-time activity tracking and automated billing using existing CCTV and RFID technology.

2.2 Product Functions (Summary)
- User Identification: Identify customers upon entry using CCTV and registered profiles.
- Item Detection: Automatically detect RFID-tagged clothing items as the customer moves in the store.
- Automated Checkout: Process payment automatically when the customer exits.
- Security Monitoring: Continuous monitoring for suspicious activity (shoplifting, insufficient balance).

2.3 User Classes and Characteristics
- Customer: Main user identified upon entry; actions tracked; account billed based on shopping.
- Staff: Manage the system, ensure accurate detection, resolve issues.
- Admin: Oversee system operation, monitor performance, security, manage user permissions.

3. FUNCTIONAL REQUIREMENTS
3.1 Functional Requirements
FR1: User Identification: The system identifies customers upon entry using camera surveillance and registered profiles.
FR2: Item Detection: RFID tags attached to clothing items are detected automatically as the customer moves within the store.
FR3: Automated Checkout: Once the customer exits the store, the system automatically processes payment through the registered payment method.
FR4: Security Monitoring: Continuous monitoring for suspicious activity, such as shoplifting or insufficient balance during checkout.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Real-Time Processing
NFR1: The system must process transactions and monitor security in real-time.

4.2 Security
NFR2: The system should detect and prevent fraud, including monitoring for attempts to steal items or checkout with insufficient funds.

4.3 Availability
NFR3: The system should have high availability and minimal downtime to support continuous operations.
