1. INTRODUCTION
1.1 Purpose
This project aims to develop a Predictive Maintenance System using anomaly detection in sensor data to identify potential equipment failures for CARE Pvt. Ltd.'s IoT devices (3-Phase Current Sensor, Thermistor, Vibration Sensor) attached to motors of different equipment.

1.2 Scope
The project deals with data from vibration, temperature, and current sensors on CARE's machinery. Features include: data preprocessing (cleaning, handling missing values), anomaly detection using statistical and machine learning techniques, real-time visualization with green/red light indicators, and predictive maintenance scheduling based on historical patterns.

1.3 Definitions, Acronyms, Abbreviations
- IoT: Internet of Things
- LSTM: Long Short-Term Memory
- MTBF: Mean Time Between Failures
- PMS: Predictive Maintenance System

2. OVERALL DESCRIPTION
2.1 Product Perspective
The system supports Pakistani industry in sustaining machinery, extending equipment lifespan, and avoiding catastrophic failures through early detection of faults.

2.2 Product Functions (Summary)
- Data preprocessing (combine datasets, alter time, handle missing values)
- Model training (LSTM on labeled time series data)
- Hybrid approach (statistical modeling + machine learning)
- Equipment failure prediction
- Model visualization

2.3 User Classes and Characteristics
- Users: Access collected data, login, select city/device, manage API configuration
- System: Import CSV files, combine files, provide preprocessing options
- Users: Train models, visualize results in graphs

2.4 Operating Environment
Jupyter notebooks, Python environment, cloud data storage.
