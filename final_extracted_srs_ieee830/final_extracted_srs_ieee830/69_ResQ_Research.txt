Project: ResQ - Multi-Agent Search and Rescue Robot System

1. INTRODUCTION
1.1 Purpose
This document describes ResQ, a project focused on integrating affordance concepts into multi-agent robotic systems for search and rescue operations, particularly in unpredictable disaster environments like earthquakes.

1.2 Scope
The research aims to develop affordance-based concepts for multi-agent robots to operate in unpredictable environments. The robots will detect affordances of objects, update paths on global maps, and coordinate with each other to enhance life-saving operations.

1.3 Definitions, Acronyms, Abbreviations
- NAMO: Navigation Among Movable Obstacles
- RL: Reinforcement Learning
- CNN: Convolutional Neural Network
- A*: A* Pathfinding Algorithm

2. OVERALL DESCRIPTION
2.1 Product Perspective
ResQ addresses the gap in search and rescue robotics where current systems rely on static maps and predefined paths, failing to adapt to dynamic, unpredictable environments.

2.2 Product Functions (Summary)
- Multi-agent coordination and communication.
- Affordance detection of objects (pushability, liftability).
- Dynamic path planning and updating.
- Simulation of rescue scenarios in Gazebo.

2.3 User Classes and Characteristics
- Rescue Teams: Intended end-users who will deploy the robots in real-world disaster scenarios.

2.4 Operating Environment
Simulated in Gazebo; intended for deployment in real-world disaster zones.

3. FUNCTIONAL REQUIREMENTS
3.1 Multi-Agent Coordination
FR1.1: The system shall support the simultaneous operation of multiple robots in a simulated environment.
FR1.2: The robots shall be able to communicate and share environmental information.

3.2 Affordance Concepts
FR2.1: The robots shall be integrated with sensors to detect object affordances.

3.3 Path Planning
FR3.1: The system shall update the global path after detecting an object to facilitate rescue.

4. NON-FUNCTIONAL REQUIREMENTS
4.1 Performance
NFR1: The system must run efficiently and process real-time interactions.

4.2 Scalability
NFR2: The system must handle more complex environments.

4.3 Accuracy
NFR3: The simulation must provide accurate representations of real-world objects.
