1.2 Scope
This project focuses on developing a mobile application that serves as a personalised Urdu
voice assistant for students in Pakistan. The app’s primary goal is to provide voice-based
interaction in Urdu, ensuring accessibility for students more comfortable with their native
language. A key feature is integrating LinkedIn to deliver curated opportunities tailored
to each student’s academic and professional needs.
While the app also includes namaz reminders and a student finance budgeting feature, the
main emphasis is on offering a seamless Urdu experience and personalized support. The
project will ensure the app is user-friendly, secure, and accessible across major mobile
platforms, with ongoing updates to enhance functionality.

System Overview
This Urdu voice assistant mobile application project aims to assist students in effectively
managing their academic and personal lives. Through voice commands in Urdu, the app
enables students to receive personalized career opportunities, financial budgeting assis-
tance, and reminders, including Namaz timing alerts based on their location. The design
emphasizes the integration of LinkedIn for career opportunities and Google Maps API
for location-based services. The system’s architecture involves gathering Urdu datasets,
refining LLaMA 3.1, and creating a user-friendly mobile interface using Flutter.

The data structures in this system are created from the information domain using Firebase,
which is Flutter’s backend solution. Firebase is a NoSQL database hosted in the cloud,
storing data in a flexible, document-oriented format arranged into collections.

3.3.1 Key Data Entities
• User Data: Contains user profiles, preferences, LinkedIn credentials, and location
data. This information is stored in Firebase Firestore for real-time updates and
retrieval.
• Job Opportunities: These are fetched from LinkedIn’s API and temporarily stored
to curate personalized job suggestions based on user preferences.
• V oice Inputs and Outputs: User voice inputs and system-generated responses are
processed by the model and stored in Firestore to improve the model’s performance
and maintain interaction history.
• Namaz Reminders: Based on the user’s location, fetched using the Google Maps

3.3.2 Data Processing and Organization
Firebase supports real-time synchronization of data, making it ideal for the interactive
nature of this app. The following key processes are implemented:
• Authentication: Firebase Authentication is used to securely manage user login
credentials and session management, ensuring user data is protected.
• Data Retrieval: Data is retrieved from Firebase and external APIs (LinkedIn,
Google Maps), processed by the machine learning model, and stored in Firestore to
generate personalized recommendations and responses.
• Real-Time Syncing: Firebase ensures that changes to user data, interactions, and
preferences are synced in real-time across the user’s devices.
