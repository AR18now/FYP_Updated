1.2 Scope
The scope of this project encompasses a cloud-based platform for business data analytics.
The focus of the project will be the Consumer Packaged Goods (CPG) industry, in partic-
ular the Personal Care and Beauty industry. It will help businesses monitor market trends
and understand consumer behavior, in real-time.
The process of collecting and cataloguing online data will be automated. Sources like
social media, e-commerce sites, blogs, forums, and articles will be used. Regular updates
to the data will ensure the platform stays effective and relevant in a rapidly changing
market.
The collected data can then be processed and further analysed. This eliminates the need
for manual survey and analysis, and provides quick access to real-time query-able in-
sights. The platform will also generate reports, visualizations, and trend analysis.
An AI-powered chat-bot will allow users to ask questions and get instant responses, mak-
ing data easy to access and understand. The platform’s design will be simple and intuitive,
so users of all technical backgrounds can navigate it easily.
To support large volumes of data, the platform will use secure, scalable cloud storage.

Owner
In the CPG industry, a Business Owner is a company’s executive/product
manager who wishes to use the platform to acquire real-time insights into

searching for reviews which may be tedious and un-insightful, a consumer
will use prompt the system for suggestions before making a decision.

2.2 Functional Requirements
2.2 Functional Requirements
Given are the functional requirements of the system in natural language. This section
is organized by feature (module-wise) with functional requirements associated for each
feature.
2.2.1 Module 1 - Web Application
1. The system shall allow users to register by providing an email, username, and pass-
word.
2. The system shall authenticate users based on their login credentials before granting
access.
3. The system shall provide a search interface where users can query the data using
keywords.
4. The system shall display visualizations of the queried data in the form of graphs
and charts.
5. The system shall maintain a query history for each user, allowing them to view
previous searches.
2.2.2 Module 2 - Data Collection
1. The system shall collect product reviews from online sources.
2. The system shall catalog collected data by brand and product type in the Personal
Care and Beauty sector.
3. The system shall update the dataset regularly by running the collection process at
least once every 24 hours.
2.2.3 Module 3 - Data Pre-processing
1. The system shall clean the raw data by removing punctuation, fixing grammatical
errors, and normalizing the text.
2. The system shall tokenize the text data to prepare it for sentiment analysis.
3. The system shall apply stemming and lemmatization to reduce words to their root
forms.

2.3 Non-Functional Requirements
Non-functional requirements for the system are given below.
2.3.1 Reliability
RELI-1: To maintain reliability, the system will implement automated logging to detect
anomalies.
RELI-2: In case of system failure, data processing should resume from the last successful
step within 10 minutes after recovery.
RELI-3: The platform should handle errors gracefully, showing clear messages to users
and logging issues without crashing.
RELI-4: The sentiment analysis module should consistently provide accurate results with
at least 75-90% reliability, ensuring meaningful insights from the user reviews.
2.3.2 Usability
USE-1: The graphical user interface (GUI) shall exhibit an intuitive design and simple
navigation, adopting a user-centered approach.
USE-2: Throughout the application, a consistent design and interaction pattern shall be
maintained to enhance user familiarity
USE-3: 90% of users should be able to understand and interact with the dashboard within
10 minutes without needing training.
USE-4: Users should be able to configure and generate analytics reports within 3 clicks.

2.3.3 Performance
PER-1: The dashboard should load all interactive charts and analytics data within 5 sec-
onds for datasets containing up to 1,000 entries.
PER-2: Users should be authenticated within 2 seconds of submitting their login creden-
tials.
PER-3: PDF reports should be generated and downloadable within 5 seconds after the
user requests them, for datasets of up to 1,000 entries.
PER-4: User chat history should load within 10 seconds when accessing any previously
initiated conversation.

SEC-1: : Robust user authentication and authorization mechanisms shall be implemented
to prevent unauthorized access to user chat histories.
SEC-2: User data, including emails and passwords, must be securely protected using
encryption.
2.3.5 Scalability
SCA-1: The platform should be able to process data sets containing up to 5,000 entries
with consistent performance.
SCA-2: The architecture must support adding new modules (e.g., data sources) with min-
imal changes for future scalability.
SCA-3: The system should efficiently manage datasets up to 0.5 million CPG records,
specifically Personal Care and Beauty, for smooth data fetch and visualization.
