1.2 Scope
The principal aim of this project is to create a web application based on blockchain tech-
nology that enables educators to safely oversee and track student-related information and
actions. The application’s main goal is to offer a tamper-proof log system with immutable
records saved on the blockchain with smart contracts to guarantee data integrity. The logs
will gather data from various events about student activities, administration actions, and
faculty inputs to prevent unauthorized additions or deletions. The data will be stored in
a decentralized fashion. The solution reduces the danger of a single point of failure and
improves availability by spreading the data among several nodes.
One important component that will allow for live tracking of system activity with au-
tomated anomaly identification is real-time monitoring. Unusual activity or unautho-
rized modifications will result in immediate notifications, guaranteeing prompt reactions
to possible security risks. Furthermore, the program will facilitate faculty members in
publishing announcements straight through the system, offering a more efficient means of
communication for notifications and changes.
The system will include data visualization elements, such as graphs, charts, and log tables
with filtering options, to further improve user experience and make it simpler for users
to study data and spot trends. Every time something changes, thorough reports will be
produced, with an emphasis on detailing abnormalities or other noteworthy occurrences.
In addition, the project will guarantee compliance with data privacy and security laws and
be compatible with the university’s current infrastructure. The scope of this solution will
finally address the major issues of data integrity, availability, and transparency by offering
a transparent, safe, and effective platform for managing student data. [33].

3.1 Architectural Design
The OneBlock architecture is designed to ensure tamper-proof record-keeping within the
student portal. Instructors interact with the portal’s front end to manage courses and input
student data (grades, attendance). This data is then forwarded to the backend which trig-
gers the Log Generation process, which extracts and formats relevant log entries. These
formatted logs are then securely transmitted to the Ethereum Blockchain via the Ethereum
node, where the OneBlock Smart Contract stores them immutably. Administrators access
the OneBlock Website to view, analyze, and verify these blockchain-secured logs, ensur-
ing transparency and accountability within the educational environment.

3.3 Data Design
The OneBlock system leverages blockchain technology for secure log storage and rela-
tional databases for managing user data, course information, and other structured content.
The data design ensures that all information is stored securely, processed efficiently, and
organized for real-time access and analysis.
3.3.0.1 Key Data Structures:
1. Blockchain-Based Log Storage:
• Log Blocks: The blockchain stores logs of student activities, faculty actions,
and system changes in an immutable structure. Each log entry is stored in a
block, which includes:
– Timestamp of the event
– Activity type (e.g., login, assessment, attendance)
– User ID (instructor or student)
24
3.3 Data Design
– Course or assessment details
– Hash of the previous block for secure chaining
• Smart Contracts: Smart contracts validate the integrity of logs, ensuring that
logs are automatically recorded upon specific events (e.g., marking attendance
or grading assessments). These smart contracts ensure tamper-proofing and
data accuracy by preventing manual log manipulation.
2. Relational Databases:
• User Information Database:
– Stores user data, including faculty, students, and administrators.
– Fields include user ID, name, role (e.g., faculty or student), credentials
(securely hashed passwords), and permissions.
• Course Database: Stores course-related data such as course ID, course name,
instructor ID, student list, and relevant time schedules.
• Attendance and Assessment Records:
– A separate table for attendance logs where the instructor’s actions (mark-
ing attendance) are recorded concerning the blockchain log entry.
– Assessment records table storing grades, submissions, and evaluations,
also linked to blockchain for traceability.
3. Data Visualization & Reporting Module:
• Analytical Data Structures: In-memory data structures for aggregating, an-
alyzing, and visualizing logs. These structures pull data from the blockchain
and databases to generate charts and reports.
• Graphical Reports: The system extracts specific fields (e.g., timestamps,
activity types) from the blockchain and presents them in visual form to ad-
ministrators and faculty.
3.3.0.2 Data Storage:
• Blockchain (Decentralized Ledger): Primarily used to store logs, ensuring they
are immutable and tamper-proof.
• Relational Databases (FireBase): Used to store structured data like instructor in-
formation, course details, student names, and assessments.
