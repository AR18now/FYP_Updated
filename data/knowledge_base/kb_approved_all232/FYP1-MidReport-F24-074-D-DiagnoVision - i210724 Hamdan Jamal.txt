1. Introduction
1.2 Scope
The DiagnoVision project aims to develop a medical chatbot model for skin disease detec-
tion for three diseases which are Acne, Eczema, and Psoriasis and takes text description
and skin images as inputs. This model’s primary functionality is diagnosis and medical
advice. The users are asked about their symptoms and even upload photos of skin condi-
tions they might be suffering from so that the chatbot can help them determine the skin
condition that they might have. Besides diagnosis, DiagnoVision has two secondary fea-
tures: monitor the rapid spread of diseases within the user’s area and recommendation of
skin care depending on the current weather. To a greater extent, the chatbot offers med-
ical help to the users regarding specific skin diseases. The elements of this project are
as follows: Data input system, skin diseases detection models, real-time disease spread
information skincare advice based on the weather. The scope of this project is imple-
mentation of Natural Language Processing (NLP) and Computer Vision (CV) models, to
offer diagnosis and medical advice for three diseases. The project will concentrate on
improving user experience because of the improved and easy layouts that can be accessed
in the mobile phones. The system will be checked and validated against standard di-
agnostic protocols with provisions for modifications depending on the feedback and the
improvements in technology
1.3 Modules
DiagnoVision is structured into several key modules that perform as a medical chatbot.
These modules are listed below:
1.3.1 Module 1
In this module, the user’s textual description of the symptoms is inputted into Natural
Language models to determine if the user has a certain skin condition. The text module
will look for the symptoms in the text and will match the symptoms for a specific disease.
It serves as the diagnostic tool and works well with the image analysis module to come
up with a better diagnosis.
1. Extract and analyze key symptoms from user input
2. Match text-based symptoms with known conditions (Acne, Eczema, Psoriasis).

2.2.1 Module 1: Text-Based Symptom Analysis
Following are the requirements for module 1:
1. The system shall allow the user to input a description of their skin-related symptoms
via a text field.
2. The system shall analyze the text input using Natural Language Processing (NLP)
to identify key symptoms.

3. The system shall match identified symptoms to known conditions (Acne, Eczema,

4. The system shall provide the user with a preliminary diagnosis based on the text

three skin conditions.
5. The system shall provide a visual diagnosis along with a confidence score for each
image.

sive diagnosis.

Following are the requirements for module 3:
1. The system shall integrate the results from both the text-based analysis and the
image-based analysis.
2. The system shall use a decision-making algorithm to reconcile conflicting results
from the two sources.

2.3 Non-Functional Requirements
This section defines the quality attributes of the software that are measurable, verifiable,
and quantitative.
2.3.1 Reliability
The DiagnoVision system shall maintain a high level of reliability, that will ensure con-
tinuous and uninterrupted service for the users seeking diagnoses and medical advice.
The system must have a Mean Time Between Failures (MTBF) of at least 1,500 hours
for all critical operations, including symptom analysis and image processing. Failures
are defined as the inability to process user inputs, diagnose conditions, or display results
within the specified performance parameters. The system will implement real-time er-
ror detection mechanisms to identify and log failures, followed by automatic correction
or prompt notifications to the technical team for resolution. Additionally, redundancy
measures will be implemented to ensure minimal disruption in case of failure, providing
seamless service continuity.

input within 6 seconds for 90% of interactions.

The security requirements for DiagnoVision focus on protecting the system and its data
from unauthorized access and ensuring user privacy. These requirements are measurable
